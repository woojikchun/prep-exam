포트폴리오
→
조직 전략·목표에 맞는 투자 집합 / 전략적 가치 창출
프로그램
→
관련 프로젝트·활동 묶음 / 개별 관리보다 큰 편익 실현
프로젝트
→
일시적이고 고유한 결과물·능력·가치를 창출하는 노력
운영
→
지속적·반복적 활동 / 프로젝트 산출물의 운영·유지
가치 전달 시스템의 핵심: 조직 전략 → 포트폴리오 → 프로그램 → 프로젝트 → 산출물 → 편익 → 조직 가치

거버넌스 (Governance)

정의
조직이 목표를 달성하기 위해 통제·관리하는 프레임워크 — 권한·책임·의사결정 구조
포트폴리오 거버넌스
투자 우선순위, 자원 배분, 전략 정렬 여부 감독
프로젝트 거버넌스
PM 권한 범위, 변경 통제, 스폰서 책임, 보고 체계
감리 연계
거버넌스 구조 미비 → 책임 공백, 의사결정 지연 → 감리 지적 대상

프로젝트 환경 — 내부·외부

내부 환경 (EEF 내부)
조직 문화·구조
기존 시스템·인프라
인적 자원 역량
조직의 위험 허용 수준
외부 환경 (EEF 외부)
시장 조건·경쟁
법규·규제
사회·문화적 영향
기술 트렌드
EEF (Enterprise Environmental Factors): 프로젝트에 영향을 주는 내외부 환경 요인
OPA (Organizational Process Assets): 조직이 축적한 지식·절차·교훈·템플릿

편익 실현 (Benefits Realization)

개념
프로젝트 산출물이 실제로 조직에 편익(이익)을 가져오는 것 — 7판 핵심 강조 사항
산출물 vs 편익
산출물(Deliverable): 프로젝트가 만든 것 / 편익(Benefit): 그것이 가져온 가치
프로젝트 종료 후
편익 실현은 프로젝트 종료 이후에도 계속됨 — 운영 단계로 이어짐
감리 관점
사업 목표 달성 여부, 기대 편익 측정 계획 수립 여부 점검

테일러링 (Tailoring)

정의
프로젝트의 특성·환경·제약에 맞게 접근 방식·프로세스·산출물을 의도적으로 조정하는 것
테일러링 대상
개발 접근 방식, 프로세스, 참여, 도구, 방법, 산출물 등
테일러링 기준
프로젝트 규모·복잡도, 조직 문화, 규제 환경, 팀 역량, 고객 요구
7판 핵심 메시지
"표준을 따르는 것"이 아니라 "상황에 맞게 적용하는 것"이 올바른 PM
테일러링 4단계: ① 맥락 선택 → ② 초기 개발 접근법 수립 → ③ 조직에 맞게 조정 → ④ 프로젝트에 맞게 추가 조정

개발 접근 방식 3가지

유형특징적합 상황예시
예측형 Predictive 요구사항·범위 사전 확정
계획 중심, 순차 진행		 요구사항 안정·명확
기술적 위험 낮음		 건설, 제조, 규제 산업
적응형 Adaptive 반복·증분 개발
변화 수용, 고객 피드백 중심		 요구사항 불명확·변경 多
혁신·탐색적 프로젝트		 소프트웨어, 신제품 개발
혼합형 Hybrid 예측형 + 적응형 결합
상황에 따라 혼용		 일부는 확정, 일부는 탐색
대형 복합 프로젝트		 IT 시스템 + 인프라 통합

7판 주요 모델 (Models)

모델 이름핵심 내용활용
상황적 리더십 II
(Situational Leadership)		구성원 역량·의지에 따라 리더십 스타일 조정팀 관리
동기부여 모델
(내재적·외재적)		내재적 동기(의미·성장) vs 외재적 동기(보상·처벌)팀 개발
커뮤니케이션 모델발신자→인코딩→채널→디코딩→수신자→피드백이해관계자 소통
변화 관리 모델
(ADKAR, Kotter 8단계)		조직 변화를 단계적으로 관리하는 프레임워크조직 변화 추진
복잡성 모델
(Cynefin 프레임워크)		단순·복잡·복잡계·혼돈 4영역 구분상황 판단·대응
팀 성과 모델
(Tuckman, Drexler/Sibbet)		팀 개발 단계 및 팀 성과 향상 프레임워크팀 빌딩

Cynefin 프레임워크 (복잡성 판단)

영역특징대응 방식
단순 (Simple/Clear)원인-결과 명확, 모범 사례 존재감지 → 분류 → 대응 (Best Practice)
난잡 (Complicated)원인-결과 파악 가능, 전문가 필요감지 → 분석 → 대응 (Good Practice)
복잡 (Complex)원인-결과 사후 파악, 창발적탐색 → 감지 → 대응 (Emergent Practice)
혼돈 (Chaotic)원인-결과 불명확, 위기 상황행동 → 감지 → 대응 (Novel Practice)
IT 프로젝트의 대부분은 '복잡(Complex)' 영역 → 탐색(Probe)이 먼저, Agile·반복적 접근 적합
