핵심 용어 정의

용어정의
웹페이지인터넷을 통해 텍스트·그림·영상·음성 등을 제공하는 웹 문서
전자정부 웹사이트전자정부 서비스(법 §2⑤) 제공을 위해 구성된 웹페이지의 집합체
웹브라우저웹페이지를 해석하여 이용자에게 정보를 전달하는 프로그램
모바일 웹스마트폰 등 모바일 기기의 웹브라우저에 최적화된 웹사이트
웹표준국제 표준화 기구(ISO, W3C 등)에서 정의하는 기술표준
웹사이트 품질호환성·접근성·개방성·접속성·편의성·효율성·신뢰성 등 품질 전반
웹사이트 품질관리품질 확보를 위한 목표 설정·계획 수립·품질진단·개선 등 제반 활동
웹사이트 품질진단웹사이트 품질을 측정하여 현재 수준을 진단·평가하는 활동

웹사이트 품질관리 7대 원칙 (제4조) — 클릭하면 상세 내용 확인

행정기관등의 장은 웹사이트 품질관리 수행 시 다음 7가지 원칙을 준수하여야 한다

원칙 01
호환성
웹표준 사용
다양한 브라우저 동등 서비스
웹 표준 기술((X)HTML, CSS, DOM 등) 사용 / 모든 웹브라우저에서 동일하게 표시 / 비표준 기술(액티브X·플러그인 등) 제거
원칙 02
접근성
신체·인지적 제약 없이
정보·기능 쉽게 접근 보장
장애인·고령자 포함 모든 이용자 접근 보장 / 웹 접근성 지침(WCAG/KWCAG) 준수 / 대체 텍스트, 키보드 접근, 색상 대비 등
원칙 03
개방성
특정 검색엔진 통한
공개 정보 자유 이용 보장
robots.txt 등으로 크롤링 허용 / 로그인 없이 공개 정보 접근 가능 / 검색엔진 색인 허용 (2025년 개정: 일부 차단 가능하도록 완화)
원칙 04
접속성
응답시간·속도·용량·
링크 최적 상태 제공
페이지 로딩 속도 최적화 / 깨진 링크(오류 404) 없음 / 서버 가용성 및 응답시간 확보 / 불필요한 파일 용량 최소화
원칙 05
편의성
디자인·스타일·기능을
이용자 중심으로 구현
직관적 UI/UX 설계 / 이용자 중심 정보 구조 / 모바일 최적화 / 검색·내비게이션 편의 / 디지털 정부서비스 UI/UX 가이드라인 준수
원칙 06
효율성
유사·중복·저활용
사이트 통합·폐기
서로 유사하거나 중복되는 웹사이트 통합 / 이용 활성화 수준이 낮거나 운영 가치가 낮은 경우 폐기 / 행안부 장관의 통합·폐기 권고 대상
원칙 07
신뢰성
정보 오류·개인정보 노출·
게시판 점검 및 개선
공개 정보의 최신성·정확성 관리 / 개인정보 노출 여부 주기적 점검 / 스팸·악성 게시물 관리 / 콘텐츠 정확성 확보
각 카드 클릭 시
상세 내용 표시
7대 원칙 암기: 호·접·개·접·편·효·신 (호환·접근·개방·접속·편의·효율·신뢰)

7대 원칙 구조적 이해

#원칙핵심 키워드주요 기술/기준
01호환성웹표준 · 다양한 브라우저HTML5, CSS, DOM, ECMA-262, 비표준 기술 제거
02접근성신체·인지적 제약 없이WCAG 2.1, 한국형 웹 콘텐츠 접근성 지침(KWCAG)
03개방성검색엔진 · 공개 정보 자유robots.txt, 크롤링 허용 (2025년: 일부 차단 가능)
04접속성응답시간·속도·링크페이지 로딩, 404 오류 없음, 서버 가용성
05편의성이용자 중심 디자인UI/UX 가이드라인, 모바일 최적화
06효율성유사중복 방지 · 통합·폐기웹사이트 통합·폐기 권고 (행안부장관)
07신뢰성정보 오류·개인정보 점검콘텐츠 정확성, 개인정보 노출 점검

전자정부 웹사이트 품질진단 기준 (별표)

7대 원칙에 따른 진단 지표 — 각 영역별 세부 진단 내용

진단 지표세부 진단 내용
1.1 (X)HTML 표준 준수웹페이지 문법이 (X)HTML 기술표준을 준수하는가? 최신 웹표준(HTML5) 적용 여부
1.2 CSS 표준 준수시각적 속성(크기·색채·배치·정렬·여백 등)이 CSS 표준을 준수하는가?
1.3 문자(한글) 부호화문자 부호화 방식을 UTF-8로 적용하는가?
1.4 제어 기능 표준동적 구성·제어 기술표준(DOM, ECMA-262 등)을 적용하는가?
1.5 비표준 기술 제거비표준 기술(플래시·실버라이트·자바애플릿·액티브X·EXE파일 등) 제거 여부
2.1 기능 호환성 확보동적 기능이 모든 브라우저에서 동등하게 동작하는가?
2.2 화면 호환성 확보웹페이지가 모든 브라우저에서 동등하게 표시되는가?
호환성 핵심: HTML5+CSS 표준 / UTF-8 / 액티브X·EXE 제거 / 모든 브라우저 동등 표시
진단 지표세부 진단 내용
대체 텍스트이미지·멀티미디어 등에 대체 텍스트 제공 여부
키보드 접근성마우스 없이 키보드만으로 모든 기능 사용 가능 여부
색상 대비전경색과 배경색의 명도 대비가 충분한가?
초점 관리키보드 포커스 순서 및 시각적 표시 여부
명확한 링크링크 텍스트가 링크 목적을 명확히 설명하는가?
근거 표준한국형 웹 콘텐츠 접근성 지침(KWCAG 2.1) / WCAG 2.1 기반
접근성 = KWCAG 2.1 기반 / 대체텍스트·키보드접근·색상대비·초점 관리
진단 지표세부 진단 내용
크롤링 허용검색엔진 로봇이 공개 정보를 수집할 수 있도록 허용하는가? (robots.txt 설정)
메타데이터 제공검색엔진이 웹사이트를 효과적으로 색인할 수 있도록 메타정보 제공 여부
공개 정보 접근로그인 없이 공개된 정보에 접근할 수 있는가?
2025년 개정기존 '완전 허용' → 일부 차단 가능으로 완화 (보안·개인정보 보호 등 사유)
개방성 = 검색엔진 크롤링 허용 / 2025년 개정: 일부 차단 허용
진단 지표세부 진단 내용
응답 시간웹페이지가 웹브라우저에 표시되기까지 응답시간이 적정한가?
링크 유효성깨진 링크(404 오류 등) 없이 모든 링크가 정상 작동하는가?
파일 용량페이지 구성 파일의 용량이 최적화되어 있는가?
서버 가용성웹사이트 서버가 정상적으로 운영되어 접속 가능한가?
접속성 = 응답시간·링크 유효성·파일 용량 최적화·서버 가용성
진단 지표세부 진단 내용
이용자 중심 UI이용자 중심의 직관적 디자인 및 기능 구성 여부
모바일 최적화모바일 기기에서도 편리하게 이용 가능한가?
검색 기능이용자가 필요한 정보를 쉽게 찾을 수 있는 검색 기능 제공 여부
내비게이션메뉴 구조·사이트맵 등 내비게이션이 직관적인가?
근거 가이드디지털 정부서비스 UI/UX 가이드라인 준수
편의성 = 이용자 중심 UI·모바일 최적화·검색·내비게이션 / UI/UX 가이드라인
진단 지표세부 진단 내용
유사·중복 웹사이트기관 내·외에 유사하거나 중복되는 웹사이트가 존재하는가?
이용 활성화 수준방문자 수, 페이지뷰 등 이용률이 현저히 낮은가?
운영 가치 평가웹사이트의 목적·역할·콘텐츠의 가치가 지속 운영을 정당화하는가?
권고 처분행안부장관이 통합 또는 폐기 권고 가능 (전자정부법 제23조)
효율성 = 유사중복 웹사이트 통합·폐기 / 행안부장관 권고 처분 (전자정부법 §23)
진단 지표세부 진단 내용
정보 최신성·정확성공개된 정보가 최신의 것으로 유지되고, 오류가 없는가?
개인정보 노출웹사이트에 주민번호·연락처 등 개인정보가 노출되지 않는가?
게시판 관리스팸·불건전 게시물, 악성 코드 유포 게시물 등이 없는가?
콘텐츠 정확성공지사항·정책 안내 등 주요 콘텐츠의 내용이 정확한가?
신뢰성 = 정보 최신·정확 + 개인정보 노출 점검 + 게시판 관리

웹사이트 품질관리 추진 체계

주체역할 및 권한
행정안전부장관 ① 웹사이트 품질진단 실시 (이용률·중요도 고려 대상 선정)
② 품질진단 결과에 따른 품질개선 권고
③ 전자정부법 §23에 따른 통합 또는 폐기 처분 권고
④ 웹사이트 품질관리 교육 실시
⑤ 행정기관등에 품질진단 결과 제출 요청 가능
행정기관등의 장 ① 웹사이트 품질관리 계획 수립 의무
② 별표 품질진단 기준으로 자체 품질진단 시행 의무
③ 행안부 권고 시 품질개선·처분 이행 후 결과 통보 의무
④ 웹사이트 품질관리자 지정 가능
NIA (한국지능정보사회진흥원) ① 웹사이트 품질진단 기준 설정 지원
② 웹사이트 품질관리 관련 현황 및 법·제도 조사연구
③ 홍보·교육·컨설팅 및 이행확인 지원
④ 각종 통계 분석 및 제도 개선사항 발굴
⑤ 기타 품질관리에 필요한 업무 지원
행안부(진단·권고) → 행정기관등(자체진단·계획 의무) → NIA(전문기관·지원)
