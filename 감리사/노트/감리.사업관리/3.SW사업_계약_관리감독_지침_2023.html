상용SW 직접구매 제도 (§7·§8)

소프트웨어진흥법 제54조 (국가기관등의 상용소프트웨어 구매)
개념: 국가기관등이 상용SW가 사용되는 SW사업을 발주할 때, 상용SW를 사업자에게 일괄 구매를 맡기지 않고 발주기관이 직접 구매하는 제도 (구 분리발주)
요건기준
적용 대상 사업총 사업규모 기준 (고시 별도 규정) - 3억 이상
직접구매 대상 SW해당 사업에서 사용되는 상용SW 중 가격 5천만원 이상(부가세 포함)
5천만원 미만 포함종합쇼핑몰·디지털서비스몰 등록 SW는 5천만원 미만도 포함 가능
직접구매 대상 SW 종류 ①국가종합전자조달시스템 종합쇼핑몰 또는 디지털서비스몰 등록 SW (SaaS 포함)
직접구매 대상 SW 종류 ②SW진흥법 §20에 따른 GS인증 SW
직접구매 대상 SW 종류 ③CC인증(정보보호시스템) 및 국정원 검증·지정 SW
직접구매 대상: 5천만원↑ + (종합쇼핑몰·디지털서비스몰 OR GS인증 OR CC인증·국정원 검증)
⚡

2023년 개정 핵심 (2가지)

개정 ① SaaS 포함
기존 설치형 SW만 적용되던 직접구매 제도를 디지털서비스몰에 등록된 서비스형 소프트웨어(SaaS)까지 확대
SaaS 사업자의 정당한 대가 보장, 클라우드 생태계 지원
개정 ② 검토 간소화 60%↑
전체 구매품목 중 상용SW 비율이 60% 이상이면 별도 심의 없이 서식(구매계획서)으로 대체
기존 50% 이상 → 60% 이상으로 강화하여 행정 부담 완화
2023 개정: ①SaaS 직접구매 포함 ②60%↑ 시 서식 제출로 심의 갈음
📋

직접구매 절차 및 의무사항

  • 구매계획서 명시: 제1항 해당 사업 발주 시 별지 제2호 서식 직접구매 대상 SW 구매계획을 발주계획서 및 입찰공고문에 명시
  • 기술지원 확약서: 직접구매 시 사후기술지원 범위 명확화를 위해 해당 SW 사업자에게 기술지원 확약서 요청 가능
  • 조달청 계약 요청: 종합쇼핑몰 등록 제품은 조달청장에게 계약 체결 요청
  • 품목 내 1개 이상 포함: 동일 SW 품목 중 직접구매 대상이 1개라도 있으면 직접구매 대상으로 분류
  • 임의 직접구매: 대상 요건 미충족 시에도 기관이 자율적으로 직접구매 가능

적정 사업기간 산정 (§10)

⭐ 핵심 의무 규정
국가기관등의 장은 SW 개발사업을 추진하는 경우 소프트웨어진흥법 §45①에 따라 별표 1의 적정 사업기간 산정 기준을 활용하여 적정 사업기간을 산정하여야 한다
구분내용
산정 의무SW 개발사업 추진 시 별표1 기준 적용하여 적정 사업기간 산정 필수
과업심의위원회 요청적정 사업기간 산정을 위해 시행령 §47①①에 따라 과업심의위원회 위원장에게 요청
예외: 1억원 이하사업금액이 1억원 이하인 사업의 경우 국가기관등의 장이 직접 적정 사업기간 산정 가능
RFP 명시제안요청서에 "소프트웨어 개발사업 적정 사업기간 산정 기준에 따른 사업"임 명시 + 종합 산정서 첨부
근거 조항 참조행정기관 구축운영 지침 §4조: "사업이 제때에 발주될 수 있도록 적정 사업기간 보장" 규정
적정사업기간 산정 의무 / 1억원↓ 예외 / RFP에 명시 + 종합산정서 첨부

표준계약서 사용 (§12)

SW 표준계약서 6종 (과기부 마련, 2020.12.31 배포·시행) — 공공SW사업에서 사용 시 기술성평가에 가점 최대 5점 부여 (SW기술성 평가기준 지침 §4④⑥)
  • 사용 장점 ①: 입찰 시 기술성평가 가점(최대 5점) 부여
  • 사용 장점 ②: 하도급 승인 시 하도급 계약 적정성 판단기준에 반영 (지침 §20③)
  • 종류: SW개발·유지보수·운영 등 6종 유형별 계약서
  • 하도급: 하도급법 원사업자 → 소프트웨어사업 표준하도급계약서 사용
표준계약서 사용 시 기술성평가 가점 최대 5점 / 하도급 적정성 판단 시 반영
🏛️

과업심의위원회 (§25)

  • 역할: 과업내용 변경 심의 — 과업 증감·설계변경·재작업 등 계약 과업 내용 변경 심의
  • 재심의 기한: 계약상대자는 이의가 있는 경우 통지받은 날부터 14일 이내에 재심의 요청 가능
  • 과업 변경 기준: 과업범위는 과업심의위원회 개최 전까지 확정된 과업내용 기준
  • 계약금액 조정: 변경 후 계약금액 조정 청구 시 30일 이내 처리 (SW사업 대가산정 가이드와 연계)
과업심의위원회: 과업내용 변경 심의 / 재심의 요청 = 통지받은 날부터 14일 이내
🏗️

사업관리 (§13)

  • 사업관리 계획: 국가기관등의 장은 사업목표 달성을 위해 상세 요구사항 포함한 추진단계별 사업관리 계획 수립 — 일정·품질·위험·산출물 확인·통제
  • 사업수행계획서: 계약상대자는 계약체결 후 사업수행계획서 제출 / 발주기관은 계약이행에 필요한 정보 제공
  • 투입인력 관리 금지: FP·SLA 방식 산정 사업에 대해서는 투입인력 수·투입기간 관리 불가
FP/SLA 방식 사업 → 투입인력 수·기간 관리 불가 (성과중심 계약 원칙)
🏠

작업장소 (원격지 개발) (§14)

SW진흥법 §49③: 소프트웨어 유지·관리를 제외한 SW사업을 발주할 때 사업자가 사업수행 장소를 제안 가능함을 RFP에 명시해야 함
  • 사업자 제안 가능: 발주기관은 RFP에 사업자가 작업장소를 제안 가능함을 명시
  • 우선 검토 의무: 작업장소 협의 시 사업자가 제안한 작업장소를 우선 검토해야 함 (행안부 구축운영 지침 §14⑥)
  • 우대 가능: 원격지 개발(재택·사업자 사무실 등) 제안 시 우대 가능
  • 예외: 정보 보안 등 사업수행 장소에 대한 요건 제시 가능
원격지 개발: 사업자 제안 우선 검토 의무 / RFP에 "작업장소 제안 가능" 명시 필수

하도급 관리 (§19)

항목내용
하도급 계획서 제출사업자는 하도급 계획을 발주기관에 사전 제출
적정성 판단발주기관은 하도급 계획서 검토 후 적정성 판단 — 하도급 계약의 공정성 확보
대금지급 의무하도급 대금을 정해진 기한 내에 협력업체에 지급하도록 규정
표준계약서 연계하도급 승인 시 SW 표준계약서 사용 여부를 적정성 판단기준에 반영
하도급: 계획서 사전제출 + 발주기관 적정성 판단 / 표준계약서 사용 여부 반영

⭐ 투입인력 요구·관리 금지 (§11·§13)

⭐⭐ 시험 핵심 금지규정
국가기관등의 장은 소프트웨어 개발비·재개발비 등을 기능점수(FP) 방식 또는 SLA(서비스수준협약) 방식으로 산정한 경우
→ RFP에 투입인력의 수·인적사항·투입기간 요구 불가 (§11③)
→ 사업 추진 중 투입인력 수·투입기간 관리 불가 (§13③)
구분금지 대상예외 허용
RFP 단계투입인력 수·인적사항·투입기간 요구컨설팅, 감리, PMO, DB구축, 디지털콘텐츠 사업
사업수행 단계투입인력 수·투입기간 관리·통제컨설팅, 감리, PMO, DB구축, 디지털콘텐츠 사업

※ 예외 허용 사업: 정보화전략계획(ISP), 업무재설계(BPR), 정보시스템 구축계획 수립, 정보보안컨설팅 등 컨설팅 성격 / 감리 / PMO / DB구축 / 디지털콘텐츠 개발

FP/SLA 방식 = 투입인력 요구·관리 모두 금지 / 컨설팅·감리·PMO·DB구축은 예외
✅

제안서 작성비 보상 (§11②)

  • 근거: 소프트웨어진흥법 제52조에 따라 국가기관등의 장은 제안서 작성비의 일부를 보상하도록 노력하여야 한다 (의무 아님, 노력 의무)
  • 보상받은 제안서 활용: 국가기관등은 보상받은 제안서 내용을 제안자의 동의 없이 해당 사업에 활용 가능
  • 거부 불가: 제안서 보상을 받은 자는 특별한 사유가 없는 한 활용을 거부할 수 없음
제안서 작성비 보상 = 노력 의무(강제 아님) / 보상받은 제안서는 동의 없이 활용 가능
