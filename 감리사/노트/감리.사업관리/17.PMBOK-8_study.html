12 → 6으로 통합된 이유

통합 배경
7판 12원칙이 중복되고 추상적이라는 현장 피드백 → 88%의 실무자가 "원칙 기반 유지하되 더 명확하게" 요청
변경 방식
유사 원칙 통합 (예: 청지기+리더십 → 책임 있는 리더) + 지속가능성 신규 독립 원칙으로 승격

7 성과 영역 — 클릭해서 상세 보기

포커스 영역 개념

정의
7판에서 제거됐던 프로세스 그룹을 '비처방적' 형태로 복원 — 착수·기획·실행·감시통제·종료의 흐름 구조 회복
6판 프로세스 그룹과 차이
6판: 순차적·처방적 (반드시 이 순서로) / 8판: 기술적·유연 (접근법 무관하게 어느 프로젝트에나 적용)
포함 프로세스
40개 비처방적 프로세스 — 예측형·적응형·혼합형 모두에서 활용 가능
변경 배경
79%의 실무자가 "프로세스 구조의 명확성이 실무에 필요하다"고 응답

5 포커스 영역 (Focus Areas)

① 착수
Initiating
프로젝트 비전 수립, 조직 전략과 정렬, 헌장·스폰서 확보
Agile: 비전 선언문 / 예측형: 공식 프로젝트 헌장
② 기획
Planning
범위·일정·자원·위험·거버넌스 정의
Agile: 반복 계획 / 예측형: 상세 선행 기획·임계경로
③ 실행
Executing
팀 관리, 이해관계자 참여, 산출물 인도, 변경 처리
Agile: 스프린트 실행 / 예측형: 작업 패키지 수행
④ 감시·통제
Monitoring & Controlling
성과 추적, EVM, 위험 재평가, 변경 통제
Agile: 번다운 차트·회고 / 예측형: 정기 상태 보고
⑤ 종료
Closing
산출물 인수, 교훈 문서화, 자원 해제, 편익 이전
Agile: 최종 검토·팀 해산 / 예측형: 공식 종료 문서
포커스 영역은 엄격한 순서가 아닌 카테고리 — 예측형·적응형·혼합형 모두에서 동일하게 적용됨
6판 프로세스 그룹의 이름은 동일하나 처방적 강제성이 없어짐

AI 프로젝트 관리 통합

위상
8판에서 처음으로 AI를 독립 부록(Appendix)으로 다룸 — 7판은 단순 언급 수준
PM에서의 AI 활용
일정·비용·위험 예측, 반복 작업 자동화, 이해관계자 감정 분석, 자연어 보고서 생성
AI 프로젝트 특수성
데이터 품질 위험, 모델 편향, 설명 가능성(XAI), 윤리적 AI 거버넌스 — 위험 영역 확대
감리 연계
AI 시스템 도입 프로젝트 감리 시: 데이터 편향·모델 신뢰성·윤리 준수 여부 점검
8판 AI 메시지: AI는 PM을 대체하지 않고, 더 나은 의사결정을 지원하는 도구

지속가능성 (Sustainability) — 독립 원칙으로 승격

7판 대비 변화
7판: 일부 원칙에 언급 → 8판: 독립 원칙 "모든 프로젝트 영역에 지속가능성 통합"
3대 지속가능성
환경(E) — 탄소 발자국·자원 효율 / 사회(S) — 이해관계자 복지·형평성 / 경제(G) — 장기 재무 가치
ESG 연계
ESG(환경·사회·지배구조) 기준과 정렬 — 조직의 지속가능성 목표를 프로젝트에 반영
감리 연계
공공 IT 시스템 감리 시 ESG·지속가능성 목표 달성 여부 점검 항목 추가 예상

PMO (프로젝트 관리 오피스) 역할 강화

PMO 유형역할8판 강조점
지원형 (Supportive)템플릿·교훈·모범사례 제공지식 관리·AI 도구 통합
통제형 (Controlling)방법론·표준 준수 요구거버넌스 강화·테일러링 가이드
지시형 (Directive)프로젝트 직접 관리·PM 파견가치 전달 책임 강화
8판: PMO는 단순 지원 조직이 아닌 전략적 가치 전달의 허브로 위상 강화

조달 및 재무 영역 변화

조달 (Procurement)
독립 성과 영역에서 제거 → 다른 영역에 통합 + 별도 부록으로 상세 가이드 제공
Cost → Finance
'원가 관리' 개념이 '재무 관리(Finance)'로 확장 — 투자 수익(ROI)·편익 실현·재무 지속가능성까지 포함
거버넌스 (Governance)
'통합 관리(Integration)'가 '거버넌스'로 전환 — 의사결정 구조·책임·감독 강조
