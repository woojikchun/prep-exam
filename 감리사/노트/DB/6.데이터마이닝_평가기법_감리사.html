핵심 분류 성능 지표 전체 수식 ⭐
▾
정확도 (Accuracy) = (TP+TN) / (TP+TN+FP+FN) → 전체 중 맞게 예측한 비율 / 클래스 불균형 시 오해 소지 정밀도 (Precision) = TP / (TP+FP) → Positive로 예측한 것 중 실제 Positive 비율 → 스팸 필터: FP 최소화 (정상→스팸 처리 방지) 재현율 (Recall·민감도·TPR) = TP / (TP+FN) → 실제 Positive 중 Positive로 맞게 예측한 비율 → 암 진단·사기탐지: FN 최소화 (환자 놓침 방지) 특이도 (Specificity·TNR) = TN / (TN+FP) → 실제 Negative 중 Negative로 맞게 예측한 비율 FPR (False Positive Rate) = FP / (FP+TN) = 1 - 특이도 → ROC 곡선의 X축 F1-Score = 2 × (Precision × Recall) / (Precision + Recall) → 정밀도와 재현율의 조화평균 / 불균형 데이터 핵심 F-β Score = (1+β²) × (P × R) / (β²×P + R) → β>1: 재현율 중시 / β<1: 정밀도 중시 → F2: 재현율 2배 중시 / F0.5: 정밀도 2배 중시
정밀도-재현율 트레이드오프 ⭐
▾
  • 임계값(Threshold) 낮춤: 더 많은 것을 Positive로 예측 → 재현율↑ 정밀도↓
  • 임계값 높임: 확실한 것만 Positive로 예측 → 정밀도↑ 재현율↓
  • PR 곡선: 임계값 변화에 따른 정밀도-재현율 관계 시각화 → 클래스 불균형 데이터에서 ROC보다 유용
  • 최적 임계값: F1이 최대가 되는 지점 / 또는 비즈니스 요구사항에 맞게 조정
지표수식분모핵심 의미최소화해야 할 오류
정밀도 ⭐TP/(TP+FP)예측 Positive예측이 얼마나 정확FP (오탐) 최소화
재현율 ⭐TP/(TP+FN)실제 Positive실제를 얼마나 잡음FN (미탐) 최소화
특이도TN/(TN+FP)실제 Negative음성을 얼마나 음성으로FP 최소화
FPRFP/(FP+TN)실제 Negative음성을 양성으로 잘못 예측ROC X축
F1-Score ⭐2PR/(P+R)조화평균정밀도·재현율 균형불균형 데이터 핵심
TP=맞게양성·TN=맞게음성·FP=오탐(1종)·FN=미탐(2종) / 재현율=TP/(TP+FN)=민감도=TPR / 정밀도=TP/(TP+FP) / F1=조화평균

ROC 곡선 & AUC ⭐

ROC (Receiver Operating Characteristic): 임계값을 0→1로 변화시킬 때 TPR(재현율) vs FPR의 변화를 그린 곡선
AUC (Area Under Curve): ROC 곡선 아래 면적 — 0~1
TPR 1.0 |*·····················● (재현율) | · ● | · ● 완벽한 분류기 (AUC=1.0) 0.7 | ● | ● ↑ 우수한 모델 (AUC≈0.85~0.95) | ● 0.5 |● ←──── 무작위 분류기 (AUC=0.5, 대각선) | 0.0 +──────────────────────────────→ FPR 0.0 0.5 1.0 (1-특이도)
AUC 해석 기준 ⭐
▾
AUC 범위해석모델 수준
AUC = 1.0완벽한 분류기 — 모든 샘플 오류 없이 분류이상적(과적합 의심)
AUC 0.9~1.0우수한 분류기매우 좋음
AUC 0.7~0.9양호한 분류기좋음
AUC 0.5~0.7성능 부족개선 필요
AUC = 0.5무작위 분류기 — 동전 던지기 수준예측력 없음
AUC < 0.5무작위보다 나쁨 (예측을 반전하면 개선)레이블 반전 고려
  • AUC의 장점: 임계값에 무관한 종합적 성능 / 클래스 불균형에 상대적으로 강인 / 스케일 불변
  • AUC의 의미: 무작위로 선택한 Positive 샘플이 무작위로 선택한 Negative 샘플보다 높은 점수를 받을 확률
  • PR-AUC: 클래스 불균형이 매우 심할 때 ROC-AUC보다 PR 곡선의 AUC가 더 적합한 지표
임계값(Threshold) 조정 전략 ⭐
▾

임계값 낮춤 (예: 0.5→0.3)

  • 더 많은 것을 Positive로 예측
  • 재현율↑ (FN 감소)
  • 정밀도↓ (FP 증가)
  • 적합: 암 진단·사기 탐지·테러 탐지
  • "놓치지 않는 것"이 중요할 때

임계값 높임 (예: 0.5→0.7)

  • 확실한 것만 Positive로 예측
  • 정밀도↑ (FP 감소)
  • 재현율↓ (FN 증가)
  • 적합: 스팸 필터·광고 추천
  • "틀리지 않는 것"이 중요할 때
임계값 기본값은 0.5지만 비즈니스 목적에 따라 조정! 의료에서는 낮추고(재현율 중시), 스팸에서는 높임(정밀도 중시)
GINI 계수 & Lift Chart (이익 도표) ⭐
▾
  • GINI 계수(모델 평가용) ⭐: GINI = 2·AUC - 1 / AUC=0.8이면 GINI=0.6 / 0~1 범위 / 높을수록 좋음 (트리 분할 기준의 지니와 다름!)
  • 리프트 차트(Lift Chart) ⭐: 모델을 사용할 때 vs 무작위 선택할 때의 성능 비율
  • Lift = (모델 정밀도) / (전체 Positive 비율)
  • 해석: Lift=3이면 무작위 대비 3배 효율 → 마케팅 캠페인에 자주 활용
  • 누적 이익 곡선(Cumulative Gain Curve): 점수 높은 순으로 데이터를 선택할 때 누적 Positive 비율 시각화
GINI(모델평가) = 2 × AUC - 1 ← 지니 불순도와 다른 개념! Lift = (모델 Precision) / (전체 Positive 비율) = (TP/예측P) / (전체P/전체N) Lift > 1: 무작위보다 우수 / = 1: 무작위와 동일
ROC=TPR vs FPR / AUC=0.5무작위·1.0완벽 / PR-AUC=불균형에 적합 / GINI(평가)=2·AUC-1 / Lift=모델정밀도/기저율

데이터 누수 (Data Leakage) 유의사항 ⭐
▾
  • 데이터 누수 정의: 미래·테스트 데이터의 정보가 학습에 유입되어 과낙관적 성능을 유발
  • 타깃 누수(Target Leakage): 타깃 변수와 인과적으로 연결된 특성이 학습에 포함 (예: 대출 상환 여부 예측 시 대출 해지 날짜 포함)
  • 시간 누수(Temporal Leakage): 시계열 데이터에서 미래 데이터로 현재를 학습 (예: 주가 예측 시 미래 정보 포함)
  • 전처리 누수: 표준화·정규화 시 전체 데이터 통계를 사용 → 훈련 세트 통계만 사용하여 변환 적용 필요
  • 방지: Pipeline으로 전처리+모델을 묶어 CV 내에서 fit 수행 / 시계열=시간순 분할
표준화 시 전체 데이터로 scaler.fit() → 데이터 누수! 반드시 훈련 세트만으로 fit, 테스트에는 transform만 적용
부트스트랩 & 과적합·과소적합 진단 ⭐
▾
  • 부트스트랩(Bootstrap) 샘플링: 복원 추출로 훈련 세트 생성 → OOB(Out-of-Bag, 약 36.8%)로 검증 / 소규모 데이터에 효과적
  • 편향-분산 분해: 총 오류 = 편향² + 분산 + 노이즈
  • 훈련·검증 학습 곡선: 두 곡선의 간격으로 과적합·과소적합 진단

회귀 모델 평가 지표 ⭐

MAE (평균 절대 오차) = (1/n)·Σ|yᵢ - ŷᵢ| → 이상치에 강인 / 원래 단위 유지 / 해석 쉬움 MSE (평균 제곱 오차) = (1/n)·Σ(yᵢ - ŷᵢ)² → 큰 오차에 더 민감(제곱) / 미분 용이 / 손실함수 RMSE (평균 제곱근 오차) = √MSE → 원래 단위 유지 / MSE보다 해석 쉬움 / 가장 많이 사용 MAPE (평균 절대 백분율) = (1/n)·Σ|yᵢ-ŷᵢ|/|yᵢ| × 100% → 단위 무관 비율 / yᵢ=0이면 계산 불가 R² (결정계수) = 1 - SS_res/SS_tot = 1 - Σ(yᵢ-ŷᵢ)²/Σ(yᵢ-ȳ)² → 0~1 / 1에 가까울수록 모델이 분산 잘 설명 → 음수 가능 (기저 모델보다 나쁜 경우) Adjusted R² = 1 - (1-R²)(n-1)/(n-p-1) → 특성 수(p) 증가 시 R² 인플레이션 보정
지표범위좋은 값이상치 민감도단위
MAE[0, ∞)낮을수록낮음원래 단위
MSE[0, ∞)낮을수록높음 (제곱)단위²
RMSE[0, ∞)낮을수록높음원래 단위
MAPE[0, ∞)낮을수록중간% (단위 무관)
R²(-∞, 1]1에 가까울수록중간무차원
MAE=절대오차·이상치강인 / MSE=제곱·미분쉬움·이상치민감 / RMSE=가장많이사용·원래단위 / R²=설명력·1이상적·음수가능

군집 평가 지표 ⭐

내부 평가 (Internal) — 레이블 없이

지표설명
실루엣 계수 ⭐응집도(a) vs 분리도(b) / s=(b-a)/max(a,b) / -1~1 / 1에 가까울수록 좋음
이너셔(SSE) ⭐군집 내 거리 제곱합 / 낮을수록 좋음 / 엘보우 방법에 사용
DB Index ⭐Davies-Bouldin / 군집 내 분산/군집 간 거리 / 낮을수록 좋음
Calinski-Harabasz군집 간 분산/군집 내 분산 비율 / 높을수록 좋음
Dunn Index최소 군집 간 거리/최대 군집 직경 / 높을수록 좋음

외부 평가 (External) — 정답 레이블로

지표설명
순도 (Purity) ⭐각 군집에서 최다 클래스 비율 평균 / 0~1 / 높을수록 좋음
NMI ⭐
(Normalized Mutual Information)		군집과 실제 레이블 간 상호정보량 정규화 / 0~1
Rand Index ⭐같은 군집/다른 군집 쌍의 일치율 / 0~1
ARI
(Adjusted Rand Index)		우연 보정 Rand Index / -1~1 / 0=우연 수준
V-Measure동질성(Homogeneity)과 완전성(Completeness)의 조화평균
실루엣 계수 s(i) = (b(i) - a(i)) / max(a(i), b(i)) a(i): 같은 군집 내 평균 거리 (응집도) b(i): 가장 가까운 다른 군집과의 평균 거리 (분리도) s=1: 완벽한 군집 / s=0: 경계 / s=-1: 잘못된 군집
실루엣=응집도 vs 분리도·-1~1 / 이너셔=SSE·낮을수록 / DB Index=낮을수록 / 순도·NMI=외부평가(레이블필요)

연관 규칙 평가 & 기타 지표 ⭐

지표수식범위해석
지지도 (Support) ⭐P(A∩B)[0,1]규칙의 발생 빈도 — 낮으면 희귀 패턴
신뢰도 (Confidence) ⭐P(B|A) = sup(A∩B)/sup(A)[0,1]규칙의 신뢰성 — 높을수록 A→B 더 확실
향상도 (Lift) ⭐conf(A→B)/P(B)[0,∞)>1 양의연관 / =1 독립 / <1 음의연관
레버리지 (Leverage)sup(A∩B) - sup(A)·sup(B)[-1,1]>0 양의연관 / 절댓값 크면 강한 연관
확신 (Conviction)(1-P(B))/(1-conf(A→B))[0,∞)>1 양의연관 / 방향성 있음 (비대칭)
Jaccard 유사도|A∩B|/|A∪B|[0,1]두 항목 집합의 공통 비율
지지도=빈도·신뢰도=정확도·향상도(>1=양의연관)·레버리지·확신(방향성있음) / 향상도=1이면 A와B 독립!

정보 검색 평가 지표 ⭐ (Precision@K, NDCG, MAP)

정보 검색 핵심 지표
▾
Precision@K: 상위 K개 결과 중 관련 문서 비율 P@5 = 5개 중 관련 문서 수 / 5 Recall@K : 전체 관련 문서 중 상위 K개에 포함된 비율 AP (Average Precision): Precision@K를 관련 문서 위치마다 평균 MAP (Mean AP) : 여러 쿼리의 AP 평균 DCG@K = Σ rel(i)/log₂(i+1) ← 순위 반영 누적 이득 NDCG@K= DCG@K / IDCG@K ← 이상적 순위 대비 정규화 / 0~1 → 상위에 관련 문서일수록 높은 점수 / 추천시스템 표준
  • MRR (Mean Reciprocal Rank): 첫 번째 관련 문서 순위의 역수 평균 — 단일 정답 Q&A 시스템
  • NDCG 장점: 관련성 등급(0·1이 아닌 0~3 등)을 처리 가능 / 순위 반영

종합 평가 전략 & 감리 체크포인트

  • 단일 지표의 함정: 하나의 지표만으로 모델을 평가하면 편향된 결론 → 여러 지표를 종합적으로 검토
  • 비즈니스 목표와 정렬: 기술적 지표(AUC·F1)와 비즈니스 KPI(수익·비용 절감)를 연결
  • 기저율(Baseline) 비교 ⭐: 무작위·다수결·도메인 전문가 기준과 비교 — 모델이 기저율보다 나은지 확인
  • 훈련/검증/테스트 분리: 테스트 세트는 최종 평가에만 1회 사용 — 반복 사용 시 낙관적 편향
  • 통계적 유의성: McNemar 검정·t-검정으로 모델 간 성능 차이가 유의미한지 검증
  • 감리 체크포인트: 평가 지표 선택의 적절성 / 데이터 누수 여부 / 클래스 불균형 처리 / 테스트 세트 독립성
