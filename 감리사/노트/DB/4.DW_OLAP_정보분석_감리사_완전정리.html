데이터 웨어하우스 (Data Warehouse) 정의 ⭐

데이터 웨어하우스(DW): 조직의 의사결정 지원을 위해 다양한 소스에서 통합·정제된 대용량 데이터를 저장하는 주제 지향적·통합적·비휘발적·시간적 데이터베이스 (W. H. Inmon 정의, 1990)

DW 4대 특성 (Inmon) ⭐

특성설명
주제 지향적 (Subject-Oriented)업무 기능이 아닌 분석 주제(고객·제품·판매) 중심으로 구성
통합적 (Integrated)이기종 소스 데이터를 일관된 형식으로 통합
비휘발적 (Non-Volatile) ⭐한 번 적재된 데이터는 삭제·수정하지 않음 — 조회·추가만
시간적 (Time-Variant) ⭐시점 정보를 포함하여 역사적 데이터 보관 (5~10년)

▾

사실 테이블 & 차원 테이블 핵심 ⭐

사실 테이블 (Fact Table)

  • 측정값(Measure): 분석 대상 수치 — 매출·수량·비용·이익
  • 가산적(Additive): 모든 차원에서 합산 가능 (매출액)
  • 반가산적(Semi-Additive): 일부 차원에서만 합산 (재고 수량은 시간 차원 합산 불가)
  • 비가산적(Non-Additive): 어떤 차원도 합산 불가 (비율·퍼센트)
  • Degenerate Dimension: 차원 테이블 없이 사실 테이블에 있는 차원 키 (주문번호)
  • 팩트리스 사실 테이블: 측정값 없이 이벤트 발생 사실만 기록 (출석·커버리지)

차원 테이블 (Dimension Table)

  • 분석 관점: 사실을 분류·필터·그룹화하는 기준
  • 서로게이트 키(Surrogate Key): DW 내부에서 생성한 인조 PK (소스 키와 독립)
  • SCD (Slowly Changing Dimension) ⭐: 시간에 따라 변하는 차원 처리
  • SCD Type 1: 덮어쓰기 (이력 없음)
  • SCD Type 2: 새 행 추가 (이력 유지 — 가장 일반적)
  • SCD Type 3: 이전값 컬럼 추가 (제한적 이력)
사실테이블=수치(측정값)+차원FK / 차원테이블=분석관점 / SCD Type2=새행추가로 이력유지(가장 일반적) / 서로게이트키=DW 인조PK

OLAP (Online Analytical Processing) ⭐

OLAP: DW의 다차원 데이터를 다양한 관점과 집계 수준에서 빠르게 분석하는 기술 — E.F. Codd(관계형 DB 창시자)가 1993년 정의

OLAP 다차원 데이터 큐브 (OLAP Cube)

OLAP Cube: 여러 차원(시간·지역·제품)으로 구성된 다차원 데이터 구조 — 각 셀에 측정값(매출·수량) 저장. 3차원을 초과하면 Hypercube

OLAP 6대 기본 연산 ⭐

🔎
Roll-Up (롤업) ⭐
세부 → 요약 방향으로 집계
하위 계층 → 상위 계층
예: 월별 → 분기별 → 연도별 매출
🔬
Drill-Down (드릴다운) ⭐
요약 → 세부 방향으로 상세화
Roll-Up의 반대 방향
예: 연도별 → 분기별 → 월별
↔️
Drill-Across (드릴어크로스)
동일 레벨에서 다른 사실 테이블로 이동
예: 판매 큐브 → 재고 큐브
여러 사실 테이블 간 이동
📋
Slice (슬라이스) ⭐
하나의 차원을 특정 값으로 고정
3D → 2D (하나의 평면)
예: 시간=2024년으로 고정
🎯
Dice (다이스) ⭐
여러 차원을 범위로 선택
큐브에서 작은 부분 큐브 추출
예: 시간=2024Q1, 지역=서울·부산
🔄
Pivot (피봇) / Rotate ⭐
차원 축을 회전하여 다른 관점으로 조망
행/열 차원 교체
예: 행=제품, 열=지역 ↔ 행=지역, 열=제품
Roll-Up=세부→요약(월→분기→연) / Drill-Down=요약→세부(반대) / Slice=1차원고정 / Dice=여러차원범위선택 / Pivot=축회전

OLAP 구현 유형 ⭐

유형저장소특징장점단점
MOLAP ⭐
(Multidimensional)		다차원 배열(큐브)데이터를 전용 다차원 구조에 미리 계산·저장가장 빠른 쿼리 / 압축 효율대용량 희소 데이터 비효율 / 유연성↓
ROLAP ⭐
(Relational)		관계형 DB(DW)RDB 위에서 SQL로 OLAP 구현 — 집계 테이블·Materialized View 활용대용량 데이터 처리 / 유연성↑쿼리 느림 / SQL 복잡
HOLAP
(Hybrid)		MOLAP+ROLAP 혼합세부 데이터는 ROLAP, 집계 데이터는 MOLAPMOLAP 속도 + ROLAP 유연성구조 복잡
DOLAP
(Desktop)		클라이언트 PC소규모 큐브를 PC에 다운로드하여 분석오프라인 분석 / 빠른 응답소규모만 가능
MOLAP=속도최고·대용량불리 / ROLAP=대용량·유연·느림 / HOLAP=혼합 / 시험에서 MOLAP vs ROLAP 비교 자주 출제!

데이터 마이닝 (Data Mining) ⭐

데이터 마이닝: 대용량 데이터에서 통계·기계학습·패턴 인식 기법으로 숨겨진 패턴·지식·규칙을 발견하는 과정 — KDD(Knowledge Discovery in Databases)의 핵심 단계
데이터 마이닝 주요 기법 ⭐
▾
기법유형설명활용 예시
분류 (Classification) ⭐지도학습미리 정의된 클래스로 데이터 분류 — 결정 트리·나이브베이즈·SVM신용 평가·스팸 필터·이탈 예측
군집화 (Clustering) ⭐비지도학습유사한 데이터끼리 그룹 형성 — K-Means·계층적 군집화고객 세분화·시장 세분화
연관 규칙 (Association) ⭐비지도학습항목 간 동시 발생 패턴 발견 — Apriori·FP-Growth장바구니 분석·추천 시스템
회귀 (Regression)지도학습연속적 수치 예측 — 선형·다항·로지스틱 회귀매출 예측·부동산 가격

연관 규칙 (Association Rule) 핵심 지표 ⭐
▾
규칙 형태: A → B (A를 구매하면 B도 구매)
지지도(Support) = P(A∩B) = (A와 B 동시 구매 건수) / 전체 거래 수
신뢰도(Confidence) = P(B|A) = (A와 B 동시) / A 구매 건수
향상도(Lift) = Confidence(A→B) / P(B) = 독립일 때 대비 연관 정도
향상도 > 1: 양의 연관 / = 1: 독립 / < 1: 음의 연관
  • Apriori 알고리즘: 최소 지지도 이상의 빈발 항목 집합 탐색 → 연관 규칙 생성 / 후보 집합 생성·검증 반복
  • FP-Growth: Apriori 개선 — FP-Tree 구조로 후보 생성 없이 빈발 패턴 탐색 (빠름)
  • 장바구니 분석: "기저귀를 사면 맥주도 산다" — 유통업 진열·추천에 활용
