비정형 데이터 마이닝 개요 ⭐

비정형 데이터(Unstructured Data): 미리 정해진 스키마·형식이 없는 데이터 — 전 세계 데이터의 약 80~90%를 차지하며 전통적 RDBMS로 처리 어려움

데이터 유형 분류 ⭐

유형예시
정형(Structured)DB 테이블·엑셀·CSV
반정형(Semi-Structured)XML·JSON·HTML·이메일
비정형(Unstructured) ⭐텍스트·이미지·오디오·비디오·SNS

비정형 DM의 특성

  • 고차원성: 수만~수십만 차원 (텍스트 어휘 수)
  • 희소성(Sparsity): 대부분의 값이 0 (TF-IDF 행렬)
  • 모호성·맥락 의존: 동일 단어 다른 의미 (중의성)
  • 노이즈: 오탈자·축약어·이모지·방언
  • 비구조적 특성: 패턴 추출 전 전처리 필수

텍스트 전처리 기법 ⭐
▾
기법설명예시
토큰화 (Tokenization) ⭐텍스트를 단어·문장 단위로 분리"I love NLP" → ["I","love","NLP"]
불용어 제거 (Stopwords) ⭐분석에 불필요한 단어(조사·접속사·관사) 제거은·는·이·가·the·a·and 제거

텍스트 벡터화 — TF-IDF vs Word2Vec ⭐
▾
TF (Term Frequency) = 문서 내 단어 출현 빈도 / 총 단어 수 IDF (Inverse Document Frequency) = log(총문서수 / 단어포함문서수) TF-IDF = TF × IDF → 자주 등장하지만 모든 문서에 흔한 단어(the,is)는 낮은 가중치 → 특정 문서에만 자주 등장하는 단어에 높은 가중치 → 중요 단어 부각
비교TF-IDF (BoW)Word2Vec·임베딩
표현 방식희소 벡터 (대부분 0)밀집 벡터 (100~300차원)
의미 포착단어 빈도만 / 의미 무관유사 의미 단어 = 가까운 벡터
순서 고려불고려 (Bag of Words)문맥 창(Window) 고려
예시문서 분류·검색·키워드유사 단어 탐색·감성 분석
모델TF-IDF, BM25Word2Vec, GloVe, FastText, BERT

텍스트 전처리: 토큰화→불용어제거→표제어추출→벡터화 / TF-IDF=빈도×희소 / Word2Vec=밀집·의미포착 / NER=개체명인식

감성 분석 (Sentiment Analysis / Opinion Mining) ⭐

감성 분석: 텍스트에서 긍정·부정·중립의 감성(Sentiment)과 의견(Opinion)을 자동으로 추출·분류하는 기법 — 마케팅·CS·정치·금융에 광범위하게 활용
감성 분석 3가지 접근법 ⭐
▾
접근법방법장점단점
어휘 기반 (Lexicon-based) ⭐감성 사전(긍정어/부정어 목록)으로 점수 합산
예: SentiWordNet, KNU 감성사전		도메인 무관·빠름·레이블 불필요신조어·풍자·맥락 처리 어려움
머신러닝 기반 ⭐레이블된 데이터로 분류기 학습
Naïve Bayes·SVM·로지스틱 회귀		도메인 특화·높은 정확도레이블 데이터 필요·도메인 이전 어려움
딥러닝 기반 ⭐LSTM·CNN·BERT·GPT로 문맥 이해
사전학습+Fine-tuning		문맥·복합 감성 이해 우수대규모 데이터·GPU 필요·해석 어려움
감성 분석 세분화 수준 & 도전 과제 ⭐
▾

분석 단위별 수준

  • 문서 수준: 전체 문서의 감성 (긍정·부정)
  • 문장 수준: 각 문장의 감성 분류
  • 속성 수준(ABSA) ⭐: 특정 속성/측면에 대한 감성
    예: "배터리는 좋은데 화면이 별로" → 배터리=긍정, 화면=부정
  • 개체 수준: 특정 인물·브랜드·제품에 대한 감성

주요 도전 과제

  • 풍자·아이러니 ⭐: "진짜 최고야…(비꼬는)" — 표면 감성과 반대
  • 부정어 처리: "좋지 않다" vs "매우 좋다"
  • 비교급: "A가 B보다 낫다" — 상대적 감성
  • 도메인 의존성: "작다"=카메라선호/배터리싫음
  • 이모지·이모티콘: 😊😢 처리
  • 다국어·방언: 표준어 외 변형
감성분석=긍정·부정·중립 분류 / 어휘기반·ML기반·딥러닝기반 / ABSA=속성별감성 / 풍자·부정어 처리가 주요 도전

토픽 모델링 (Topic Modeling) ⭐

토픽 모델링: 레이블 없이 대량의 문서에서 숨겨진 주제(토픽)를 자동으로 발견하는 비지도 학습 — 문서 분류·탐색·요약에 활용
LDA (Latent Dirichlet Allocation) ⭐ — 가장 중요
▾
  • 핵심 가정: 각 문서는 여러 토픽의 혼합이고, 각 토픽은 단어의 확률 분포로 구성
  • Dirichlet 분포: 문서-토픽 분포와 토픽-단어 분포에 사전 분포로 적용
  • 생성 과정: ①토픽 혼합 비율(θ) 선택 → ②각 단어의 토픽(z) 선택 → ③해당 토픽의 단어(w) 선택
  • 추론: Gibbs Sampling·변분 추론(Variational Inference)으로 잠재 변수 추정
  • 하이퍼파라미터: K=토픽 수(사전 지정 필요) / α(문서-토픽 밀도) / β(토픽-단어 밀도)
  • 출력: 각 문서의 토픽 비율 + 각 토픽의 대표 단어 목록
  • 평가: 혼잡도(Perplexity, 낮을수록) / 토픽 일관성(Coherence, 높을수록)
LDA 결과 예시 토픽 1 (기술): AI(0.15), 딥러닝(0.12), 모델(0.10) 토픽 2 (정치): 선거(0.18), 정당(0.14), 국회(0.11) 토픽 3 (경제): 금리(0.20), 주식(0.15), 환율(0.09) 문서 1: 토픽1=0.6, 토픽2=0.1, 토픽3=0.3
기타 토픽 모델링 기법
▾
모델특징LDA 대비
LSA/LSI (잠재의미분석) ⭐SVD로 단어-문서 행렬 차원 축소확률 모델 아님 / 토픽 해석 어려움
NMF (비음수 행렬 분해)비음수 제약으로 부분 기반 표현 / 직관적 토픽해석 용이 / 확률 해석 불가
CTM (Correlated Topic Model)토픽 간 상관관계 모델링LDA 확장 / 관련 토픽 발견
BERTopicBERT 임베딩 + HDBSCAN + TF-IDF최신 · 의미 기반 · 고성능
  • LSA ⭐: 단어-문서 행렬에 SVD 적용 → 잠재 의미 공간에서 유사도 계산 / 동의어·다의어 처리
  • LSA vs LDA: LSA=SVD·결정론적·의미공간 / LDA=확률론적·토픽혼합·베이지안
LDA=문서는 토픽혼합·토픽은 단어분포·K사전지정·Gibbs Sampling / 평가=Perplexity(낮을수록)·Coherence(높을수록) / LSA=SVD차원축소

웹 마이닝 (Web Mining) ⭐

웹 마이닝: 웹 데이터(콘텐츠·구조·사용 로그)에서 유용한 지식·패턴을 추출하는 데이터 마이닝의 특수 분야
웹 마이닝 3대 유형 ⭐
▾
유형대상기법활용 예시
웹 콘텐츠 마이닝 ⭐웹 페이지 텍스트·이미지·멀티미디어텍스트 마이닝·NLP·이미지 처리뉴스 분류·제품 설명 추출
웹 구조 마이닝 ⭐웹 링크 구조 (하이퍼링크 그래프)PageRank·HITS·링크 분석검색엔진 랭킹·웹 커뮤니티
웹 사용 마이닝 ⭐서버 로그·클릭스트림·세션 데이터순서 패턴·연관 규칙·군집추천 시스템·UX 개선·광고
웹 크롤링 & 스크래핑 ⭐
▾
  • 웹 크롤러(Spider/Bot) ⭐: URL을 자동으로 순회하며 웹 페이지를 수집하는 프로그램 — 검색엔진의 기반
  • BFS vs DFS 크롤링: BFS=넓이우선(인기 페이지 먼저) / DFS=깊이우선(특정 사이트 심층)
  • robots.txt ⭐: 크롤링 허용·금지 경로를 명시한 파일 — 윤리적 크롤링의 기준
  • Scrapy·BeautifulSoup·Selenium: 파이썬 주요 크롤링 라이브러리
  • 동적 웹(JavaScript) ⭐: Selenium·Playwright로 브라우저 자동화하여 렌더링 후 수집
  • 크롤링 윤리: 과도한 요청→서버 부하 / 저작권·개인정보 고려 필수
HITS 알고리즘 ⭐ (PageRank 대안)
▾
  • HITS (Hyperlink-Induced Topic Search): Jon Kleinberg(1999) — 쿼리 의존적 링크 분석
  • 두 가지 점수: ①Hub(허브) 점수: 좋은 Authority 페이지로 많이 링크하는 페이지 / ②Authority 점수: 좋은 Hub 페이지로부터 많이 링크 받는 페이지
  • 상호 강화: 좋은 Hub은 좋은 Authority를 가리키고, 좋은 Authority는 좋은 Hub에 가리켜짐
  • 반복 업데이트: Hub↔Authority 점수를 수렴할 때까지 반복
비교PageRankHITS
의존성쿼리 독립적 (전역 계산)쿼리 의존적 (검색 후 계산)
점수단일 점수Hub + Authority 두 가지
속도빠름 (사전 계산)느림 (실시간 계산)
웹마이닝 3종: 콘텐츠(텍스트·이미지)·구조(링크·PageRank)·사용(로그·클릭스트림) / robots.txt=크롤링 규칙 / HITS=Hub+Authority

정보 검색 (Information Retrieval) ⭐

모델원리특징
불리언 모델AND·OR·NOT 논리 연산으로 문서 검색정확·단순 / 순위 없음 / 이진 판별
벡터공간 모델 (VSM) ⭐문서·쿼리를 TF-IDF 벡터로 표현 → 코사인 유사도 계산순위 가능 / 유사도 측정 / 가장 많이 사용
확률 모델 (BM25) ⭐쿼리 단어의 문서 내 확률 기반 랭킹 / 문서 길이 정규화VSM 개선 / 현재 검색엔진 표준
신경망 검색 (Dense Retrieval)BERT 임베딩으로 의미 기반 검색 / 벡터 DB 활용의미 검색·다국어·RAG 기반
코사인 유사도 cos(d,q) = (d·q) / (|d|·|q|) → 1에 가까울수록 문서와 쿼리가 유사 BM25 score(D,Q) = Σ IDF(qᵢ) · f(qᵢ,D)·(k₁+1) / (f(qᵢ,D)+k₁·(1-b+b·|D|/avgdl)) → k₁=1.2~2.0 (TF 포화), b=0.75 (문서길이 정규화)
VSM=TF-IDF 벡터·코사인유사도·순위가능 / BM25=확률기반·문서길이정규화·현재검색표준 / 밀집검색=BERT임베딩·의미기반

소셜 네트워크 분석 (SNA) ⭐

SNA (Social Network Analysis): 노드(개인·조직)와 엣지(관계·상호작용)로 이루어진 네트워크를 분석하여 영향력·커뮤니티·정보 전파 패턴을 탐구
중심성 지표 (Centrality Measures) ⭐
▾
지표정의의미활용
연결 중심성 ⭐
(Degree Centrality)		노드의 직접 연결 수 / (N-1)직접 연결 많을수록 중요인기도·즉시 영향력
근접 중심성 ⭐
(Closeness Centrality)		(N-1) / Σ 최단경로 거리다른 노드에 빨리 도달 가능정보 전파 속도·효율
매개 중심성 ⭐
(Betweenness Centrality)		다른 쌍의 최단경로 중 통과 비율정보 흐름의 병목·브로커정보 통제·게이트키퍼
고유벡터 중심성
(Eigenvector)		연결된 이웃의 중요도를 반영중요한 노드와 연결될수록 중요PageRank 원리와 동일
PageRank고유벡터 중심성 + 댐핑팩터웹·인용 네트워크 중요도검색엔진·논문 영향력
연결중심성=직접연결 수 / 근접중심성=평균 거리 짧음=빠른전파 / 매개중심성=경로 사이 병목·브로커 — 세 지표 구분 핵심!
커뮤니티 탐지 & 네트워크 특성 ⭐
▾
  • 커뮤니티(Community) ⭐: 커뮤니티 내부 연결은 밀집하고 커뮤니티 외부 연결은 드문 노드 집합
  • 모듈성(Modularity, Q) ⭐: 커뮤니티의 품질 척도 — -1~1 / 높을수록 명확한 커뮤니티 구조
  • Girvan-Newman 알고리즘: 매개 중심성 높은 엣지를 반복 제거 → 커뮤니티 분리
  • Louvain 알고리즘 ⭐: 탐욕적(Greedy) 모듈성 최적화 — 대규모 네트워크에 효율적
  • 소세계 현상(Small World) ⭐: 6단계 분리(Six Degrees of Separation) — 매우 짧은 평균 경로
  • 척도 없는 네트워크(Scale-Free) ⭐: 거듭제곱 법칙(Power Law) 연결 분포 — 허브(Hub) 노드 존재 / 인터넷·SNS 특성
  • 구조적 공백(Structural Hole): 연결되지 않은 그룹 사이를 연결하는 브로커 포지션 — 정보·권력 이점
중심성: 연결(직접연결)·근접(평균거리)·매개(병목·브로커) / Louvain=모듈성최적화 / 소세계=6단계분리 / 척도없는=허브·거듭제곱법칙

이미지·오디오·멀티미디어 마이닝 ⭐

이미지 마이닝 핵심 기법 ⭐
▾
태스크설명대표 모델
이미지 분류 ⭐이미지를 미리 정의된 클래스로 분류AlexNet·VGG·ResNet·EfficientNet
객체 탐지 ⭐이미지 내 여러 객체의 위치(Bounding Box)+클래스YOLO·SSD·Faster R-CNN
의미론적 분할 (Semantic Segmentation)픽셀 단위로 클래스 분류FCN·U-Net·DeepLab
얼굴 인식 (Face Recognition)얼굴 탐지·식별·검증FaceNet·ArcFace·DeepFace
GAN (생성적 적대 신경망)생성자+판별자 적대 학습으로 새 이미지 생성GAN·DCGAN·StyleGAN·Stable Diffusion
특성 기반 검색 (CBIR) ⭐이미지 특성(색상·질감·형태)으로 유사 이미지 검색SIFT·HOG·CNN 특성 벡터
  • CNN 특성 추출 ⭐: 합성곱 레이어가 자동으로 계층적 특성(에지→질감→객체) 학습
  • 전이 학습(Transfer Learning) ⭐: ImageNet으로 사전 학습된 모델을 새 도메인에 fine-tuning
오디오·텍스트-이미지 멀티모달 ⭐
▾
  • 음성 인식(STT·ASR) ⭐: 음성 → 텍스트 변환 / HMM → RNN → Transformer(Whisper)
  • 텍스트-음성 변환(TTS): 텍스트 → 자연스러운 음성 / WaveNet·Tacotron
  • 화자 다이어리제이션: 여러 화자의 발화를 분리·식별
  • 음악 정보 검색(MIR): 장르·템포·멜로디 분류 / 훔쳐듣기 → 노래 인식(Shazam 원리)
  • 멀티모달 AI ⭐: 텍스트+이미지+오디오를 함께 처리하는 모델 — CLIP(이미지-텍스트), GPT-4V, Gemini
  • CLIP ⭐: 이미지와 텍스트를 같은 임베딩 공간에 정렬 → 텍스트로 이미지 검색 / Zero-Shot 분류
이미지분류=ResNet / 객체탐지=YOLO·Faster R-CNN / 음성인식=Whisper / 전이학습=ImageNet사전학습+Fine-tuning / CLIP=이미지-텍스트정렬
