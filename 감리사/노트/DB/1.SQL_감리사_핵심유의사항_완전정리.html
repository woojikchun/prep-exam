SELECT 실행 순서 & NULL 처리 ⭐

SELECT 논리적 실행 순서 ⭐ (작성 순서 ≠ 실행 순서)
▾
-- 작성 순서 SELECT 컬럼 -- 5번째 실행 FROM 테이블 -- 1번째: 테이블 결정 WHERE 조건 -- 2번째: 행 필터링 (집계함수 사용 불가!) GROUP BY 그룹기준 -- 3번째: 그룹핑 HAVING 그룹조건 -- 4번째: 그룹 필터링 (집계함수 사용 가능) ORDER BY 정렬기준 -- 6번째 (SELECT 별칭 사용 가능!)
WHERE에서 집계함수(COUNT·SUM·AVG 등) 사용 불가! 집계 조건은 반드시 HAVING에서 / ORDER BY는 SELECT 별칭 사용 가능
NULL 처리 핵심 규칙 ⭐ — 가장 많은 함정 포인트
▾
  • NULL 비교는 = 사용 불가: IS NULL / IS NOT NULL 사용해야 함
  • NULL 산술 연산 결과 = NULL: NULL + 1 = NULL, NULL * 0 = NULL
  • NULL 논리 연산: NULL AND TRUE = NULL / NULL OR TRUE = TRUE / NULL AND FALSE = FALSE
  • 집계 함수와 NULL: COUNT(*) = NULL 포함 / COUNT(컬럼) = NULL 제외 / SUM·AVG·MAX·MIN도 NULL 제외
  • GROUP BY와 NULL: NULL 값도 하나의 그룹으로 처리됨
  • ORDER BY와 NULL: Oracle=NULL을 가장 큰 값, SQL Server=가장 작은 값으로 처리 (기본)
-- NULL 처리 예시 SELECT COUNT(*) FROM emp; -- NULL 포함 전체 행 수 SELECT COUNT(salary) FROM emp; -- NULL인 salary 제외 SELECT AVG(salary) FROM emp; -- NULL 제외 평균 (주의!) -- NULL 함수 처리 NVL(salary, 0) -- Oracle: NULL이면 0으로 대체 ISNULL(salary, 0) -- SQL Server: NULL이면 0으로 대체 COALESCE(a, b, c) -- 표준SQL: 첫 번째 NOT NULL 반환 NULLIF(a, b) -- a=b이면 NULL, 아니면 a 반환
AVG(salary): NULL인 행은 분모에서도 제외 → 전체 행 대비 실제 평균보다 높게 나올 수 있음! / COUNT(*)≠COUNT(컬럼)
GROUP BY · HAVING · 집계함수 유의사항 ⭐
▾
-- GROUP BY 유의사항 -- SELECT에 집계함수가 아닌 컬럼은 반드시 GROUP BY에 포함! SELECT deptno, ename, COUNT(*) FROM emp GROUP BY deptno; -- 오류! ename 누락 SELECT deptno, COUNT(*) FROM emp GROUP BY deptno; -- OK -- HAVING vs WHERE SELECT deptno, AVG(salary) FROM emp GROUP BY deptno HAVING AVG(salary) > 3000; SELECT deptno, AVG(salary) FROM emp WHERE AVG(salary) > 3000 GROUP BY deptno; -- 오류! -- ROLLUP: 소계+합계 자동 생성 SELECT deptno, job, SUM(salary) FROM emp GROUP BY ROLLUP(deptno, job);
SELECT 절 집계함수 아닌 컬럼 → 반드시 GROUP BY에 포함 / HAVING = 집계 후 조건 / WHERE = 집계 전 조건

CASE 문 & 조건 함수

-- CASE 표현식 (표준SQL) SELECT ename, CASE WHEN salary >= 5000 THEN 'HIGH' WHEN salary >= 3000 THEN 'MID' ELSE 'LOW' END AS grade FROM emp; -- DECODE (Oracle 전용) DECODE(deptno, 10, '영업', 20, '인사', '기타') -- NULL 주의: CASE WHEN salary = NULL → 항상 FALSE! IS NULL 사용 CASE WHEN salary = NULL THEN 'NULL값' -- 절대 실행 안 됨! CASE WHEN salary IS NULL THEN 'NULL값' -- 올바른 표현
CASE WHEN에서 NULL 비교는 반드시 IS NULL 사용 / CASE는 조건을 위에서부터 순서대로 평가하므로 순서가 중요!

JOIN 유형 완전 정리 ⭐

JOIN 종류별 결과 행 수 & 특징 ⭐
▾
JOIN 종류반환 행특징
INNER JOIN ⭐양쪽 모두 일치하는 행만가장 일반적 / JOIN 조건 불일치 시 제외
LEFT OUTER JOIN ⭐왼쪽 테이블 전체 + 오른쪽 일치오른쪽 불일치 시 NULL로 채움
RIGHT OUTER JOIN오른쪽 테이블 전체 + 왼쪽 일치왼쪽 불일치 시 NULL로 채움
FULL OUTER JOIN양쪽 모두 포함불일치 시 양쪽 모두 NULL로 채움
CROSS JOINM × N (카르테시안 곱)JOIN 조건 없음 / 주의해서 사용
SELF JOIN같은 테이블을 두 번별칭(Alias) 필수 / 계층적 데이터
NATURAL JOIN같은 이름·타입 컬럼 자동 JOIN동명 컬럼 자동 매칭 / 컬럼 중복 없음
-- LEFT JOIN 예시: 부서가 없는 직원도 포함 SELECT e.ename, d.dname FROM emp e LEFT JOIN dept d ON e.deptno = d.deptno; -- deptno 없는 직원: dname = NULL로 표시 -- INNER JOIN vs LEFT JOIN 결과 차이 핵심 -- emp: {A:10, B:20, C:30(없음)} / dept: {10:'영업', 20:'인사'} -- INNER: A-영업, B-인사 (C 제외) -- LEFT: A-영업, B-인사, C-NULL (C 포함)
OUTER JOIN의 WHERE 조건 주의사항 ⭐ — 단골 함정!
▾
-- LEFT JOIN 후 WHERE에 오른쪽 컬럼 조건 → INNER JOIN으로 변질! SELECT e.ename, d.dname FROM emp e LEFT JOIN dept d ON e.deptno = d.deptno WHERE d.dname = '영업'; -- NULL인 행(부서 없는 직원) 모두 제외 → INNER JOIN과 동일! -- 올바른 방법: ON 절에 조건 포함 SELECT e.ename, d.dname FROM emp e LEFT JOIN dept d ON e.deptno = d.deptno AND d.dname = '영업'; -- 부서 없는 직원도 포함, d.dname='영업'이 아닌 경우 d컬럼=NULL
OUTER JOIN + WHERE 오른쪽 테이블 조건 = INNER JOIN으로 변질! / 필터 조건은 ON절에 넣거나 OUTER쪽 IS NULL 처리 필요
집합 연산자 — UNION·INTERSECT·EXCEPT ⭐
▾
연산자의미중복 처리유의사항
UNION ⭐합집합중복 제거컬럼 수·데이터 타입 일치 필수
UNION ALL ⭐합집합중복 포함 (빠름)정렬 없음 / 성능 유리
INTERSECT교집합중복 제거양쪽에 모두 있는 행만
EXCEPT (MINUS)차집합중복 제거왼쪽에만 있는 행 / Oracle=MINUS
-- UNION: 중복 제거 (정렬 발생 → 느림) SELECT deptno FROM emp UNION SELECT deptno FROM dept; -- UNION ALL: 중복 포함 (정렬 없음 → 빠름) SELECT deptno FROM emp UNION ALL SELECT deptno FROM dept; -- ORDER BY는 마지막 SELECT에만 1번! SELECT a FROM t1 UNION SELECT a FROM t2 ORDER BY a;
UNION은 중복 제거로 느림 / 중복 유지·성능 필요 시 UNION ALL / ORDER BY는 전체 결과에 1번만 마지막에 사용
INNER JOIN=교집합행 / LEFT JOIN=왼쪽전체+오른쪽NULL / LEFT JOIN+WHERE우측컬럼=INNER JOIN변질 / UNION=중복제거 / UNION ALL=중복포함

서브쿼리 (Subquery) 핵심 ⭐

서브쿼리 위치별 분류 & 유의사항
▾
위치명칭반환예시
SELECT절스칼라 서브쿼리단일 값 (1행 1열)SELECT (SELECT MAX(salary) FROM emp)
FROM절인라인 뷰가상 테이블FROM (SELECT * FROM emp WHERE...) e
WHERE절중첩 서브쿼리단일값/다중행WHERE salary = (SELECT MAX(salary)...)
-- 단일행 서브쿼리: =, >, <, >=, <= 사용 SELECT ename FROM emp WHERE salary = (SELECT MAX(salary) FROM emp); -- 다중행 서브쿼리: IN, ANY, ALL, EXISTS 사용 SELECT ename FROM emp WHERE deptno IN (SELECT deptno FROM dept WHERE loc = 'SEOUL'); -- ANY vs ALL WHERE salary > ANY (1000, 2000, 3000) -- salary > 1000 (최솟값 초과) WHERE salary > ALL (1000, 2000, 3000) -- salary > 3000 (최댓값 초과)
단일행 서브쿼리에 = 사용 시 2행 이상 반환되면 오류! / 다중행에는 IN·ANY·ALL·EXISTS 사용
EXISTS vs IN 비교 ⭐
▾
-- IN: 서브쿼리 결과를 목록으로 비교 SELECT * FROM emp WHERE deptno IN (SELECT deptno FROM dept); -- 서브쿼리 전체 실행 후 IN 목록과 비교 -- NULL이 포함된 IN 목록: WHERE col IN (1, NULL) → col=NULL 절대 매칭 안 됨! -- EXISTS: 존재 여부만 확인 (TRUE/FALSE) SELECT * FROM emp e WHERE EXISTS (SELECT 1 FROM dept d WHERE d.deptno = e.deptno); -- 한 행이라도 찾으면 즉시 TRUE 반환 → 성능 유리 (대용량) -- NULL에 안전 (EXISTS는 존재 여부만, 값 비교 없음)
NOT IN + NULL 주의: NOT IN (1, NULL) → 항상 FALSE! 공집합 반환 / 서브쿼리에 NULL 가능성 있으면 NOT EXISTS 사용 권장
윈도우 함수 (Window Function / 분석 함수) ⭐
▾
-- 윈도우 함수 기본 구문 함수명() OVER ([PARTITION BY 그룹기준] [ORDER BY 정렬기준] [ROWS/RANGE 프레임]) -- 순위 함수 RANK() -- 1,2,2,4 (동순위 다음 건너뜀) DENSE_RANK() -- 1,2,2,3 (동순위 다음 안 건너뜀) ROW_NUMBER() -- 1,2,3,4 (항상 유일한 순번) NTILE(4) -- N개 구간으로 분할 -- 집계 윈도우 함수 SUM(salary) OVER(PARTITION BY deptno ORDER BY empno) -- 누적합 -- LAG·LEAD: 이전/다음 행 참조 LAG(salary, 1, 0) OVER(ORDER BY empno) -- 이전 행 salary LEAD(salary, 1, 0) OVER(ORDER BY empno) -- 다음 행 salary
RANK()는 동순위 다음 번호 건너뜀(1,2,2,4) / DENSE_RANK()는 연속(1,2,2,3) / ROW_NUMBER()는 항상 고유 / WHERE에서 윈도우함수 사용 불가 → 인라인뷰로 감싸야
RANK=1,2,2,4 / DENSE_RANK=1,2,2,3 / ROW_NUMBER=1,2,3,4 / LAG=이전행·LEAD=다음행

트랜잭션 (TCL) & 격리 수준 ⭐

COMMIT·ROLLBACK·SAVEPOINT ⭐
▾
-- 트랜잭션 예시 BEGIN TRANSACTION; -- 트랜잭션 시작 (일부 DB) UPDATE emp SET salary = salary * 1.1; SAVEPOINT sp1; -- 중간 저장점 DELETE FROM emp WHERE salary < 1000; ROLLBACK TO SAVEPOINT sp1; -- sp1 시점으로 되돌림 (UPDATE는 유지) COMMIT; -- 확정 저장
명령동작유의사항
COMMIT변경사항 영구 저장COMMIT 후 ROLLBACK 불가
ROLLBACK변경사항 취소·되돌림마지막 COMMIT 시점까지 취소
SAVEPOINT트랜잭션 중간 저장점ROLLBACK TO 세이브포인트명

▾

  • Dirty Read: 커밋 안 된 데이터 읽기 — 롤백 시 잘못된 데이터
  • Non-Repeatable Read: 같은 쿼리를 두 번 실행했을 때 결과 다름 (중간에 다른 트랜잭션이 UPDATE·DELETE)
  • Phantom Read: 같은 범위 쿼리 두 번 실행 시 행 수 다름 (중간에 INSERT)
격리 수준 높을수록 이상 현상 감소하지만 동시성 낮아짐(성능 저하) / 기본값: Oracle=READ COMMITTED, MySQL InnoDB=REPEATABLE READ
인덱스 (Index) 핵심 유의사항 ⭐
▾
  • 인덱스가 사용되지 않는 경우 (함정 포인트!):
WHERE salary * 12 > 60000; -- 컬럼에 함수/연산 적용 → 인덱스 미사용! WHERE UPPER(ename) = 'SCOTT'; -- 컬럼에 함수 적용 → 미사용! WHERE salary != 3000; -- 부정 연산자(!= <>) → 전체 스캔 WHERE salary IS NULL; -- IS NULL → 인덱스 미사용 (일반 B-Tree) WHERE name LIKE '%SCOTT%'; -- 앞에 % → 인덱스 미사용! WHERE salary > 60000 / 12; -- 상수 연산으로 변경 → 인덱스 사용! WHERE name LIKE 'SCOTT%'; -- 앞에 % 없으면 인덱스 사용!
인덱스 종류특징
클러스터드 인덱스데이터 물리 순서 정렬 / 테이블당 1개 / PK 자동 생성
논클러스터드 인덱스별도 인덱스 페이지 / 테이블당 여러 개 가능
복합 인덱스선행 컬럼 조건 없으면 인덱스 미사용 (최 좌측 접두사 규칙)
컬럼에 함수·연산 적용, LIKE '%앞', !=, IS NULL → 인덱스 무효화! / 복합인덱스는 선두컬럼 조건 없으면 미사용
DCL — GRANT·REVOKE & 뷰(VIEW)
▾
-- GRANT: 권한 부여 GRANT SELECT, INSERT ON emp TO user1; GRANT SELECT ON emp TO user1 WITH GRANT OPTION; -- 재부여 권한 포함 -- REVOKE: 권한 회수 REVOKE SELECT ON emp FROM user1; -- WITH GRANT OPTION으로 부여받은 권한도 연쇄 회수! -- VIEW: 가상 테이블 CREATE VIEW emp_view AS SELECT empno, ename, deptno FROM emp WHERE deptno = 10; -- VIEW의 DML 제한 -- 아래 경우 뷰를 통한 UPDATE/INSERT 불가: -- 집계함수 포함 / GROUP BY / DISTINCT / ROWNUM -- 여러 테이블 JOIN / 서브쿼리 포함
WITH GRANT OPTION 부여 후 REVOKE 시 연쇄 권한 회수 / 집계함수·GROUP BY·DISTINCT 포함 뷰는 DML 불가
READ COMMITTED=Dirty Read방지 / REPEATABLE READ=NonRepeatableRead방지 / SERIALIZABLE=Phantom방지 / 컬럼함수→인덱스무효
