PET (Privacy Enhancing Technologies) 정의

PET(개인정보 보호 강화 기술)란 개인정보를 수집·활용하면서도 개인을 식별할 수 없도록 하거나, 개인정보를 노출하지 않고 연산·분석을 가능하게 하는 기술 체계

항목내용
목적개인정보의 데이터 활용성(Utility)과 프라이버시 보호를 동시에 달성
배경빅데이터·AI 시대 데이터 활용 수요 ↑, 개인정보보호법·GDPR 등 규제 강화
핵심 딜레마데이터 활용성 ↑ ↔ 프라이버시 보호 ↑ — 두 목표가 서로 상충
관련 법령개인정보보호법(§28조의2~§28조의7), 데이터 3법, GDPR(Article 25 Privacy by Design)
PET = 데이터 활용성 ↑과 프라이버시 보호 ↑를 동시 달성하는 기술 체계

프라이버시 by Design (PbD) 7원칙 — Ann Cavoukian

원칙내용
① 사전 예방사전에 프라이버시 위험을 예방 (사후 치료 아님)
② 기본값으로 프라이버시별도 조치 없어도 자동으로 프라이버시 보호 (Privacy as Default)
③ 설계에 내재화시스템 설계 단계부터 프라이버시 내장 (Add-on 아님)
④ 전합 기능프라이버시 vs 보안 이분법 거부 — 둘 다 달성 (Positive-Sum)
⑤ 종단간 보안데이터 수명 전 주기에 걸쳐 보안 유지
⑥ 가시성·투명성공개·투명하게 운영 (Openness)
⑦ 사용자 중심사용자 프라이버시 이익 최우선 (User Centricity)
PbD 핵심 = 기본값 프라이버시 + 설계 내재화 + Positive-Sum (보안과 프라이버시 동시 달성)
⭐

가명처리 vs 익명처리 — 개인정보보호법 핵심 구분

개인정보보호법 제28조의2~7 (2020 데이터 3법 개정): 가명정보는 통계작성·과학적 연구·공익적 기록보존 목적으로 정보주체 동의 없이 활용 가능
구분가명처리 (Pseudonymization)익명처리 (Anonymization)
정의추가 정보 없이는 특정 개인을 알아볼 수 없도록 처리 (복원 가능)시간·비용·기술을 합리적으로 고려할 때 복원이 불가능한 처리
복원 가능성가능 (키 테이블 등 추가 정보 있으면)불가능 (실질적으로)
법적 지위개인정보 (규제 적용, 특례 있음)개인정보 아님 (규제 미적용)
활용동의 없이 과학연구·통계·공익 목적 활용 가능제한 없이 자유롭게 활용 가능
기술 예시암호화, 토큰화, 일관된 대체값k-익명성, 차분프라이버시, 집계
추가 정보 관리분리 보관 필수 (결합 방지)별도 관리 불필요
GDPR 관련Article 4(5) — 여전히 개인정보GDPR 적용 제외
⭐가명처리=복원가능·개인정보·특례활용 / 익명처리=복원불가·개인정보아님·자유활용

비식별화 기법 (개인정보보호위원회 가이드)

데이터 마스킹 (Data Masking)
개인 식별 값을 숨김 문자로 대체
직관적
▾
  • 부분 마스킹: 홍*동, 010-****-1234 — 일부만 숨김
  • 전체 마스킹: *****, XXXXXXXX — 전체 숨김
  • 랜덤 노이즈: 숫자에 랜덤 값 추가 (나이 27 → 24~30 범위로 교란)
  • 데이터 스크램블링: 개인 속성값을 레코드 간 섞음
일반화 (Generalization)
값을 더 넓은 범주로 대체
범주화
▾
  • 수치 범주화: 나이 27세 → 20대, 소득 350만원 → 300~400만원
  • 지역 일반화: 서울시 강남구 역삼동 → 서울시 강남구
  • 날짜 일반화: 1995.03.15 → 1995년 3월 (또는 1990년대)
  • Top/Bottom Coding: 극단값을 최대·최솟값 이상/이하로 처리
데이터 삭제·집계
식별자 제거 및 통계 집계
삭제
▾
  • 식별자 삭제: 이름·주민번호·연락처 등 직접 식별자 제거
  • 레코드 삭제: 소수의 특이한 속성 조합의 레코드 전체 삭제
  • 집계(Aggregation): 개별 값 대신 합계·평균·빈도 제공
  • 마이크로집계: 유사한 레코드를 그룹화 후 그룹 평균으로 대체
비식별화 기법: 마스킹(숨김)·일반화(범주화)·삭제·집계·가명화(대체값) 5가지
📐

k-익명성 · l-다양성 · t-근접성

모델정의보호 대상한계
k-익명성
(k-Anonymity)		준식별자 속성의 동일한 조합이 최소 k개 이상의 레코드에 존재개인 재식별 방지민감 속성의 동질성 공격에 취약 (k명 모두 같은 질병)
l-다양성
(l-Diversity)		동일 준식별자 그룹 내 민감 속성이 최소 l가지 이상 존재동질성·배경지식 공격민감 속성 분포가 실제와 크게 다를 수 있음 (skewness)
t-근접성
(t-Closeness)		그룹 내 민감 속성 분포가 전체 데이터 분포와의 차이가 t 이하속성 추론 공격데이터 유용성(Utility) 감소
// k=3 익명성 예시 (나이·지역이 동일한 레코드가 3개 이상) 나이 지역 질병 20~29 서울 고혈압 ← 3개 이상의 레코드가 동일한 20~29 서울 당뇨 ← 준식별자(나이+지역) 조합을 가져야 20~29 서울 암 ← k-익명성 만족
k-익명성(k개 동일 준식별자) → l-다양성(민감속성 l종) → t-근접성(분포 차이 t이하) 순으로 강화

암호 기반 PET 핵심 3종

동형암호 (Homomorphic Encryption, HE)
암호화된 상태에서 연산 가능 — 복호화 없이 분석
⭐핵심
▾
  • 정의: 암호문(Ciphertext)에 대해 연산을 수행해도, 복호화한 결과가 원문에 같은 연산을 수행한 결과와 동일
  • 핵심: 데이터를 복호화하지 않고 클라우드에서 연산 가능 → 클라우드 서버가 데이터 내용을 모름
  • 부분 동형(PHE): 덧셈 또는 곱셈 중 하나만 지원 (RSA: 곱셈, Paillier: 덧셈)
  • 완전 동형(FHE): 덧셈·곱셈 모두 지원 — 임의의 함수 계산 가능 (속도 느림)
  • 실제 활용: 클라우드 암호화 데이터 분석, 의료 데이터 AI 분석, 금융 데이터 공동 분석
// 동형암호 특성 E(a) + E(b) = E(a+b) // 암호문 덧셈 = 원문 합계의 암호문 E(a) × E(b) = E(a×b) // 암호문 곱셈 = 원문 곱의 암호문 // → 서버는 암호문만 보고 연산하지만 결과는 정확

주요 FHE 스킴: CKKS(실수), BFV/BGV(정수) / 라이브러리: HElib, Microsoft SEAL, OpenFHE

영지식 증명 (Zero-Knowledge Proof, ZKP)
정보를 공개하지 않고 사실 증명 — 알리지 않고 증명
⭐핵심
▾
  • 정의: 증명자(Prover)가 검증자(Verifier)에게 특정 사실이 참임을 해당 정보 자체를 공개하지 않고 증명
  • 3대 요건: ①완전성(Completeness) ②건전성(Soundness) ③영지식성(Zero-Knowledge)
  • 활용 예시: "나이가 19세 이상임"을 생년월일 공개 없이 증명 / 소득 조건 충족을 실제 소득 미공개로 증명
  • 종류: zk-SNARKs(짧은 증명·비인터랙티브) / zk-STARKs(양자 내성·투명 설정) / Groth16
  • 블록체인 활용: Zcash(개인정보 거래), zkEVM(이더리움 확장성), Polygon zkEVM
// ZKP 활용 예시 나이 증명: "저는 19세 이상입니다" ← 생년월일 공개 없이 자산 증명: "저는 100만원 이상 보유" ← 정확한 금액 미공개 비밀번호: "맞는 비밀번호 알고 있음" ← 비밀번호 자체 미전송
안전한 다자간 계산 (Secure Multi-Party Computation, MPC)
여러 참여자가 각자 데이터 비공개로 공동 연산
⭐핵심
▾
  • 정의: 여러 참여자가 각자의 데이터를 공유하지 않고 공동으로 함수를 계산하고 결과만 얻음
  • 예시: A·B·C 세 회사가 각자 급여 데이터를 공개하지 않고 업계 평균 급여를 계산
  • 활용: 금융 데이터 공동 분석, 의료 데이터 연합 연구, 광고 효과 측정
  • 주요 프로토콜: 비밀 분산(Secret Sharing), 조작 방지 회로(GMW), OT(Oblivious Transfer)
  • MPC vs 동형암호: MPC = 여러 참여자 간 분산 계산 / HE = 단일 클라우드에서 암호문 연산
동형암호=암호화 상태에서 연산 / ZKP=공개없이 증명 / MPC=여러 참여자 데이터 비공개 공동연산
📊

차분 프라이버시 (Differential Privacy, DP)

차분 프라이버시: 쿼리 결과에 랜덤 노이즈(Random Noise)를 추가하여 특정 개인의 데이터가 포함되었는지 추론 불가능하게 하는 기법
항목내용
ε (엡실론)프라이버시 예산(Privacy Budget) — 값이 작을수록 프라이버시 강하지만 데이터 정확도 낮음
노이즈 메커니즘라플라스 메커니즘(연속값), 가우시안 메커니즘, 지수 메커니즘(범주값)
글로벌 DP신뢰할 수 있는 중앙 서버가 집계 후 노이즈 추가 (Apple 통계 수집)
로컬 DP각 사용자가 디바이스에서 직접 노이즈 추가 후 전송 (서버 불신 환경)
활용 사례Apple(사용자 통계), Google RAPPOR, 미국 인구조사, 연합학습 결합
합성 민감도한 레코드가 빠졌을 때 쿼리 결과가 최대로 달라지는 값 (노이즈 크기 결정)
// 차분 프라이버시 예시 (평균 나이 계산) 실제 평균 나이: 35.2세 노이즈 추가 후: 35.2 + Laplace(0, 1/ε) = 34.7세 // 개인 특정 불가하지만 통계적 유용성 유지 // ε 작을수록 노이즈 크고 프라이버시 보호 강함
차분프라이버시 = 랜덤 노이즈 추가 / ε(엡실론) 작을수록 보호 강함 / 글로벌DP vs 로컬DP
🤝

연합학습 (Federated Learning, FL)

연합학습: 원시 데이터를 중앙 서버로 전송하지 않고, 각 기기에서 로컬 학습 후 모델 파라미터(기울기)만 공유하여 글로벌 AI 모델을 학습하는 방식
항목내용
Google 제안2017년 Google이 스마트폰 키보드(Gboard) 개선에 최초 적용
동작 흐름글로벌 모델 배포 → 로컬 데이터로 학습 → 파라미터(gradient) 업로드 → 중앙 서버 집계 → 모델 갱신
장점원시 개인정보 중앙 집중 불필요 / 네트워크 부하 감소 / 다양한 데이터로 학습
한계모델 파라미터에서 개인정보 역추론 가능 (Gradient Inversion) / 비잔틴 공격(악의적 참여자)
결합 기술차분 프라이버시(DP) + 연합학습 / MPC + 연합학습 / SecAgg(보안 집계)
수직/수평 FL수평 FL: 동일 특성, 다른 사용자 / 수직 FL: 동일 사용자, 다른 특성 (기관 간 협력)
활용의료 AI(병원 간 협력), 금융 사기 탐지, 스마트폰 AI 학습
연합학습 = 원시 데이터 미전송 + 파라미터만 공유 / DP+FL 결합으로 기울기 역추론 방어
🔐

합성 데이터 (Synthetic Data) · 토큰화

합성 데이터

  • 정의: 실제 데이터의 통계적 특성을 유지하면서 인공적으로 생성한 가상 데이터
  • 생성 방법: GAN(적대적 생성 네트워크), VAE, 통계 모델 기반
  • 장점: 민감 정보 없음 / 대용량 생성 가능 / AI 학습 데이터 증강
  • 한계: 실제 데이터와 완전 동일하지 않음 / 멤버십 추론 공격 가능

토큰화 (Tokenization)

  • 정의: 민감 데이터를 의미 없는 토큰(Token)으로 대체 — 원본과 토큰 매핑 테이블은 별도 보관
  • 예시: 신용카드 번호 → 4532-xxxx-xxxx-7891 토큰
  • vs 암호화: 토큰은 수학적 관계 없음 / 암호화는 수학적으로 변환
  • 활용: PCI DSS 결제 데이터 보호, 의료 ID 관리
합성데이터=GAN으로 통계적 특성 유지 가상 생성 / 토큰화=의미없는 토큰 대체+매핑테이블 분리

DPIA (개인정보 영향평가) · 개인정보 보호법 주요 조항

DPIA (Data Protection Impact Assessment)
개인정보 영향평가 / 개인정보보호법 §33
⭐핵심
▾
  • 정의: 개인정보 처리 시스템 구축·변경 시 프라이버시 위험을 사전에 식별·평가·완화하는 절차
  • 국내 의무 대상: 5만 명 이상 정보주체 민감정보·고유식별정보 처리 / 공공기관 시스템 구축·변경 시
  • GDPR Article 35: 고위험 처리 시 DPIA 의무 — 자동화 의사결정, 대규모 민감정보, 대규모 공개 장소 모니터링
  • 단계: 위험 식별 → 위험 평가 → 완화 방안 수립 → DPA 사전 협의 (필요시)

§28조의3 (가명정보 결합): 결합전문기관을 통해서만 가명정보 결합 허용

§39조의3 (개인정보 이동권): 금융·의료 등 이동권 행사 가능
