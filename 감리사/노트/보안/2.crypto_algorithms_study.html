Feistel 구조 vs SPN 구조

Feistel 구조
블록을 L·R 절반으로 나눠 라운드 반복
복호화 = 암호화 역순 (역함수 불필요)
예: DES, 3DES, SEED, Blowfish
SPN 구조 (대입-치환)
전체 블록에 S-box(대입)·P-box(치환) 동시 적용
병렬 처리 가능 → 빠름
예: AES, ARIA

주요 비대칭키 알고리듬 (클릭해서 상세 보기)

비대칭키 비교표

알고리듬수학적 기반키 길이(보안)용도특징
RSA소인수 분해 어려움2048비트 이상암호화·서명·키 교환가장 널리 사용, 키 길어짐
ECC타원곡선 이산대수256비트 (RSA 3072 수준)암호화·서명·키 교환짧은 키로 동등 보안, 모바일 최적
DH이산대수 어려움2048비트 이상키 교환 전용암호화·서명 불가, 키 협의만
ElGamal이산대수 어려움2048비트 이상암호화·서명암호문이 평문의 2배 크기
DSA이산대수 어려움1024~3072비트전자서명 전용암호화 불가, 서명만 가능
ECDSA타원곡선 이산대수256비트전자서명 전용DSA+ECC, 비트코인·TLS 사용
DH: 키 교환만 / DSA: 서명만 / RSA: 암호화+서명+키교환 모두 가능
ECC = 짧은 키로 높은 보안 → 모바일·IoT·스마트카드에 최적

DH 키 교환 원리 (중간자 공격 취약)

Alice
a 생성
→ A=g^a mod p →
공개
채널
← B=g^b mod p ←
Bob
b 생성
Alice 계산: K = B^a mod p = g^(ab) mod p
Bob 계산: K = A^b mod p = g^(ab) mod p
→ 같은 K(세션 키)를 키 없이 공유 성공
DH 취약점: 중간자 공격(MITM) — 인증 기능 없음 → PKI·인증서로 보완 필요
ECDH: DH + 타원곡선 → 성능 개선 버전 (TLS 1.3 기본)

해시 함수의 3대 보안 성질

일방향성
(One-way)
해시값 → 원문 역산 불가 (역함수 없음)
충돌 저항성
(Collision Resistance)
H(x) = H(y)인 x≠y를 찾기 어려움 — MD5·SHA-1은 충돌 발견됨
역상 저항성
(Preimage Resistance)
주어진 해시값 h에 대해 H(x)=h인 x 찾기 어려움

HMAC (Hash-based MAC)

정의
해시 함수 + 비밀 키 → 메시지 인증 코드 생성
공식
HMAC(K, m) = H((K⊕opad) ∥ H((K⊕ipad) ∥ m))
용도
무결성 + 인증 동시 제공 (전자서명 아님 — 부인방지 없음)
해시 vs HMAC
해시: 키 없음, 무결성만 / HMAC: 키 있음, 무결성+인증

패딩 (Padding) 방식

PKCS#7
부족한 바이트 수만큼 그 값으로 채움 (예: 3바이트 부족 → 03 03 03)
OAEP
RSA 암호화에서 사용하는 랜덤 패딩 — RSA-PKCS#1 v1.5 대체
Zero Padding
0으로 채움 — 원문 길이 복원이 어려워 권장 안 함
패딩 오라클 공격: CBC 모드 + 패딩 검증 오류 메시지 → 평문 복호화 가능 → TLS POODLE 취약점 원인
