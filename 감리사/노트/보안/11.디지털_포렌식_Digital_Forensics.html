디지털 포렌식 (Digital Forensics) 정의

디지털 포렌식이란 법적 절차에 따라 디지털 장치·시스템에서 전자적 증거(Digital Evidence)를 수집·보전·분석·제출하는 과학적·체계적 과정

항목내용
목적사이버 범죄·침해 사고·내부 비위 조사 시 법적으로 유효한 디지털 증거 확보
대상PC·서버·모바일·IoT·클라우드·네트워크 장비·저장매체 등 모든 디지털 기기
핵심 가치증거의 법적 증거 능력(Admissibility) 확보 — 수집부터 제출까지 무결성 유지
관련 법형사소송법(전자증거 규정), 개인정보보호법, 전자금융거래법 등
디지털 포렌식 = 법적 절차에 따른 전자 증거 수집·보전·분석·제출 / 법적 증거 능력이 핵심
⚖️

디지털 포렌식 5대 원칙 (ACPO 기반)

① 정당성 원칙
증거 수집은 적법한 절차와 권한에 따라 수행
불법 수집 증거 → 법원에서 증거 능력 배제
② 무결성 원칙
수집된 증거는 원본과 동일하게 보전 — 어떤 변경도 없어야 함
해시값(MD5·SHA-256) 비교로 무결성 검증
③ 재현성 원칙
동일한 조건에서 분석을 반복하면 동일한 결과가 나와야 함
분석 절차·도구·환경의 문서화 필수
④ 신속성 원칙
휘발성 증거(RAM·프로세스 등)는 최대한 신속하게 수집
휘발성 순서에 따른 우선 수집 (Order of Volatility)
⑤ 연계 보관성 원칙 (Chain of Custody)
증거의 수집부터 법정 제출까지 모든 취급 기록 유지
누가·언제·어디서·무엇을 했는지 완전한 이력 문서화
⑥ 객관성 원칙
분석자의 편견 없이 객관적 사실만 기반으로 결론 도출
특정 결론을 유도하는 분석 금지
6대 원칙: ①정당성 ②무결성 ③재현성 ④신속성 ⑤연계보관성(CoC) ⑥객관성
📊

휘발성 데이터 수집 우선순위 (Order of Volatility)

순위데이터 유형특징
1순위 (최고 휘발)CPU 레지스터·캐시전원 차단 즉시 소멸
2순위RAM (메인 메모리)실행 중 프로세스·암호키·네트워크 연결 포함
3순위네트워크 상태·ARP 캐시현재 연결·세션 정보
4순위실행 중 프로세스·열린 파일ps, netstat 결과
5순위임시 파일·페이지 파일(Swap)시스템 재시작 시 소멸 가능
6순위디스크(HDD·SSD)비휘발성 — 전원 꺼도 유지
7순위 (최저 휘발)광학 매체·원격 로그가장 안정적 보존
휘발성 우선 수집: CPU레지스터→RAM→네트워크→프로세스→스왑→디스크→광학매체
📋

디지털 포렌식 수행 절차 (6단계)

1
사전 준비 (Preparation)
포렌식 수행 권한 확보, 조사 목적·범위 확정, 도구·장비 준비
영장·동의서 확보 / 포렌식 도구 검증 / 조사팀 구성
2
현장 보존 (Identification & Preservation)
현장 접근 통제, 증거 후보 식별, 현장 상태 그대로 보존
현장 사진 촬영 / 증거 태깅 / 작동 중인 시스템 상태 유지 판단
3
증거 수집 (Collection)
휘발성 순서에 따라 증거 수집 / 디스크 이미징(Bit-by-Bit 복사)
쓰기 방지 장치(Write Blocker) 사용 / 해시값 생성(MD5·SHA-256) / 원본 봉인
4
증거 분석 (Examination & Analysis)
복사본(이미지)으로 분석 수행 — 원본 절대 사용 금지
파일 복구·타임라인 분석·키워드 검색·아티팩트 추출 / 로그 분석
5
보고서 작성 (Reporting)
분석 결과를 객관적·명확하게 문서화 / 기술·법적 보고서 작성
발견된 증거·분석 과정·결론 기술 / 법정 증언 대비 명확한 기술
6
증거 제출 (Presentation)
법원·수사기관에 Chain of Custody 문서와 함께 증거 제출
보고서·증거 원본·복사본·해시값 전달 / 법정 증언
포렌식 6단계: 준비→현장보존→증거수집→분석→보고서→제출 / 복사본으로만 분석!
💡

디스크 이미징 (Disk Imaging)

항목내용
정의원본 저장매체를 비트 단위(Bit-by-Bit)로 완전히 복사한 이미지 파일 생성
목적원본 보존 + 복사본으로 반복 분석 가능 + 무결성 증명
쓰기 방지 장치Write Blocker(쓰기 차단기) — 이미징 시 원본 매체에 쓰기 작업이 발생하지 않도록 차단
이미지 형식RAW/DD, E01(EnCase), AFF, EWF 등
해시 검증이미징 전·후 MD5·SHA-256 해시 계산 → 일치하면 무결성 입증
슬랙 공간비트 단위 복사이므로 삭제 파일·슬랙 공간까지 포함
이미징 = 비트단위 완전 복사 / Write Blocker 필수 / MD5·SHA-256 해시로 무결성 증명

파일시스템 포렌식 주요 개념

개념설명
삭제 파일 복구파일 삭제 시 실제 데이터는 남고 디렉터리 엔트리·FAT만 변경 → 복구 가능
SSD는 TRIM 명령으로 데이터 즉시 삭제 → 복구 어려움
슬랙 공간
(Slack Space)		파일이 클러스터를 완전히 채우지 못해 생기는 빈 공간 — 이전 파일 데이터 잔존 가능
파일 슬랙 : 마지막 섹터의 빈 공간 / 볼륨 슬랙 : 파티션 끝의 미사용 영역
MFT
(Master File Table)		NTFS 파일시스템의 핵심 — 모든 파일·폴더 메타데이터(생성·접근·수정 시간, 크기, 위치) 저장
타임스탬프
(Timestamp)		MACB — Modified(수정) / Accessed(접근) / Changed(변경=메타데이터) / Born(생성)
타임라인 분석의 핵심 / 안티포렌식으로 조작 가능
카빙 (Carving)파일 시그니처(헤더·푸터)를 기반으로 파일시스템과 무관하게 데이터 복구
포맷된 디스크에서도 파일 복구 가능
파일 시그니처파일 형식별 고유 매직 넘버(Magic Number) — JPG: FF D8 FF / PDF: %PDF / ZIP: 50 4B 03 04
MACB 타임스탬프 / 슬랙공간 = 이전 데이터 잔존 / 카빙 = 시그니처 기반 복구 / MFT = NTFS 핵심

Chain of Custody (연계 보관성) — 핵심 법적 요건

Chain of Custody(연계 보관성): 증거가 수집된 순간부터 법정 제출까지 누가·언제·어디서·어떻게 취급했는지를 빠짐없이 기록·유지하는 것

기록 항목내용
수집 정보수집 일시, 수집자 이름·직위, 수집 장소, 수집 방법
증거 식별증거 번호, 설명(장치명·모델·SN), 해시값(MD5·SHA-256)
이전 기록증거 이동·접근 시마다 서명·날짜·사유 기록
보관 환경봉인 상태, 보관 장소, 온도·습도 조건
법적 효과Chain of Custody 누락·단절 → 증거 능력 상실 위험
CoC = 수집~제출까지 완전한 취급 이력 / 단절 시 법적 증거 능력 상실 위험
🔐

해시 무결성 검증

해시 알고리즘특징포렌식 활용
MD5128비트 / 빠름 / 충돌 취약점 있음속도가 중요한 초기 검증 (단독 사용 지양)
SHA-1160비트 / MD5보다 안전 / 충돌 가능 확인레거시 환경 (단독 사용 지양)
SHA-256256비트 / 현재 표준 / 충돌 저항성 높음현행 포렌식 표준 — 단독 또는 MD5와 병행
  • 검증 방법: 이미징 전 원본 해시 계산 → 이미징 후 복사본 해시 계산 → 일치 시 무결성 입증
  • 이중 해시: MD5 + SHA-256 병행 사용으로 신뢰성 향상
  • 해시 불일치: 데이터 변조 또는 전송 오류 → 해당 증거 사용 불가
MD5(128비트·빠름) / SHA-256(256비트·현행 표준) / 이중 해시 병행 권장
⚖️

법적 증거 능력 요건 (Admissibility)

법적 증거 요건 4가지

  • 진정성(Authenticity): 증거가 주장하는 것과 실제로 동일 — 해시 검증
  • 무결성(Integrity): 수집 이후 변경 없음 — Chain of Custody
  • 완전성(Completeness): 사건과 관련된 모든 증거 수집
  • 신뢰성(Reliability): 검증된 도구·방법론 사용

증거 배제 사유

  • 불법 수집 (영장 없는 압수)
  • Chain of Custody 단절
  • 해시 불일치 (무결성 훼손)
  • 비검증 도구·방법 사용
  • 원본이 아닌 복사본 제출 (적절한 절차 없이)
법적 증거 요건: 진정성·무결성·완전성·신뢰성 / 불법수집·CoC 단절 → 증거 배제

안티포렌식 (Anti-Forensics)

안티포렌식: 디지털 증거의 수집·분석·복구를 방해하거나 증거를 위·변조·은닉·삭제하려는 행위
기법내용대응 방법
데이터 삭제·덮어쓰기파일 완전 삭제 도구(DoD 7회 덮어쓰기), SSD TRIM이미징 후 카빙 시도 / 슬랙 공간 분석
스테가노그래피이미지·오디오·파일 내부에 데이터 은닉스테가노그래피 탐지 도구(StegDetect)
암호화BitLocker, VeraCrypt 등으로 데이터 암호화메모리 포렌식으로 키 추출 / 비밀번호 크래킹
타임스탬프 변조MACB 시간 정보 수정(Timestomping)MFT·레지스트리·로그와 교차 검증
메타데이터 제거파일에서 작성자·위치 등 메타데이터 삭제원본 파일 분석 / 복구 도구 사용
VPN·Tor 사용IP 추적 방해엔드포인트 포렌식 / ISP 협조 요청
루트킷운영체제 수준에서 프로세스·파일 은폐메모리 포렌식 / 오프라인 분석
안티포렌식: 삭제·스테가노그래피·암호화·타임스탬프 변조·루트킷 / 메모리포렌식으로 대응

구분스테가노그래피암호화
