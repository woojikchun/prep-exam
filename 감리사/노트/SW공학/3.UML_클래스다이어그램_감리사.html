클래스 다이어그램 정의 & 목적 ⭐

클래스 다이어그램: 시스템의 정적(Static) 구조를 표현하는 UML 구조 다이어그램 — 클래스의 속성·메서드·클래스 간 관계를 시각화하여 객체지향 설계의 청사진을 제공

목적 & 활용

  • OOP 설계 표현: 클래스·인터페이스·추상 클래스 구조
  • 데이터 모델 매핑: ER 다이어그램과 대응 가능
  • 코드 생성 기반: 다이어그램 → 코드 자동 생성(Forward Engineering)
  • 역공학(Reverse Engineering): 기존 코드 → 다이어그램
  • 팀 소통: 설계 의도를 시각적으로 공유

클래스 관계 I — 연관·의존·상속 ⭐

① 연관 (Association) ⭐ — 가장 기본 관계
▾
  • 정의: 두 클래스가 구조적으로 연결된 관계 — 한 클래스가 다른 클래스를 참조
  • 표기: 실선 (방향성 없는) / 방향 연관 = 화살표 실선
  • 다중성(Multiplicity) ⭐: 관계에 참여하는 객체 수 표시
  • 역할명(Role Name): 연관 선 끝에 역할 이름 표기 가능
  • 예시: Customer가 여러 Order를 가짐

1:0..* — 고객 1명당 0개 이상의 주문

다중성(Multiplicity) 표기 ⭐

표기의미예시
1정확히 1개학생은 반드시 1개 학번
0..10개 또는 1개 (선택적)고객은 프로필 0~1개
0..* 또는 *0개 이상 (제한 없음)고객은 주문 0건 이상
1..*1개 이상주문은 상품 1개 이상 포함
n정확히 n개주사위는 면 6개
m..nm개 이상 n개 이하팀원은 3..10명
② 의존 (Dependency) ⭐
▾
  • 정의: 한 클래스가 다른 클래스를 일시적으로 사용하는 약한 관계 — 메서드 매개변수·지역 변수·반환 타입으로만 사용
  • 표기: 점선 화살표 (- - -→) — "사용하는 쪽"에서 "사용되는 쪽"으로
  • 관계 강도: 가장 약한 관계 (연관보다 약함)
  • 예시: PaymentService가 결제 시 CreditCard를 매개변수로 받아 처리

점선 화살표 — 일시적 사용

의존=점선화살표·일시적사용 / 연관=실선·구조적참조(속성으로 보유) — 구분이 핵심!
③ 상속/일반화 (Inheritance/Generalization) ⭐
▾
  • 정의: 자식 클래스가 부모 클래스의 속성·메서드를 상속(is-a 관계)
  • 표기: 속이 빈 삼각형 + 실선 — 자식→부모 방향
  • 의미: "A는 B이다(is-a)" — 학생은 사람이다
  • 추상 클래스 상속: 추상 메서드를 자식에서 반드시 구현
  • 특징: 가장 강한 정적 결합 / 재사용·다형성의 기반

속이 빈 삼각형 실선 — 자식→부모

연관=실선·구조적참조·다중성표기 / 의존=점선화살표·일시적사용 / 상속=속이빈삼각형+실선·is-a관계·자식→부모

클래스 관계 II — 집합·합성·실현 ⭐

④ 집합 (Aggregation) ⭐ — "has-a" 약한 포함
▾
  • 정의: 전체-부분 관계이지만 부분이 전체에 독립적으로 존재 가능
  • 표기: 속이 빈 다이아몬드 + 실선 — 다이아몬드가 "전체" 쪽
  • 생명주기: 전체가 소멸해도 부분은 독립 생존
  • 의미: "A는 B를 가진다(has-a)" — 느슨한 포함
  • 예시: 대학교(전체)는 학생(부분)을 포함 — 대학 폐교해도 학생은 존재

속이 빈 다이아몬드 — 전체(다이아몬드) 쪽

⑤ 합성/복합 (Composition) ⭐ — "part-of" 강한 포함
▾
  • 정의: 전체-부분 관계이며 부분이 전체에 완전히 종속 — 강한 포함
  • 표기: 속이 찬 다이아몬드 + 실선 — 다이아몬드가 "전체" 쪽
  • 생명주기: 전체가 소멸하면 부분도 함께 소멸
  • 의미: "A는 B로 이루어진다(part-of)" — 강한 포함
  • 예시: 주문(전체)은 주문항목(부분)으로 구성 — 주문 삭제 시 주문항목도 삭제

속이 찬 다이아몬드 — 생명주기 공유

집합 vs 합성: 다이아몬드 색깔로 구분! 속이 빈=집합(독립 생존) / 속이 찬=합성(함께 소멸) — 시험 단골 함정
⑥ 실현 (Realization) ⭐ — 인터페이스 구현
▾
  • 정의: 클래스가 인터페이스(또는 추상 클래스)의 명세를 구현하는 관계
  • 표기: 속이 빈 삼각형 + 점선 — 구현 클래스→인터페이스 방향
  • vs 상속: 상속=실선+삼각형(클래스→클래스) / 실현=점선+삼각형(클래스→인터페이스)
  • 의미: "A는 B를 구현한다(implements)"
  • 예시: Circle이 Drawable 인터페이스를 구현

속이 빈 삼각형 + 점선 — 인터페이스 구현

집합=빈다이아몬드·독립생존 / 합성=찬다이아몬드·함께소멸 / 실현=빈삼각형+점선·인터페이스구현 / 상속=빈삼각형+실선·클래스상속
🖼️

온라인 쇼핑몰 클래스 다이어그램 예시 ⭐

다이어그램 읽기: 실선=구조적관계 / 점선=약한관계 / 삼각형=상속·실현 / 다이아몬드=집합·합성 / 화살표방향=참조방향

클래스 관계 6종 완전 비교표 ⭐ — 시험 핵심

#관계표기법선 종류의미생명주기강도
① 연관 (Association) ────→ 실선 (화살표) 구조적 참조 / has 관계 독립 중간
② 의존 (Dependency) - - -→ 점선 화살표 일시적 사용 (매개변수·지역변수) 독립 약함
③ 상속/일반화 (Generalization) ────▷ 실선 + 빈 삼각형 is-a 관계 / 클래스→클래스 종속 가장 강함
④ 집합 (Aggregation) ◇──── 실선 + 빈 다이아몬드 has-a / 전체-부분 (약한 포함) 독립 생존 중간 강함
⑤ 합성 (Composition) ◆──── 실선 + 찬 다이아몬드 part-of / 전체-부분 (강한 포함) 함께 소멸 강함
⑥ 실현 (Realization) - - -▷ 점선 + 빈 삼각형 인터페이스 구현 / implements 종속 강함
혼동하기 쉬운 관계 구분법 ⭐
▾
비교 쌍차이점암기법
상속 vs 실현 상속: 실선+빈삼각형(클래스→클래스) / 실현: 점선+빈삼각형(클래스→인터페이스) 인터페이스면 점선 / 클래스면 실선
집합 vs 합성 집합: 빈 다이아몬드(부분 독립) / 합성: 찬 다이아몬드(함께 소멸) 찬(꽉찬)=합성=강한=함께소멸
연관 vs 의존 연관: 실선(속성으로 보유) / 의존: 점선(일시적 매개변수) 점선=약한=일시적=의존
집합·합성 다이아몬드 위치 다이아몬드는 항상 "전체(Owner)" 클래스 쪽에 붙음 다이아몬드=전체쪽 / 반대편=부분
SOLID 원칙과 클래스 다이어그램 연관 ⭐
▾
원칙클래스 다이어그램 표현
SRP (단일책임)하나의 클래스가 하나의 역할만 — 클래스 수 적정, 메서드 분리
OCP (개방폐쇄)인터페이스·추상클래스로 확장 열고 수정 닫음 — 실현 관계 활용
LSP (리스코프 치환)상속 계층이 올바른지 검토 — 자식이 부모를 완전히 대체 가능
ISP (인터페이스 분리)비대한 인터페이스 분리 — 작은 인터페이스 여러 개
DIP (의존역전)구체 클래스가 아닌 인터페이스에 의존 — 점선 화살표가 추상 쪽으로
감리 관점 체크포인트 ⭐
▾
  • 관계 정확성: 집합·합성·연관 관계가 올바르게 사용되었는가 / 생명주기 정확히 표현
  • 다중성 명시: 모든 연관 관계에 다중성(1, 0..*, 1..*) 표기 여부
  • 추상화 수준: 인터페이스와 추상 클래스가 적절히 사용되었는가
  • 가시성 표기: 속성·메서드의 가시성(-/+/#/~) 정확히 표기 여부
  • 응집도·결합도: 높은 응집도·낮은 결합도 설계 원칙 준수 여부
  • SOLID 원칙: 단일책임·개방폐쇄·의존역전 원칙 적용 여부
  • 코드 추적성: 다이어그램이 실제 구현 코드와 일치하는가
6관계: 연관(실선)·의존(점선)·상속(빈△실선)·집합(빈◇실선)·합성(찬◇실선)·실현(빈△점선) / 점선=약한관계 / 빈=독립 / 찬=강한
