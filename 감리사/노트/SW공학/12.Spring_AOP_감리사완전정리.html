Spring Framework 개요 ⭐

Spring Framework: Rod Johnson이 2003년 발표한 Java 기반 오픈소스 경량 애플리케이션 프레임워크 — IoC(제어의 역전)와 AOP(관점 지향 프로그래밍)를 핵심으로 하며, POJO(Plain Old Java Object) 기반 개발 지원

Spring의 3대 핵심 철학 ⭐

  • POJO 기반 개발 ⭐: 특정 인터페이스·클래스 상속 없이 순수 Java 객체로 개발
  • IoC/DI ⭐: 객체 생성·의존성 관리를 Spring 컨테이너에 위임
  • AOP ⭐: 횡단 관심사(Logging·Transaction·Security)를 핵심 비즈니스에서 분리

DI (Dependency Injection) — 의존성 주입 ⭐⭐

DI 3가지 방식 비교 ⭐
▾

① 생성자 주입 (Constructor Injection) ⭐ 권장

@Service public class OrderService { private final OrderRepository repo; @Autowired // Spring 4.3+ 생략 가능 public OrderService(OrderRepository repo) { this.repo = repo; // final 가능 → 불변 } }
  • final 필드 가능 → 불변성 보장
  • 순환 의존성 컴파일 타임 감지
  • 테스트 시 Mock 주입 쉬움
  • Spring Boot에서 @RequiredArgsConstructor(Lombok)

② Setter 주입 (Setter Injection)

@Service public class OrderService { private OrderRepository repo; @Autowired public void setOrderRepository( OrderRepository repo) { this.repo = repo; } }

③ 필드 주입 (Field Injection) ⚠️ 비권장

@Service public class OrderService { @Autowired private OrderRepository repo; // final 불가 / 테스트 어려움 }
방식권장도불변성순환의존 감지테스트
생성자 주입 ⭐최우선 권장final 가능컴파일 타임쉬움
Setter 주입선택적 의존성에 적합불가런타임보통
필드 주입비권장불가런타임어려움

AOP (Aspect-Oriented Programming) 핵심 ⭐⭐

AOP: 핵심 비즈니스 로직과 횡단 관심사(Cross-Cutting Concerns)를 분리하는 프로그래밍 패러다임 — 로깅·보안·트랜잭션·성능 측정 등을 핵심 코드에서 분리하여 모듈화
AOP 핵심 용어 ⭐⭐ — 시험 단골
▾
용어영문정의예시
Aspect ⭐Aspect횡단 관심사를 모듈화한 것 — Advice + PointcutLoggingAspect, SecurityAspect
Advice ⭐AdviceAspect가 실제로 수행하는 동작 — 언제 무엇을 할지메서드 실행 전 로그 출력
Joinpoint ⭐JoinpointAdvice가 적용될 수 있는 모든 지점 — 메서드 호출·실행·예외 등모든 메서드 호출 시점
Pointcut ⭐PointcutAdvice를 적용할 Joinpoint를 선별하는 표현식com.example.service.* 메서드
Target ⭐Target ObjectAdvice가 적용될 대상 객체 (핵심 비즈니스 로직)OrderService
Weaving ⭐WeavingAspect를 Target에 적용하는 과정컴파일·로드·런타임 시
Proxy ⭐ProxyTarget을 감싸는 대리 객체 — Advice 적용 후 Target 호출JDK Dynamic Proxy / CGLIB Proxy
Advice 5가지 종류 ⭐⭐
▾
Advice 종류어노테이션실행 시점사용 예
Before ⭐@Before메서드 실행 전권한 체크·파라미터 로깅
After Returning ⭐@AfterReturning메서드 정상 반환 후반환값 로깅·캐시 업데이트
After Throwing ⭐@AfterThrowing메서드 예외 발생 후예외 로깅·알림 발송
After (Finally) ⭐@After메서드 실행 후 (정상+예외)리소스 해제·감사 로그
Around ⭐⭐@Around메서드 실행 전·후 모두트랜잭션·성능 측정·캐시
@Aspect @Component public class LoggingAspect { // Pointcut 표현식: service 패키지의 모든 메서드 @Before("execution(* com.example.service.*.*(..))") public void logBefore(JoinPoint jp) { log.info("호출: " + jp.getSignature().getName()); } // Around: 메서드 전후 모두 제어 @Around("execution(* com.example.service.*.*(..))") public Object measureTime(ProceedingJoinPoint pjp) throws Throwable { long start = System.currentTimeMillis(); Object result = pjp.proceed(); // 실제 메서드 호출 long end = System.currentTimeMillis(); log.info("실행 시간: " + (end-start) + "ms"); return result; } }

Pointcut 표현식 ⭐

// execution 표현식 형식 // execution(수식어패턴 반환타입 클래스패턴.메서드패턴(파라미터)) // 예시들 execution(* *(..)) // 모든 메서드 execution(public * *(..)) // public 메서드만 execution(* com.example.service.*(..)) // service 패키지 execution(* com.example..*.*(..)) // 하위 패키지 포함 execution(* com.example.Service.*(String,..)) // 첫 파라미터 String // @annotation: 특정 어노테이션 붙은 메서드 @annotation(org.springframework.transaction.annotation.Transactional)

시점방법

Spring 트랜잭션 관리 ⭐⭐

@Transactional 핵심 속성 ⭐⭐
▾
@Service public class OrderService { // propagation: 전파 동작, isolation: 격리 수준 @Transactional( propagation = Propagation.REQUIRED, // 기본값 isolation = Isolation.READ_COMMITTED, rollbackFor = Exception.class, // 롤백 조건 readOnly = true, // 읽기 전용 최적화 timeout = 30 // 30초 타임아웃 ) public Order createOrder(CreateOrderRequest req) { return orderRepository.save(order); } }

전파(Propagation) 속성 ⭐⭐

전파 속성기존 트랜잭션 있을 때없을 때
REQUIRED ⭐기존 트랜잭션 참여 (기본값)새 트랜잭션 생성
REQUIRES_NEW ⭐기존 트랜잭션 일시 중단 후 새 트랜잭션새 트랜잭션 생성
SUPPORTS기존 트랜잭션 참여트랜잭션 없이 실행
NOT_SUPPORTED기존 일시 중단 후 비트랜잭션 실행비트랜잭션 실행
MANDATORY기존 트랜잭션 참여예외 발생
NEVER예외 발생비트랜잭션 실행
NESTED중첩 트랜잭션 (SavePoint)새 트랜잭션 생성
@Transactional 주의사항 & 데이터 접근 기술 ⭐
▾

@Transactional 주의사항 ⭐

  • self 호출 미적용 ⭐: 같은 클래스 내 메서드 호출 시 AOP 프록시 우회 → 트랜잭션 미적용
  • 기본 롤백 ⭐: RuntimeException(Unchecked)만 롤백 / Checked Exception은 commit
  • rollbackFor ⭐: Checked Exception 롤백 필요 시 명시
  • private 메서드 미적용: 프록시가 override 불가
  • readOnly=true 효과: 읽기 최적화·더티체킹 비활성화
