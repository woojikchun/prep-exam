검증(Verification) vs 확인(Validation) ⭐⭐ — 시험 단골
▾

검증 (Verification) ⭐

"Are we building the product RIGHT?"
제품을 올바르게 만들고 있는가?

  • 명세서·표준·계획 대비 제품 일치 여부
  • 개발 단계 내에서 수행
  • 기법: 코드 리뷰·인스펙션·워크스루
  • 정적 분석·단위 테스트

확인 (Validation) ⭐

"Are we building the RIGHT product?"
올바른 제품을 만들고 있는가?

  • 사용자 요구사항·의도된 사용 충족 여부
  • 최종 완성 후 또는 종료 시 수행
  • 기법: 인수 테스트·사용자 테스트
  • 실제 사용 환경에서 동작 확인
검증(Verification)=명세서 대비 올바르게 만들었나 / 확인(Validation)=사용자 요구 대비 올바른 것을 만들었나 — 자주 혼동!
ISO/IEC 15288 — 시스템 생명주기 프로세스 ⭐
▾
  • 목적: 시스템 공학(Systems Engineering) 관점에서 시스템의 전체 생명주기 프로세스 정의 — SW뿐 아니라 HW·조직·인적 요소 포함
  • 12207과 관계 ⭐: 12207=SW 특화 / 15288=시스템 전체 — 두 표준은 정합성(Harmonization)이 이루어져 있음
  • 4대 프로세스 그룹: ①합의(계약) 프로세스 ②기술 프로세스 ③프로젝트 프로세스 ④조직 가능 프로세스
  • 기술 프로세스 ⭐: 이해관계자 요구 정의→요구사항 분석→아키텍처→설계→구현→통합→검증→전환→확인→운용→유지보수→처분
  • 2015년 개정: ISO/IEC 15288:2015가 현행 버전
12207=SW생명주기 3분류(기본·지원·조직) / 기본=획득·공급·개발·운용·유지보수 / 검증=명세대비(RIGHT way)·확인=요구대비(RIGHT thing)

ISO/IEC 15504 (SPICE) — 프로세스 능력 평가 ⭐⭐

SPICE (Software Process Improvement and Capability dEtermination): SW 프로세스의 역량 수준을 평가하고 개선하기 위한 국제 표준 프레임워크 — 2003년 ISO/IEC 15504로 공식화, 현재는 ISO/IEC 33000 시리즈로 대체 중
SPICE 6가지 능력 수준 상세 ⭐⭐
▾
레벨명칭핵심 특징프로세스 속성(PA)
레벨 0 ⭐ 불완전 (Incomplete) 프로세스가 구현되지 않거나 목적 달성 실패 / 산출물 없음 (없음)
레벨 1 ⭐ 수행 (Performed) 프로세스가 목적을 달성 / 작업 산출물 산출 가능 / 계획·문서화 미흡 PA 1.1 프로세스 수행
레벨 2 ⭐ 관리 (Managed) 프로세스가 계획·모니터링·조정되며 관리됨 / 작업 산출물이 정의·통제됨 PA 2.1 수행 관리 / PA 2.2 작업 산출물 관리
레벨 3 ⭐ 확립 (Established) 조직 표준 프로세스 기반으로 수행 / 프로세스가 문서화·표준화·정의됨 PA 3.1 프로세스 정의 / PA 3.2 프로세스 배치
레벨 4 예측 (Predictable) 프로세스가 정량적 한계 내에서 일관되게 수행 / 측정 데이터 수집·분석 PA 4.1 프로세스 측정 / PA 4.2 프로세스 통제
레벨 5 혁신 (Innovating) 지속적 개선으로 현재·미래 비즈니스 목표 충족 / 혁신적 프로세스 변화 PA 5.1 프로세스 혁신 / PA 5.2 프로세스 최적화

등급달성 비율

CMMI 심화 — 2가지 표현 방식 & 프로세스 영역 ⭐⭐

CMMI(Capability Maturity Model Integration): SEI(미 소프트웨어공학연구소)가 개발한 프로세스 개선 모델 — 단계 표현(Staged)과 연속 표현(Continuous) 두 가지 표현 방식 제공. 현재 버전: CMMI v2.0 (2018)
2가지 표현 방식: 단계 표현 vs 연속 표현 ⭐⭐
▾

단계 표현 (Staged Representation) ⭐

  • 조직 전체의 성숙도를 5레벨로 평가
  • 레벨 1~5 순서로 단계적 개선
  • 측정 지표: 성숙도 레벨(Maturity Level)
  • 적용 단위: PA(Process Area) 그룹
  • 장점: 조직 전체 개선 로드맵 명확
레벨PA 그룹
레벨 2REQM·PP·PMC·SAM·MA·PPQA·CM
레벨 3RD·TS·PI·VER·VAL·OPF·OPD·OT·IPM·RSKM·DAR
레벨 4OPP·QPM
레벨 5OID·CAR

연속 표현 (Continuous Representation) ⭐

  • 개별 프로세스 영역(PA)의 능력 수준을 독립적 평가
  • 각 PA마다 0~5 능력 수준(CL) 독립 결정
  • 측정 지표: 능력 수준(Capability Level)
  • 적용 단위: 개별 PA
  • 장점: 특정 영역 집중 개선 가능·유연성
능력 레벨(CL)명칭
CL 0불완전(Incomplete)
CL 1수행됨(Performed)
CL 2관리됨(Managed)
CL 3정의됨(Defined)
단계 표현=조직 전체 성숙도 레벨(1~5) / 연속 표현=개별 PA 능력 레벨(0~5) — 두 표현은 동일 목표·다른 관점!

ISO 9001 · ISO/IEC 27001 · ISO/IEC 25000 ⭐

① ISO 9001 — 품질 경영 시스템 (QMS) ⭐⭐
▾
  • 대상: 모든 산업 분야 — 제조·서비스·IT 포함
  • 목적: 고객 만족도를 지속적으로 향상시키는 QMS 요구사항 정의
  • PDCA 사이클 ⭐: Plan(계획)→Do(실행)→Check(점검)→Act(조치)
  • 위험 기반 사고 ⭐: 2015 개정판에서 위험 관리 강화
  • ISO/IEC 90003 ⭐: ISO 9001을 SW에 적용하는 가이드라인
  • 인증 방법: 외부 인증 기관(CB)의 심사 후 인증서 발급

② ISO/IEC 27001 — 정보보안 경영 시스템 (ISMS) ⭐⭐
▾
  • 목적: 정보 자산의 기밀성·무결성·가용성(CIA)을 보호하는 ISMS 요구사항
  • 적용 범위: 조직의 정보보안 전반 — 기술적·물리적·관리적 통제
  • 구조: ISO 9001과 유사한 PDCA 기반 / HLS(High Level Structure) 준수
  • 부속서 A: 93개 통제 항목(ISO/IEC 27002 기반) — 2022년 개정
  • 인증: 제3자 인증 기관 심사 → 3년 인증 유효기간
  • 국내 ISMS-P ⭐: KISA가 관리하는 국내 인증 — 정보보호 및 개인정보보호 관리체계
