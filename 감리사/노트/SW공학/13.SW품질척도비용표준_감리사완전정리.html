ISO/IEC 25010 — 제품 품질 8특성 ⭐⭐

SW 메트릭 (Metrics) & 척도 ⭐

① McCabe 순환 복잡도 (Cyclomatic Complexity) ⭐⭐
▾
  • 정의: Tom McCabe(1976) 제안 — 프로그램의 독립적 실행 경로 수를 측정하는 복잡도 지표
  • 계산 공식 ⭐: V(G) = E - N + 2P (그래프 기반)
  • 간편 공식 ⭐: V(G) = 분기문 수(if·while·for·case·&&·||·?) + 1
  • 권장 기준 ⭐: 10 이하 권장 / 20 이상 위험 / 50 이상 재설계
  • 테스트 수: V(G)는 최소 필요 테스트 케이스 수와 동일
  • 활용: 코드 품질 측정·리팩토링 우선순위·테스트 설계
V(G) = E - N + 2P E = 간선(Edge) 수 N = 노드(Node) 수 P = 연결 컴포넌트 수 (보통 1) // 간편 계산 예시 if (a > 0) { // +1 if (b > 0) { // +1 while (c > 0) { // +1 c--; } } } // V(G) = 3 + 1 = 4 // 해석: 최소 4개 테스트케이스로 // 모든 경로 커버 가능
V(G) 범위평가권고사항
1 ~ 10낮음 ✅권장 범위 — 간단하고 이해하기 쉬움
11 ~ 20보통 ⚠️주의 필요 — 테스트 강화
21 ~ 50높음 ❌위험 — 리팩토링 권장
50 초과매우 높음 🚨재설계 필요
② Halstead 복잡도 메트릭 ⭐
▾
  • 정의: Maurice Halstead(1977) — 토큰 기반 복잡도 측정
  • n1: 고유 연산자 수
  • n2: 고유 피연산자 수
  • N1: 연산자 총 출현 수
  • N2: 피연산자 총 출현 수
// 주요 지표 Program Length: N = N1 + N2 Program Vocab: n = n1 + n2 Volume: V = N × log₂(n) Difficulty: D = (n1/2) × (N2/n2) Effort: E = D × V Time to Implement: T = E / 18 Delivered Bugs: B = V / 3000

④ 기타 주요 품질 척도 ⭐
▾
척도설명계산/기준

① COCOMO (Constructive Cost Model) ⭐⭐
▾

Barry Boehm(1981) 제안 — LOC(소스코드 줄 수) 기반 비용 추정

COCOMO 기본 모델 공식 ⭐

// 노력(인월) 계산 MM = a × (KLOC)^b // 개발 기간 계산 TDEV = c × (MM)^d // 인원 수 P = MM / TDEV // 예시 (Organic 모드) // a=2.4, b=1.05, c=2.5, d=0.38 MM = 2.4 × (50)^1.05 ≈ 140 인월

모드ab특징

FP 계산 절차 ⭐

// 1단계: 미조정 기능점수(UFP) 계산 // 각 기능 × 복잡도 가중치(단순/보통/복잡) UFP = Σ (기능 유형별 가중치) // 2단계: 기술복잡도 인수(TCF) // 14개 일반시스템특성(GSC) 점수 합산 TCF = 0.65 + (0.01 × Σ GSC) // TCF 범위: 0.65 ~ 1.35 // 3단계: 조정 기능점수(AFP) AFP = UFP × TCF // FP → 비용 환산 비용 = AFP × 단위비용/FP
항목LOC 기반기능점수(FP) 기반
측정 기준소스코드 줄 수기능 단위
언어 독립성언어마다 다름언어 독립적
측정 시점개발 후요구사항 분석 단계부터 가능

③ Putnam 모델 & 기타 추정 기법 ⭐
▾
  • Putnam 모델 ⭐: Rayleigh 분포 곡선 기반 — 인력 투입이 프로젝트 중반에 최대가 되는 패턴 모델링
  • LOC 공식: L = C_K × t_d^{4/3} × P^{1/3} (C_K: 기술 상수)
  • SLIM(Software Lifecycle Management): Putnam 모델 자동화 도구
  • Delphi 기법 ⭐: 전문가 합의 기반 추정 — 익명 토론·반복 수렴
  • 3점 추정 ⭐: (낙관·기대·비관) 평균 — (O+4M+P)/6 (PERT 분포)
// 3점 추정 (PERT) E = (O + 4M + P) / 6 SD = (P - O) / 6 O = 낙관치 (Optimistic) M = 기대치 (Most Likely) P = 비관치 (Pessimistic) // 예시 O=3, M=5, P=10 E = (3+20+10)/6 = 5.5 인월 SD = (10-3)/6 = 1.17
COCOMO=LOC기반·Organic(2.4·1.05)·Embedded(3.6·1.20) / FP=기능점수·언어독립·5유형(EI·EO·EQ·ILF·EIF) / 3점추정=(O+4M+P)/6
