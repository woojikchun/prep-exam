XML 정의 및 특징

XML(eXtensible Markup Language) — W3C 표준(1998). SGML의 단순화 버전.
데이터를 구조적으로 기술하는 메타 언어. 태그를 사용자가 직접 정의 가능.
표현(Presentation)과 데이터(Content)를 분리하는 것이 핵심 철학.

Well-formed vs Valid XML

Well-formed (형식 적합)
XML 문법 규칙 준수

템플릿 기반(Template Matching)으로 동작. XPath를 내부적으로 사용.

XML 네임스페이스 (Namespace)

서로 다른 XML 어휘(vocabulary)에서 동일한 태그 이름이 충돌하는 것을 방지.
URI로 네임스페이스를 식별 (URI가 실제로 접근 가능할 필요는 없음 — 식별자 역할만).

기출 예상 Q&A

Well-formed XML과 Valid XML의 차이를 설명하시오 ▾
Well-formed(형식 적합)은 XML 문법 규칙만 준수한 상태. 루트 요소 하나, 올바른 태그 중첩, 종료 태그 존재, 속성값 따옴표, 대소문자 일치. XML 파서가 파싱 가능한 최소 조건. Valid(유효)는 Well-formed 조건에 더해 DTD 또는 XML Schema(XSD)에 정의된 구조·데이터 타입 규칙까지 만족하는 더 엄격한 조건. Valid이면 반드시 Well-formed이지만, Well-formed라도 Valid가 아닐 수 있음.
DTD와 XML Schema(XSD)의 차이점과 XSD의 장점은? ▾
DTD는 SGML 기반 독자 문법, 데이터 타입은 PCDATA(문자열)만 지원, 네임스페이스 미지원. XSD(XML Schema)는 XML 문법 자체로 작성, 40여 가지 내장 데이터 타입(int, date, decimal 등) 지원, 네임스페이스 완전 지원, minOccurs/maxOccurs로 출현 횟수 정밀 지정, complexType 상속·확장 가능. XSD는 DTD의 한계를 극복한 W3C 표준으로 실무에서 사실상 표준으로 사용됨.
DOM 파서와 SAX 파서의 차이와 각각의 적합한 사용 사례는? ▾
DOM(Document Object Model)은 XML 문서 전체를 메모리에 트리 구조로 로드. 임의 접근·수정·탐색이 자유로우나 대용량 문서에서 메모리 부담이 큼. SAX(Simple API for XML)는 이벤트 기반 순차 파싱. 태그 시작·종료 등 이벤트 발생 시 콜백 호출. 메모리 효율적이고 빠르나 임의 접근 불가, 코드가 복잡. DOM은 소~중용량 문서를 복잡하게 조작할 때, SAX는 대용량 문서를 순차 처리할 때 적합.
XSLT의 역할과 동작 원리를 설명하시오 ▾
XSLT(XSL Transformations)는 XML 문서를 다른 형식으로 변환하는 W3C 표준 언어. 동작 원리: XML 원본 문서 + XSLT 스타일시트(.xsl)를 XSLT 프로세서가 처리해 결과 문서(HTML, PDF, 다른 XML, 텍스트 등)를 생성. 템플릿 매칭(xsl:template match) 방식으로 동작하며 XPath를 내부적으로 사용해 소스 노드를 선택. xsl:for-each, xsl:value-of, xsl:if, xsl:apply-templates 등의 요소를 사용.
XML 네임스페이스가 필요한 이유와 선언 방법은? ▾
서로 다른 XML 어휘에서 동일한 태그 이름이 충돌하는 것을 방지하기 위해 필요. 예: HTML의 <table>과 가구 데이터의 <table>이 같은 문서에 있을 때. 선언: xmlns:prefix="URI" 형식. 접두사(prefix)는 문서 내 구분자, URI는 네임스페이스 식별자(실제 접근 가능 URL일 필요 없음). 기본 네임스페이스는 xmlns="URI"로 선언하며 자식 요소에 상속됨. xmlns=""로 기본 네임스페이스 해제 가능.
SOAP, WSDL, UDDI의 역할과 관계를 설명하시오 ▾
SOAP(Simple Object Access Protocol): XML 기반 메시지 교환 프로토콜. Envelope-Header-Body 구조. HTTP·SMTP 등 전송 프로토콜 독립. WSDL(Web Services Description Language): 웹서비스 인터페이스를 XML로 기술하는 서비스 계약서. 서비스 위치(endpoint), 오퍼레이션, 메시지 형식 정의. UDDI(Universal Description, Discovery, Integration): 웹서비스 등록·검색 디렉터리. 관계: 서비스 제공자가 WSDL 작성 → UDDI에 등록 → 서비스 소비자가 UDDI에서 검색 → WSDL 참조 → SOAP 메시지로 호출.
XQuery의 FLWOR 표현식을 SQL과 비교해 설명하시오 ▾
XQuery는 XML 데이터베이스 질의 언어. FLWOR 표현식은 For(FROM), Let(변수 할당), Where(WHERE), Order by(ORDER BY), Return(SELECT)으로 구성. For는 반복 변수 바인딩, Let은 변수 선언·할당(SQL에 없는 개념), Where는 조건 필터, Order by는 정렬, Return은 결과 XML 생성. XPath를 내부적으로 사용해 XML 노드를 탐색. SQL이 관계형 DB의 질의 언어이듯 XQuery는 XML DB의 질의 언어.
