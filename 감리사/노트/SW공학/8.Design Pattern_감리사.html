팩토리메서드서브클래스가 객체 결정추상팩토리팩=단일계층, 추상=관련객체 군
싱글톤인스턴스 1개 보장—getInstance() 정적메서드
빌더복잡한 객체를 단계적 생성추상팩토리빌더=구성과정 중시, 추상=결과 중시
어댑터인터페이스 변환데코레이터·브리지·퍼사드어댑터=호환, 데코=기능추가, 퍼사드=단순화
데코레이터동적 기능 추가 (래핑)프록시데코=기능추가, 프록시=접근제어
컴포지트트리구조, 부분-전체데코레이터컴포지트=동일타입 계층, 데코=다른타입 래핑
옵저버1:N 의존, 상태변화 통보메디에이터옵저버=발행구독 직접, 메디에이터=중개자
스트래티지알고리즘 교체 가능스테이트·템플릿메서드스트래티지=외부주입, 스테이트=내부상태, 템플릿=상속
커맨드요청을 객체로 캡슐화책임연쇄커맨드=Undo/Redo 가능, 책임=다음핸들러 위임
템플릿메서드알고리즘 골격, 서브클래스 구현스트래티지템플릿=상속(컴파일 타임), 스트래티지=합성(런타임)
이터레이터순차 접근, 내부구조 은닉—hasNext()/next() 인터페이스

★★ 어댑터, 커맨드, 템플릿메서드, 추상팩토리, 퍼사드

감리 포인트 (설계·개발 단계 감리)
설계 단계: 패턴 적용 근거가 아키텍처 설계서에 명시되었는가? 불필요한 복잡성(Over-engineering) 여부
구현 단계: 싱글톤 — 멀티스레드 환경에서 동기화(synchronized) 처리 여부
                팩토리 — 객체 생성 로직이 클라이언트 코드와 분리되었는가
                옵저버 — 메모리 누수(리스너 해제) 처리 여부
유지보수: OCP(개방-폐쇄 원칙) 준수 여부; 변경 시 기존 코드 수정 없이 확장 가능한가
SOLID 원칙 ↔ 패턴 연결 (자주 출제)
SRP(단일책임) → 퍼사드, 커맨드
OCP(개방폐쇄) → 스트래티지, 데코레이터, 팩토리메서드
LSP(리스코프치환) → 팩토리메서드, 컴포지트
ISP(인터페이스분리) → 어댑터, 퍼사드
DIP(의존역전) → 추상팩토리, 팩토리메서드, 브리지
