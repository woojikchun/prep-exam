순서 다이어그램 (Sequence Diagram) ⭐⭐

순서 다이어그램: 객체 간의 메시지 교환을 시간 순서에 따라 표현하는 상호작용 다이어그램 — 유스케이스의 특정 시나리오를 상세화하는 데 가장 많이 사용
순서 다이어그램 핵심 구성 요소 ⭐
▾
구성 요소표기설명
생명선 (Lifeline) ⭐박스:이름 + 수직 점선객체의 존재 기간 표현 / 상단 박스에 :클래스명 표기
활성화 (Activation) ⭐생명선 위 좁은 직사각형객체가 메서드를 실행 중인 기간 (실행 명세)
동기 메시지 ⭐실선 + 채운 삼각형 →호출하고 응답을 기다림 (블로킹)
비동기 메시지실선 + 빈 화살표 →호출 후 응답을 기다리지 않음
반환 메시지 ⭐점선 + 빈 화살표 --→메서드 실행 후 호출자에게 결과 반환
자기 호출자신으로 되돌아오는 화살표객체가 자신의 메서드 호출
생성 메시지점선 + «create»새 객체 생성 (생명선 시작)
소멸 메시지생명선 끝 X객체 소멸
결합 프래그먼트 (Combined Fragment) ⭐ — 제어 구조
▾
키워드의미대응 코드
alt ⭐조건 분기 — 조건에 따라 하나만 실행if-else / switch
opt ⭐선택적 실행 — 조건이 참일 때만if (조건만, else 없음)
loop ⭐반복 — 조건이 참인 동안 반복for / while
par병렬 실행 — 동시에 실행Thread / 비동기 병렬
break중단 조건 — 조건 충족 시 전체 중단break / exception
ref다른 순서 다이어그램 참조메서드 호출 참조
critical원자적 실행 — 중단 없이 완전 실행synchronized / lock
alt=조건분기(if-else 모두) / opt=조건만 있고 else 없음 — 혼동 주의!
순서다이어그램: 생명선(수직점선)·활성화(직사각형)·동기(실선+채운화살표)·반환(점선+빈화살표) / alt=if-else / opt=if만 / loop=반복

통신 다이어그램 (Communication Diagram) ⭐

정의 & 특징

  • 정의: 객체 간 관계와 메시지를 함께 표현 — 순서 다이어그램과 동일한 정보를 다른 형식으로
  • 메시지 순서: 번호(1, 1.1, 2...)로 순서 표시
  • 장점: 객체 간 구조적 관계 파악 용이
  • 단점: 시간 흐름 파악이 순서 다이어그램보다 어려움
  • 구 명칭: 협력 다이어그램 (Collaboration Diagram)

통신 다이어그램 — 번호로 순서 표현

순서 vs 통신 다이어그램 비교 ⭐

항목순서 다이어그램통신 다이어그램
표현 축수직 시간축 (위→아래)객체 네트워크 + 번호 순서
시간 흐름한눈에 파악 쉬움번호로만 순서 파악
관계 파악객체 관계 파악 어려움객체 간 구조 파악 용이
정보량동일 — 의미상 동등동일 — 서로 변환 가능
선호 상황시간 순서 중요한 설계객체 구조 중심 설계
순서·통신=의미상 동등·상호변환가능 / 순서=시간축 명확 / 통신=관계구조 명확·메시지번호로 순서 / 구 명칭=협력다이어그램

활동 다이어그램 (Activity Diagram) ⭐⭐

활동 다이어그램: 시스템이나 비즈니스 프로세스의 흐름을 표현 — 순차·분기·병렬·반복 처리를 시각화. 순서도(Flowchart)와 유사하지만 병렬 처리 표현이 강점
핵심 구성 요소 ⭐
▾
요소표기의미
초기 노드● 채운 원활동의 시작점 (하나)
종료 노드◉ 원 안 채운 원활동의 종료점 (여럿 가능)
액션 노드 ⭐둥근 직사각형단일 작업·행동
결정 노드 ⭐◇ 마름모분기 조건 (1개 입력, 다수 출력)
병합 노드◇ 마름모분기 합류 (다수 입력, 1개 출력)
포크 (Fork) ⭐굵은 수평선병렬 시작 (1개 입력→다수 출력)
조인 (Join) ⭐굵은 수평선병렬 합류 (다수 입력→1개 출력)
스윔레인 ⭐수직 구획책임 영역 분리 (역할별)

스윔레인·결정·포크·조인 표현

활동다이어그램: ●시작·◉종료·둥근사각형=액션·◇마름모=결정/병합·굵은선=포크/조인(병렬) / 스윔레인=역할구획 / 병렬처리가 순서도와의 차이
⚡

상태 머신 다이어그램 (State Machine Diagram) ⭐⭐

상태 머신 다이어그램: 객체가 생명주기 동안 거치는 상태 변화와 상태 전이를 일으키는 이벤트·조건·동작을 표현 — 복잡한 객체 동작 모델링에 적합

핵심 구성 요소 ⭐

요소표기설명
초기 상태 ⭐● 채운 원시작점 (딱 1개)
최종 상태 ⭐◉ 이중 원종료점 (여럿 가능)
상태 (State) ⭐둥근 직사각형객체의 특정 조건·상황 (진입·퇴출 동작 포함)
전이 (Transition) ⭐화살표 실선상태 간 이동
이벤트 ⭐화살표 위 레이블상태 전이를 일으키는 촉발 조건
가드 (Guard) ⭐[조건]전이 허용 여부 조건
동작 (Action)/ 동작명전이 시 실행되는 행동

전이 레이블 형식

이벤트 [가드] / 동작
# 예: 결제완료 [금액>0] / 영수증발행

주문 상태 변화 다이어그램

상태 내부 구조 & 복합 상태 ⭐
▾
  • 진입 동작(entry): 상태에 진입할 때 실행 — entry / 동작명
  • 퇴출 동작(exit): 상태를 떠날 때 실행 — exit / 동작명
  • do 동작: 상태 유지 중 지속 실행 — do / 동작명
  • 내부 전이: 상태를 떠나지 않고 이벤트 처리 (진입/퇴출 동작 없음)
  • 복합 상태(Composite State): 상태 안에 하위 상태 포함 — 계층적 상태 기계
  • 히스토리 상태(History State) ⭐: H 기호 — 이전에 있던 하위 상태로 복귀
배송중 상태 {
  entry / 배송알림전송()
  do / 위치추적()
  exit / 도착확인요청()
}
상태머신: ●초기·◉최종·둥근사각형=상태·화살표=전이 / 레이블=이벤트[가드]/동작 / entry·do·exit=상태 내 동작 / 히스토리=H기호·이전상태복귀

컴포넌트 다이어그램 (Component Diagram) ⭐⭐

컴포넌트 다이어그램: 시스템을 구성하는 소프트웨어 컴포넌트들의 구조·인터페이스·의존 관계를 표현 — 아키텍처 설계, 모듈 간 인터페이스 명세에 활용
구성 요소표기설명
컴포넌트 ⭐직사각형 + 우측 상단 컴포넌트 아이콘 + «component» 스테레오타입독립 교체 가능한 SW 모듈
제공 인터페이스 (Lollipop) ⭐컴포넌트 경계에 붙은 원 ○다른 컴포넌트에 제공하는 서비스
필요 인터페이스 (Socket) ⭐컴포넌트 경계에 붙은 반원 )다른 컴포넌트에서 필요한 서비스
어셈블리 커넥터제공 인터페이스 ─○)─ 필요 인터페이스컴포넌트 간 인터페이스 연결
포트 (Port)컴포넌트 경계의 작은 사각형특정 인터페이스 진입점
컴포넌트다이어그램: 컴포넌트(사각형+아이콘)·제공인터페이스(원·Lollipop)·필요인터페이스(반원·Socket)·어셈블리커넥터 / SW 아키텍처 표현
🖥️

배포 다이어그램 (Deployment Diagram) ⭐⭐

배포 다이어그램: 하드웨어 노드에 소프트웨어 아티팩트가 어떻게 배포되는지를 표현 — 물리적 인프라·시스템 구성·네트워크 토폴로지 시각화
구성 요소표기설명
노드 (Node) ⭐3D 직육면체 또는 «device»/«execution environment» 스테레오타입HW 장치 또는 SW 실행 환경
아티팩트 (Artifact) ⭐«artifact» 스테레오타입 직사각형배포 가능한 SW 산출물 (.war·.jar·.exe·DB)
통신 경로노드 간 실선네트워크 연결 (HTTP·JDBC·RMI)
«device»물리적 HW 노드서버·PC·스마트폰·네트워크 장비
«execution environment»SW 실행 컨테이너JVM·Node.js·Docker·OS
배포 관계 (deploy)점선 화살표 «deploy»아티팩트가 노드에 배포됨
배포다이어그램: 노드(HW/SW실행환경)·아티팩트(배포파일)·통신경로(프로토콜) / «device»=HW / «execution environment»=JVM·컨테이너 / 물리 인프라 표현

객체 다이어그램 (Object Diagram) ⭐

객체 다이어그램: 클래스 다이어그램의 특정 시점의 인스턴스(스냅샷)를 표현 — 클래스 다이어그램이 타입을 표현한다면 객체 다이어그램은 런타임 인스턴스를 표현

클래스 vs 객체 다이어그램 ⭐

항목클래스 다이어그램객체 다이어그램
표현 대상클래스 (타입 정의)객체 (인스턴스)
이름 표기 ⭐ClassNameobjectName:ClassName
밑줄 ⭐없음이름에 밑줄
속성값타입만 표기실제 값 표기
시점정적 (설계 시)특정 시점 (런타임)
메서드포함없음 (인스턴스 수준)
객체다이어그램: 이름=객체명:클래스명(밑줄) / 속성=실제값 / 메서드 없음 / 링크=연관의 인스턴스 / 런타임 스냅샷 표현

패키지 다이어그램 (Package Diagram) ⭐

정의 & 목적

  • 정의: 클래스·컴포넌트 등을 패키지 단위로 그룹화하고 패키지 간 의존 관계를 표현
  • 목적: 대규모 시스템의 논리적 구조·모듈 의존성 관리
  • 표기: 탭(Tab) 달린 폴더 모양 — 패키지 이름은 탭 또는 중앙
  • 가시성: 패키지 내 요소에 +/- 가시성 표기 가능

패키지 관계

관계표기의미
import ⭐«import» 점선 화살표공개 요소를 자신의 네임스페이스에 추가
access«access» 점선 화살표공개 요소 접근 (네임스페이스 추가 안 함)
merge«merge» 점선 화살표두 패키지를 논리적으로 병합
패키지=폴더모양·네임스페이스단위 / «import»=공개요소 네임스페이스추가 / «access»=접근만 / 대규모시스템 모듈의존성 관리
