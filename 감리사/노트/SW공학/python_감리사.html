다중 상속 & MRO

Python은 다중 상속 지원. 메서드 탐색 순서(MRO — Method Resolution Order)는 C3 선형화 알고리즘으로 결정. ClassName.__mro__로 확인 가능. 다이아몬드 문제를 MRO로 해결
한 줄 암기인스턴스(self)·클래스(cls)·정적(없음) 메서드 구분. 매직 메서드(__xx__)로 연산자 오버로딩. 다중 상속 → C3 MRO로 탐색 순서 결정

기출 예상 Q&A

Python의 GIL이란 무엇이며, 멀티스레딩에 미치는 영향은? ▾
GIL(Global Interpreter Lock)은 CPython 인터프리터에서 한 번에 하나의 스레드만 파이썬 바이트코드를 실행하도록 제한하는 뮤텍스. I/O 바운드 작업(파일, 네트워크)은 I/O 대기 중 GIL을 해제하므로 멀티스레딩 효과가 있음. CPU 바운드 작업(연산)은 GIL로 인해 멀티스레딩이 오히려 오버헤드 증가. 해결책: multiprocessing 모듈(프로세스 기반), Jython/IronPython(GIL 없음), C 확장(NumPy 등).
동적 타이핑과 강한 타이핑의 차이를 Python과 JavaScript를 예로 설명하시오 ▾
동적 타이핑: 변수의 타입이 런타임에 결정됨. Python과 JS 모두 해당. 강한 타이핑: 암묵적 타입 변환 없음. Python에서 "5"+1은 TypeError 발생. 약한 타이핑: 암묵적 변환 허용. JS에서 "5"+1은 "51"(문자열 연결), "5"-1은 4(숫자 변환). Python은 동적이지만 강함, JS는 동적이며 약함. Java는 정적이며 강함.
가변(Mutable)과 불변(Immutable) 객체의 차이와 감리 관점 시사점은? ▾
불변 객체(int, str, tuple)는 생성 후 값 변경 불가. 해시 가능하여 dict 키·set 원소로 사용 가능. 스레드 안전. 가변 객체(list, dict, set)는 생성 후 내부 변경 가능. 해시 불가. 함수 인자로 전달 시 함수 내부에서 수정되면 원본에 영향(참조 전달). 감리 관점: 기본 인자로 가변 객체(mutable default argument) 사용은 함수 호출 간 상태 공유 버그의 주요 원인 — 코드 리뷰 체크리스트 포함 필요.
제너레이터(Generator)란 무엇이며 언제 사용하는가? ▾
제너레이터는 yield 키워드를 사용해 값을 하나씩 생성하는 함수. 호출 시 즉시 실행되지 않고 이터레이터 객체 반환. next() 호출 시마다 yield까지 실행 후 일시정지(지연 평가, Lazy Evaluation). 전체 데이터를 메모리에 올리지 않아 대용량 파일 처리·스트리밍 데이터 파이프라인에 적합. 예: 수백만 행 로그 파일을 한 줄씩 처리 시 list 대신 generator 사용하면 메모리 사용량 대폭 절감.
Python의 다중 상속과 MRO(메서드 탐색 순서)를 설명하시오 ▾
Python은 다중 상속을 지원하며, 같은 메서드가 여러 부모 클래스에 존재할 때 다이아몬드 문제가 발생. 이를 C3 선형화 알고리즘으로 MRO를 결정해 해결. MRO는 자식 클래스 → 왼쪽 부모 → 오른쪽 부모 → 공통 조상 순서. ClassName.__mro__ 또는 ClassName.mro()로 확인 가능. Java는 다중 상속 대신 인터페이스(interface)만 허용해 다이아몬드 문제 회피.
Python 코드 감리 시 주요 확인 항목은? ▾
① 정적 분석 도구(pylint, flake8, mypy) 적용 여부 및 CI 연동. ② 테스트 커버리지(coverage.py) 목표치 설정·준수 여부. ③ 가상환경 및 의존성 관리(requirements.txt/pyproject.toml) 체계. ④ 보안 취약점 분석(bandit) 수행 여부. ⑤ PEP 8 스타일 가이드 준수(black/autopep8 자동화). ⑥ 타입 힌팅 적용 여부(대규모 프로젝트). ⑦ mutable default argument 등 Python 특유의 안티패턴 점검.
