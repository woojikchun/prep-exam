고전 컴퓨터 vs 양자 컴퓨터 ⭐

💻
고전 컴퓨터
기본 단위: 비트 (Bit)
상태: 0 또는 1 (확정)
연산: 순차·병렬 처리
n비트 = 2ⁿ 중 1가지 상태
안정적·현재 상용화
⚛️
양자 컴퓨터
기본 단위: 큐비트 (Qubit)
상태: 0과 1의 중첩 (확률)
연산: 중첩·얽힘·간섭 활용
n큐비트 = 2ⁿ 상태 동시 표현
특수 문제에서 지수적 우위
비트=0 또는 1 확정 / 큐비트=0과 1의 확률적 중첩 / n큐비트=2ⁿ 상태 동시 표현 → 지수적 병렬성
🌀

양자역학 3대 핵심 원리 ⭐

🌊
중첩 (Superposition) ⭐
큐비트는 측정 전까지
|0⟩과 |1⟩ 두 상태를 동시에 가짐
측정 시 확률적으로 하나로 붕괴
→ n큐비트=2ⁿ 상태 동시처리
🔗
얽힘 (Entanglement) ⭐
두 큐비트가 상관 관계로 연결
하나 측정 시 다른 큐비트 상태
즉시 결정 (비국소 상관)
→ 양자 통신·암호 기반
🌊
간섭 (Interference) ⭐
양자 상태의 파동 특성
올바른 답의 확률 진폭↑
틀린 답의 확률 진폭↓
→ 양자 알고리즘 가속의 핵심
🎲
측정 (Measurement)
양자 상태를 관찰하면
중첩이 붕괴(Collapse)되어
0 또는 1로 확정됨
측정이 상태를 변화시킴
🚫
비복제 정리 (No-Cloning)
임의의 양자 상태를 복사 불가
→ 양자 통신의 보안 보장
도청 시 상태 붕괴로 탐지 가능
QKD의 보안 근거
❄️
결맞음 (Coherence)
큐비트가 양자 상태 유지
결어긋남(Decoherence):
환경 간섭으로 양자 상태 손실
→ 극저온(-273°C) 필요

큐비트 수학적 표현 (디랙 표기법·브라-켓)

# 단일 큐비트 상태 |ψ⟩ = α|0⟩ + β|1⟩ # α, β: 복소수 진폭 / |α|² + |β|² = 1 (확률의 합=1) # 측정 시: |0⟩으로 붕괴될 확률 = |α|², |1⟩ 확률 = |β|² # 벨 상태 (최대 얽힘 상태) — 양자 암호·텔레포테이션에 사용 |Φ⁺⟩ = (|00⟩ + |11⟩) / √2 # → 두 큐비트가 항상 같은 값으로 측정됨
중첩=0과1 동시(측정시붕괴) / 얽힘=두큐비트 상관(비국소) / 간섭=올바른답 확률↑ / 비복제=양자상태 복사불가

양자 게이트 (Quantum Gates) ⭐

양자 게이트: 큐비트의 상태를 변환하는 유니터리 연산자 — 고전 로직 게이트와 달리 가역적(Reversible)이며 확률 진폭을 회전시킴
게이트기호역할효과
파울리-X (NOT) ⭐X큐비트 반전 — 고전 NOT 게이트와 동일|0⟩→|1⟩, |1⟩→|0⟩
아다마르 (H) ⭐H중첩 상태 생성 — 가장 중요한 게이트|0⟩→(|0⟩+|1⟩)/√2
|1⟩→(|0⟩-|1⟩)/√2
파울리-YYX+Z 결합 회전|0⟩→i|1⟩, |1⟩→-i|0⟩
파울리-ZZ위상(Phase) 반전|0⟩→|0⟩, |1⟩→-|1⟩
CNOT (CX) ⭐⊕제어 큐비트가 |1⟩일 때 목표 큐비트 반전
얽힘 생성의 핵심 게이트		|00⟩→|00⟩, |01⟩→|01⟩
|10⟩→|11⟩, |11⟩→|10⟩
위상 게이트 (S, T)S, T상대 위상 회전 — 간섭 조작S: 90°, T: 45° 위상 회전
Toffoli (CCX)CCX2개 제어 큐비트가 모두 |1⟩일 때 목표 반전
양자 범용 계산의 기반		고전 AND+NOT 조합
아다마르 게이트 — 중첩 생성의 핵심 ⭐
▾
|0⟩ ──[H]──── (|0⟩+|1⟩)/√2 ← 균등 중첩 (50:50) |1⟩ ──[H]──── (|0⟩-|1⟩)/√2 ← 위상 다른 중첩 ── n개의 H 게이트 적용 → 2ⁿ 상태 동시 중첩 ──
  • 핵심 역할: 고전 0을 양자 중첩 상태로 변환 — 양자 알고리즘의 시작점
  • H² = I: 아다마르 게이트를 두 번 적용하면 원래 상태로 복귀 (자기역원)
  • H를 n큐비트에 적용 → 2ⁿ 개의 가능한 입력을 동시에 탐색 가능
양자 회로 (Quantum Circuit) 구조
▾
# 벨 상태 생성 회로 (얽힘 생성) q₀: |0⟩ ──[H]──●──── │ → (|00⟩+|11⟩)/√2 q₁: |0⟩ ────[X]──── # H로 q₀ 중첩 → CNOT으로 얽힘 생성
  • 회로 구성: 초기화(|0⟩) → 게이트 적용 → 측정(M) 순서
  • 가역성: 모든 양자 게이트는 유니터리 → 역연산 존재 (정보 손실 없음)
  • 범용 게이트 집합: {H, T, CNOT} 조합으로 모든 양자 연산 구현 가능
  • 회로 깊이(Circuit Depth): 연속 게이트 수 → 노이즈·오류 영향
아다마르(H)=중첩생성·양자알고리즘 시작 / CNOT=얽힘생성 / 양자게이트=가역적·유니터리 / {H·T·CNOT}=범용게이트집합

주요 양자 알고리즘 ⭐

쇼어 알고리즘 (Shor's Algorithm, 1994) ⭐ — 소인수분해
▾
  • 목적: 큰 정수(N)를 소인수분해 — RSA 암호 해독의 핵심
  • 고전 vs 양자 복잡도: 고전=지수 시간 O(exp(n^1/3)) / 양자=다항 시간 O(n³) → 지수적 가속
  • 원리: 양자 푸리에 변환(QFT)으로 주기 함수의 주기를 빠르게 탐색 → 소인수 도출
  • 보안 위협: 2048-bit RSA → 현재 고전 컴퓨터=불가능 / 충분한 양자 컴퓨터=수 시간 내 해독
  • 필요 큐비트: RSA-2048 해독에 약 4000개 이상의 논리 큐비트 필요 (오류 수정 포함 수백만)
  • 대응: 양자 내성 암호(PQC) 도입 — NIST PQC 표준화 완료 (2024)
# 쇼어 알고리즘 흐름 N 소인수분해 목표 ① 무작위 a 선택 (1 < a < N) ② gcd(a, N) ≠ 1 이면 인수 발견 (완료) ③ 양자 푸리에 변환으로 f(x)=aˣ mod N 의 주기 r 탐색 ④ r이 짝수이고 a^(r/2) ≢ -1 (mod N) 이면 ⑤ gcd(a^(r/2)±1, N) 으로 소인수 도출
그로버 알고리즘 (Grover's Algorithm, 1996) ⭐ — 탐색
▾
  • 목적: 비정렬 데이터베이스에서 특정 항목 탐색
  • 고전 vs 양자 복잡도: 고전=O(N) 선형 / 양자=O(√N) 제곱근 가속
  • 원리: 오라클(Oracle) + 진폭 증폭(Amplitude Amplification) — √N번 반복으로 정답 확률↑
  • 보안 위협: 대칭키 암호(AES)의 브루트포스 공격 가속 → AES-128=보안 강도 64비트로 감소
  • 대응: 대칭키 길이 2배 증가 (AES-128→AES-256)

양자 암호 기술 체계 ⭐

QKD (양자 키 분배) — BB84 프로토콜 ⭐
▾
QKD(Quantum Key Distribution): 양자역학 원리(비복제 정리·측정 붕괴)를 이용하여 도청이 물리적으로 불가능한 암호 키를 분배

BB84 프로토콜 4단계 (Bennett & Brassard, 1984)

단계내용
① 양자 채널 전송Alice가 무작위 비트(0/1)를 무작위 기저(+/×)로 인코딩한 광자 전송
② Bob 측정Bob이 무작위로 기저(+/×) 선택하여 광자 측정 — 50% 확률로 같은 기저 선택
③ 기저 비교 (고전 채널)공개 채널로 기저 선택 정보 교환 → 같은 기저를 선택한 비트만 유지 (Sifting)
④ 도청 검출일부 비트 비교 → 오류율이 임계값(11%) 초과 시 도청 발생 → 키 폐기 후 재시도
  • 도청 탐지 원리: 도청자(Eve)가 측정하면 양자 상태가 붕괴되어 오류 발생 → 통계적으로 탐지 가능
  • 무조건 안전성(Unconditional Security): 계산량이 아닌 물리 법칙에 기반 → 미래 양자 컴퓨터도 해독 불가
  • 한계: 거리 제한(수백 km) / 광자 손실 / 양자 중계기 필요
  • 상용 현황: 도시 규모 QKD 네트워크 (중국·한국·일본·유럽) / 위성 QKD (중국 묵자호)

양자 컴퓨팅 오류 수정 & 한계

주요 한계·도전 과제

  • 결어긋남(Decoherence): 환경과의 상호작용으로 양자 상태 손실 → 극저온(-273.15°C) 필요
  • 게이트 오류율: 현재 0.1~1% → 오류 수정 필수
  • 큐비트 수 확장: 논리 큐비트 1개 = 물리 큐비트 수백~수천 개 필요 (오류 수정)
  • 결맞음 시간: 마이크로초~밀리초 단위 → 복잡한 계산 한계

양자 오류 수정 (QEC)

  • 표면 코드(Surface Code): 가장 유망한 오류 수정 방식 — 인접 큐비트로 오류 탐지·수정
  • 논리 큐비트 1개: 물리 큐비트 수백~수천 개 필요
  • 오류 임계값: 물리 오류율 1% 미만이어야 오류 수정 의미 있음
  • Google Willow(2024): 큐비트 늘릴수록 오류↓ — 오류 수정의 이정표
