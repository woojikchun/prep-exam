Transformer 등장 배경 — RNN의 한계

Transformer: 2017년 Google Brain이 발표한 논문 "Attention is All You Need" — RNN 없이 Attention 메커니즘만으로 순차 데이터를 처리하는 혁신적 아키텍처

❌ RNN·LSTM의 한계

  • 순차 처리(Sequential): 토큰을 하나씩 순서대로 처리 → 병렬화 불가 → 학습 느림
  • 장기 의존성 문제: 시퀀스가 길어질수록 초기 정보 소실
  • 기울기 소실: LSTM 완화했지만 여전히 존재
  • 고정 크기 컨텍스트: Seq2Seq의 병목 — 모든 정보를 단일 벡터에 압축
  • GPU 활용 한계: 순차 의존성으로 GPU 병렬성 미활용

✅ Transformer의 혁신

  • 완전 병렬 처리: 모든 토큰을 동시에 처리 → GPU 완전 활용
  • 직접 연결: 모든 위치 간 직접 Attention → 장거리 의존성 탁월
  • 스케일 가능: 모델 크기를 쉽게 확장 (GPT-3: 1750억 파라미터)
  • 범용성: NLP를 넘어 Vision·Audio·멀티모달로 확장
  • 학습 효율: 대용량 데이터로 빠른 사전학습 가능
Transformer = RNN 없이 Attention만으로 순차처리 / 병렬계산 가능 → 대용량 사전학습·스케일업 → LLM 시대 열음

Scaled Dot-Product Attention ⭐ 핵심 수식

Attention은 주어진 Query(Q)에 대해 각 Key(K)와의 유사도를 계산하고, 그 가중치로 Value(V)를 가중합하여 출력 — "무엇에 주목할 것인가"를 수치화
Attention(Q, K, V) = softmax( QKᵀ / √dₖ ) · V ─── 각 항목 의미 ──────────────────────────────── Q (Query) : 현재 처리 중인 토큰의 "질문 벡터" K (Key) : 각 토큰의 "색인 벡터" — Q와 내적으로 유사도 계산 V (Value) : 각 토큰의 "실제 값 벡터" — 유사도 가중합 dₖ : Key의 차원(dimension) — 스케일링으로 기울기 안정화 √dₖ로 나누는 이유: dₖ가 크면 내적값이 커져 softmax 기울기 소실 → 스케일링으로 방지 ─── 처리 과정 ──────────────────────────────────── ① Q·Kᵀ → 각 토큰 쌍의 유사도 점수 행렬 (Attention Score) ② / √dₖ → 스케일링 (수치 안정화) ③ softmax → 합이 1인 가중치 (Attention Weight) ④ · V → 가중치로 Value 가중합 → 최종 Context Vector
Self-Attention 동작 예시 ⭐
▾
  • Self-Attention: Q·K·V가 모두 같은 입력 시퀀스에서 나옴 → 문장 내 토큰들 간 관계를 스스로 포착
  • 예시 문장: "The animal didn't cross the street because it was too tired"
  • 결과: "it"에 대한 Attention을 계산하면 "animal"에 높은 가중치 → "it"이 무엇을 가리키는지 파악
  • Q·K·V 생성: 입력 임베딩 X에 학습 가능한 가중치 행렬 Wᵠ·Wᴷ·Wᵛ를 곱하여 생성
  • Cross-Attention: Q는 디코더, K·V는 인코더에서 가져옴 → 번역 시 입력 문장의 어느 부분에 집중할지
# Self-Attention: Q,K,V 모두 동일 입력 X에서 생성 Q = X · Wᵠ (학습 가능한 가중치 행렬) K = X · Wᴷ V = X · Wᵛ # Cross-Attention: Q는 디코더, K·V는 인코더 출력 Q = 디코더 상태 · Wᵠ K = 인코더 출력 · Wᴷ V = 인코더 출력 · Wᵛ
Multi-Head Attention ⭐ — 여러 관점 병렬 처리
▾
  • 개념: Scaled Dot-Product Attention을 h개 헤드로 병렬 수행 — 각 헤드가 서로 다른 관점(구문·의미·위치)에서 관계 포착
  • 수식: MultiHead(Q,K,V) = Concat(head₁, ..., headₕ) · Wᵒ
  • 각 헤드: headᵢ = Attention(QWᵢᵠ, KWᵢᴷ, VWᵢᵛ) — 헤드별 독립적 가중치 행렬
  • Concat 후 투영: h개 헤드 출력을 연결(Concat)하여 Wᵒ로 선형 변환 → 원래 차원으로
  • 효과: 단일 Attention보다 풍부한 표현 학습 / 앙상블 효과
headᵢ = Attention(QWᵢᵠ, KWᵢᴷ, VWᵢᵛ) MultiHead(Q,K,V) = Concat(head₁,...,headₕ) · Wᵒ # BERT-base: h=12, dmodel=768, dₖ=64 # BERT-large: h=16, dmodel=1024 # GPT-3: h=96, dmodel=12288
Attention=softmax(QKᵀ/√dₖ)·V / Q=질문·K=색인·V=값 / √dₖ=스케일링(기울기안정화) / Multi-Head=여러관점병렬처리·Concat후투영

Transformer 핵심 구성요소 상세

위치 인코딩 (Positional Encoding) ⭐
▾
  • 필요 이유: Transformer는 병렬 처리로 토큰 순서를 알 수 없음 → 위치 정보를 명시적으로 주입
  • 덧셈 방식: 토큰 임베딩(Token Embedding) + 위치 인코딩(Positional Encoding) → 같은 차원의 두 벡터를 더함
  • 사인·코사인 PE (원본 Transformer·BERT): PE(pos, 2i) = sin(pos/10000²ⁱ/ᵈ), PE(pos, 2i+1) = cos(pos/10000²ⁱ/ᵈ)
  • 학습 가능 PE: 위치 임베딩을 파라미터로 학습 (GPT, BERT 변형) — 간단하지만 최대 위치 제한
  • 상대적 PE (RoPE): Rotary Position Embedding — GPT-Neo·LLaMA·Mistral 채택 / 긴 컨텍스트 일반화 우수
  • ALiBi: Attention score에 위치 편향 추가 — 학습 길이 초과도 일반화
Feed-Forward Network (FFN) ⭐
▾
  • 구조: 각 위치마다 독립적으로 적용되는 2층 완전연결 네트워크 (Position-wise FFN)
  • 수식: FFN(x) = max(0, xW₁ + b₁)W₂ + b₂ (ReLU) 또는 GELU 사용
  • 내부 차원: 보통 dmodel의 4배 (BERT-base: 768→3072→768)
  • 역할: Self-Attention이 포착한 관계를 비선형 변환으로 더 풍부하게 표현
  • SwiGLU: 최신 LLM(LLaMA·Mistral)에서 사용하는 FFN 활성화 함수 (GELU의 변형)
Add & Norm — 잔차 연결 + 레이어 정규화 ⭐
▾
  • 잔차 연결(Residual Connection): 각 서브레이어의 입력을 출력에 더함 → 기울기 소실 방지 / 학습 안정화
  • 수식: output = LayerNorm(x + Sublayer(x)) — x가 지름길(shortcut)로 직접 전달
  • 레이어 정규화(Layer Normalization): 각 샘플의 피처 차원을 정규화 → 배치 크기에 무관 / Transformer 표준
  • Pre-LN vs Post-LN: 원본=Post-LN(서브레이어 후 정규화) / 최신 LLM 대부분=Pre-LN(서브레이어 전 정규화, 학습 안정성↑)
  • RMS Norm: 최신 LLM(LLaMA)에서 LayerNorm 대신 사용 — 연산 효율적
# Post-LN (원본 Transformer) output = LayerNorm(x + Sublayer(x)) # Pre-LN (GPT-2 이후·안정적) output = x + Sublayer(LayerNorm(x))
Masked Self-Attention (마스킹) ⭐
▾
  • 필요 이유: 디코더는 텍스트를 하나씩 생성(자기회귀) — 현재 위치 이후 토큰은 볼 수 없어야 함
  • 방법: Attention Score 행렬에서 미래 위치에 해당하는 값을 -∞로 설정 → softmax 후 0이 됨
  • 효과: 각 위치는 자신과 그 이전 위치의 토큰만 참조 (인과적 마스킹·Causal Masking)
  • BERT의 [MASK]와 다름: BERT의 마스킹은 MLM 학습용(특정 토큰 예측) / 디코더 마스킹은 미래 정보 차단
# 마스킹 행렬 예시 (4토큰, 하삼각 행렬) pos1 pos2 pos3 pos4 pos1 [ 0 -∞ -∞ -∞ ] pos2 [ 0 0 -∞ -∞ ] pos3 [ 0 0 0 -∞ ] pos4 [ 0 0 0 0 ] # softmax 후 -∞는 0이 되어 미래 토큰에 0 가중치
토크나이저 & 임베딩 — 텍스트→수치 변환
▾
