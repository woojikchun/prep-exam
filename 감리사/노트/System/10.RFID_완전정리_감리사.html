RFID (Radio Frequency Identification) 정의

RFID란 무선 주파수(Radio Frequency)를 이용하여 태그(Tag)에 저장된 정보를 비접촉·비가시선(Non-LoS) 방식으로 자동 인식하는 기술

항목내용
핵심 특징비접촉 인식 / 동시 다수 태그 인식(Anti-collision) / 가시선 불필요 / 원거리 인식 가능
바코드와 차이바코드=광학·가시선 필요·1D/2D / RFID=전파·가시선 불필요·읽기·쓰기 가능
구성 3요소태그(Tag) + 리더(Reader/Interrogator) + 미들웨어·백엔드 시스템
정보 전달 방식태그 → 전자기파 → 리더 → 미들웨어 → 데이터베이스·ERP
표준ISO/IEC 18000 시리즈, EPCglobal Gen2 (UHF), ISO 15693 (HF)
RFID = 무선 주파수로 비접촉 자동 인식 / 3요소: 태그 + 리더 + 미들웨어

RFID 동작 원리 — 전자기 유도 vs 후방 산란

전자기 유도 방식
(Electromagnetic Induction)
  • 주파수: LF · HF (125kHz, 13.56MHz)
  • 리더 안테나 자기장이 태그 안테나에 유도전류 발생
  • 태그가 부하 변조로 데이터 전송
  • 통신 거리: 수 cm ~ 1m
  • 원리: 변압기의 1차·2차 코일 유사
vs
후방 산란 방식
(Backscatter)
  • 주파수: UHF · 마이크로파 (860~960MHz, 2.45GHz)
  • 리더가 RF 에너지 송신 → 태그가 반사파 변조
  • 레이더 원리와 유사
  • 통신 거리: 수 m ~ 10m 이상
  • 장거리·고속 이동 물체 인식에 적합
LF·HF=전자기 유도(근거리) / UHF·마이크로파=후방산란(원거리) / 수동형 태그는 리더 전파에서 전력 수확

EPC (Electronic Product Code) — 전자제품코드

항목내용
정의RFID 태그에 저장되는 전자제품코드 — 모든 개별 상품을 전 세계적으로 고유 식별
구조헤더(버전) + EPC 관리자 번호 + 객체 클래스 + 일련번호 (총 96비트~256비트)
EPC Gen2UHF 대역 RFID 태그 국제 표준 (ISO/IEC 18000-63) — 현재 물류·유통 표준
ONSObject Naming Service — EPC 코드를 이용해 제품 정보 서버 위치를 찾는 DNS와 유사한 시스템
EPCISEPC Information Services — 제품 이동 이력·이벤트 정보 공유 표준 (공급망 가시성)
EPC = RFID 태그의 전 세계 고유 식별 코드 / ONS(위치 찾기) + EPCIS(이력 공유) = IoT 기반 공급망

Anti-Collision (충돌 방지) 알고리즘

여러 태그가 동시에 응답할 때 충돌(Collision)을 방지하고 각 태그를 개별 인식하는 기술

확률적 방식

  • Slotted ALOHA: 태그가 랜덤 타임슬롯에서 응답 — EPC Gen2 기반 / Q-Algorithm
  • ALOHA: 순수 랜덤 전송 — 충돌 시 재시도 / 간단하지만 효율 낮음

결정적 방식

  • 이진 트리 탐색(Binary Tree): 태그 ID를 이진 트리로 탐색 — 확실한 인식 보장
  • ISO 15693: 슬롯 방식 Anti-collision 사용
Anti-Collision = 복수 태그 동시 인식 핵심 기술 / Slotted ALOHA(EPC Gen2) vs 이진 트리 탐색

RFID 보안 위협 분류

📡 도청 (Eavesdropping / Sniffing)
무선 통신 채널 도청
⭐핵심
▾
  • 원리: 리더-태그 간 무선 통신을 제3자가 수동적으로 도청 — 태그 ID, 저장 데이터 탈취
  • 위험: 개인정보 유출(교통카드 번호, 여권 정보), 제품 정보 노출
  • 특징: 수동적 공격 — 태그나 리더가 공격 사실을 인지 못함
  • 대응: 통신 데이터 암호화(AES), 물리적 차폐(패러데이 케이지), 보안 채널 사용
🔄 재전송 공격 (Replay Attack)
캡처한 통신 데이터 재전송
⭐핵심
▾
  • 원리: 이전에 캡처한 태그 응답 데이터를 가짜 리더에서 재전송하여 정당한 태그처럼 위장
  • 예시: 출입통제 시스템에서 정당한 직원의 카드 응답을 녹화 후 재사용 → 무단 출입
  • 대응: 챌린지-응답(Challenge-Response) 인증 / 타임스탬프·논스(Nonce) 사용 / 세션 키 방식
🎭 스푸핑 (Spoofing) / 클로닝
태그 위조 · 복제
⭐핵심
▾
  • 스푸핑: 합법적인 태그로 가장하는 위조 태그 사용 — 진짜 태그 정보를 복제하거나 가짜 ID 전송
  • 클로닝: 도청한 태그 ID를 동일하게 복제한 태그 제작 → 정품 위조, 도난 방지 태그 우회
  • 위험: 제품 위조, 출입 통제 우회, 결제 시스템 부정 사용
  • 대응: 암호화·디지털 서명, 태그 고유 물리적 특성(PUF) 활용, 보안 칩 사용
📵 서비스 거부 공격 (DoS / Jamming)
신호 방해·재핑
DoS
▾
  • 재밍(Jamming): RFID 주파수와 동일한 신호를 강하게 방사하여 리더-태그 통신 방해
  • 물리적 차단: 금속 포일·패러데이 케이지로 태그 전파 차단
  • RFID Killer: 강한 전자기 펄스로 태그 IC 칩 파괴 → 영구 비활성화
  • 대응: 주파수 호핑(Frequency Hopping), 신호 강도 모니터링, 이중화 리더 배치
🔓 RFID 중간자 공격 (Man-in-the-Middle)
리더-태그 간 통신 가로채기
MITM
▾
  • 원리: 공격자가 가짜 리더로 태그와 통신하고, 가짜 태그로 실제 리더와 통신 — 양쪽을 중계하며 데이터 변조
  • 릴레이 공격(Relay Attack): NFC·RFID 통신 신호를 장거리로 중계 — 사용자 지갑에서 멀리 떨어진 곳의 리더에 접근
  • 대응: 거리 측정(Distance Bounding Protocol), 상호 인증(Mutual Authentication), TLS 적용
🔐 프라이버시 침해 (Privacy Threats)
개인 추적·위치 정보 노출
프라이버시
▾
  • 위치 추적: 개인이 소지한 RFID 태그(교통카드·여권·의류 태그)를 지속 스캔하여 이동 경로 추적
  • 프로파일링: 구매한 상품에 부착된 태그로 소비 패턴 분석
  • 킬 명령(Kill Command): 태그 기능을 영구 비활성화 — 판매 후 소비자 프라이버시 보호 (EPC Gen2 지원)
  • 슬리핑(Sleeping): 일시적으로 태그 응답 비활성화
  • 대응: Kill 명령, 임시 ID 사용(롤링 코드), 해시 기반 익명화, 메타데이터 최소화
RFID 6대 위협: 도청·재전송·스푸핑(클로닝)·DoS(재밍)·MITM(릴레이)·프라이버시 / Kill Command=영구 비활성화

NFC (Near Field Communication) — RFID 특수 응용

항목내용
정의HF 13.56MHz 기반 단거리(10cm 이내) 무선 통신 기술 — RFID의 특수 응용
통신 모드리더 모드(능동) / 카드 에뮬레이션 모드(수동) / P2P 모드(양방향)

NFC = RFID의 특수 응용 / 13.56MHz / 10cm 이내 / P2P·카드에뮬·리더 3가지 모드 / 모바일 결제
