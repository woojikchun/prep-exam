역전파 (Backpropagation) & 손실 함수

순전파 → 역전파 학습 과정

  • 순전파(Forward): 입력→출력 방향으로 예측값 계산
  • 손실 계산: 예측값과 실제값 차이(Loss) 계산
  • 역전파(Backward): 연쇄 법칙(Chain Rule)으로 출력→입력 방향으로 기울기 계산
  • 가중치 업데이트: 경사 하강법으로 기울기 방향 반대로 가중치 조정
  • 기울기 소실(Vanishing Gradient): 층 깊어질수록 기울기가 0에 수렴 → ReLU·배치 정규화로 완화
  • 기울기 폭발(Exploding Gradient): 기울기가 무한대로 발산 → Gradient Clipping으로 완화

RNN · LSTM · GRU — 순차 데이터 처리 ⭐

RNN (Recurrent Neural Network) 순환 신경망
▾
  • 원리: 이전 시점의 은닉 상태(Hidden State)를 다음 시점에 전달 → 순서 정보(시계열·텍스트) 처리
  • 수식: hₜ = tanh(Wₕhₜ₋₁ + Wₓxₜ + b)
  • 장기 의존성 문제: 시퀀스가 길어질수록 초기 정보가 소실(기울기 소실)
  • 응용: 텍스트 분류·언어 모델·감성 분석 (현재는 Transformer로 대체)
LSTM (Long Short-Term Memory) ⭐핵심
▾
  • 목적: RNN의 장기 의존성 문제 해결 — 게이트(Gate) 메커니즘으로 정보 선택적 기억·망각
  • 핵심 구조 — 3개 게이트:
게이트역할출력 범위
망각 게이트 (Forget Gate) ⭐이전 셀 상태에서 삭제할 정보 결정 — σ(W·[hₜ₋₁,xₜ]+b)0(전부 삭제)~1(전부 유지)
입력 게이트 (Input Gate) ⭐현재 입력에서 저장할 정보 결정 — σ(W·[hₜ₋₁,xₜ]+b)0~1
출력 게이트 (Output Gate) ⭐셀 상태에서 출력할 정보 결정 — σ(W·[hₜ₋₁,xₜ]+b)0~1
  • 셀 상태(Cell State): 컨베이어 벨트처럼 장기 기억을 전달하는 핵심 통로
  • 응용: 기계 번역·음성 인식·시계열 예측·감성 분석
GRU (Gated Recurrent Unit) LSTM 경량화
▾
  • LSTM 대비: 셀 상태를 제거하고 은닉 상태만 사용 / 게이트 2개(업데이트·리셋)로 단순화
  • 업데이트 게이트: 이전 은닉 상태를 얼마나 유지할지
  • 리셋 게이트: 이전 은닉 상태를 얼마나 잊을지
  • 장점: 파라미터 적음·학습 빠름 / 짧은 시퀀스에서 LSTM과 성능 유사
Seq2Seq & Attention (초기 형태) 번역·요약
▾
  • Seq2Seq: 인코더(입력 시퀀스 → 컨텍스트 벡터) + 디코더(컨텍스트 → 출력 시퀀스)
  • 병목 문제: 고정 크기 컨텍스트 벡터에 모든 정보 압축 → 긴 문장에서 성능 저하
  • Attention 메커니즘: 디코더가 매 시점마다 인코더의 모든 은닉 상태를 참조 → 관련 위치에 집중
  • 효과: 긴 문장 번역 성능 개선 / Transformer의 직접 전신
RNN=이전 은닉상태 전달 / LSTM=3게이트(망각·입력·출력)로 장기의존성 해결 / GRU=2게이트 경량화 / Seq2Seq→Attention
⚡

Transformer 아키텍처 ⭐ "Attention is All You Need" (2017)

Transformer: RNN 없이 Self-Attention만으로 순차 데이터를 처리 — 병렬 계산 가능하여 학습 속도↑ / 모든 위치 간 직접 연결 → 장거리 의존성 탁월

인코더 (Encoder)

  • Self-Attention: 입력 시퀀스 내 모든 토큰 간 관계 계산
  • Feed-Forward Network: 위치별 독립적 변환
  • Add & Norm: 잔차 연결 + Layer Normalization
  • N개 인코더 블록 반복 (BERT=12·24개)
  • 용도: 이해(이진 분류·NER·QA)

디코더 (Decoder)

  • Masked Self-Attention: 미래 토큰을 보지 못하게 마스킹
  • Cross-Attention: 인코더 출력을 Key·Value로 참조
  • Feed-Forward Network
  • N개 디코더 블록 반복 (GPT=12~96개)
  • 용도: 생성(번역·요약·텍스트 생성)
Transformer = Encoder(Self-Attention) + Decoder(Masked-Self-Attention + Cross-Attention) / 병렬처리·장거리의존성

Attention & Self-Attention ⭐ — 핵심 메커니즘

Scaled Dot-Product Attention 수식 ⭐최중요
▾
Attention(Q, K, V) = softmax(QKᵀ / √dₖ) · V # Q(Query): 현재 처리 중인 토큰의 "질문" # K(Key): 각 토큰의 "색인" — Q와 유사도 계산 # V(Value): 각 토큰의 실제 "값" — 유사도로 가중합 # √dₖ: 키 차원으로 나눠 스케일링 → 기울기 안정화
  • Self-Attention: Q·K·V가 모두 같은 시퀀스에서 나옴 → 문장 내 토큰 간 관계 포착
  • 예시: "The animal didn't cross the street because it was too tired" → "it"이 "animal"을 참조
Multi-Head Attention ⭐ 여러 관점 병렬 처리
▾
  • 원리: Attention을 h개의 헤드(Head)로 병렬 수행 → 서로 다른 관점(구문·의미·위치)에서 관계 포착
  • 수식: MultiHead(Q,K,V) = Concat(head₁,...,headₕ)·Wᵒ
  • 각 헤드: headᵢ = Attention(QWᵢQ, KWᵢK, VWᵢV)
  • BERT-base: 12헤드 / BERT-large: 16헤드 / GPT-3: 96헤드
위치 인코딩 (Positional Encoding) ⭐ 순서 정보 주입
▾
  • 문제: Transformer는 병렬 처리이므로 토큰 순서 정보가 없음 → 위치 정보를 별도로 주입
  • 사인·코사인 PE: 각 위치를 사인·코사인 함수로 인코딩 (BERT·원본 Transformer)
  • 학습 가능 PE: 위치 임베딩을 학습으로 결정 (GPT, BERT 변형)
  • 상대적 PE: RoPE(GPT-Neo·LLaMA)·ALiBi — 긴 컨텍스트 일반화에 유리
Attention = softmax(QKᵀ/√dₖ)·V / Q=질문·K=색인·V=값 / Multi-Head=여러 관점 병렬 / 위치인코딩=순서정보 주입

BERT 사전학습 — MLM & NSP ⭐
▾
  • MLM(Masked Language Model): 입력 토큰의 15%를 [MASK]로 대체 → 마스킹된 토큰 예측 학습 → 양방향 문맥 학습
  • 15% 중 처리: 80%=[MASK] 대체 / 10%=랜덤 토큰 / 10%=원래 토큰 유지
  • NSP(Next Sentence Prediction): 두 문장이 연속인지(IsNext) 무작위인지(NotNext) 예측 → 문장 간 관계 학습
  • 입력 형식: [CLS] 문장A [SEP] 문장B [SEP]
  • [CLS] 토큰: 전체 문장 표현 — 분류 태스크에서 이 벡터 사용
BERT=인코더·양방향·MLM+NSP·이해강점 / GPT=디코더·단방향·CLM·생성강점 / [CLS]=전체문장표현

전이 학습 & 자기 지도 학습 ⭐
▾
개념설명
전이 학습 ⭐대용량 데이터로 사전 학습(Pre-training)한 모델을 소규모 태스크에 파인튜닝(Fine-tuning) — 적은 데이터로 높은 성능
특성 추출사전학습 모델 가중치 동결 → 새 분류 헤드만 학습 (소규모 데이터에 적합)
전체 파인튜닝사전학습 모델 전체를 낮은 학습률로 재학습 (중규모 이상 데이터)
자기 지도 학습레이블 없이 데이터 자체에서 지도 신호 생성 (BERT의 MLM, GPT의 CLM)
대조 학습(Contrastive)유사 샘플은 임베딩 공간에서 가깝게, 다른 샘플은 멀게 학습 (SimCLR, CLIP)
LLM 양자화 & 경량화 기법
▾
  • 양자화(Quantization): FP32 가중치를 INT8·INT4로 압축 → 모델 크기↓·추론 속도↑ (GPTQ·GGUF 포맷)
  • 지식 증류(Knowledge Distillation): 큰 교사(Teacher) 모델의 출력을 소프트 레이블로 작은 학생(Student) 모델 학습 (DistilBERT=BERT 절반 크기)
  • 가지치기(Pruning): 중요도 낮은 가중치를 0으로 만들어 희소 모델 생성
  • MoE(Mixture of Experts): 입력에 따라 일부 전문가 네트워크만 활성화 → 파라미터↑ but 실제 연산↓ (GPT-4·Mistral 8x7B)
  • Speculative Decoding: 소형 모델이 먼저 예측 → 대형 모델이 검증 → 생성 속도↑
