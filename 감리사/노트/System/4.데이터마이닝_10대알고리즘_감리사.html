① C4.5 — 의사결정 트리 ⭐

ALGORITHM #01 · 분류 · 지도학습 · Ross Quinlan(1993) · ID3 개선판
핵심 원리 — 이득 비율(Gain Ratio) ⭐
▾
  • ID3의 문제: 정보 이득(Information Gain)만 사용 → 카디널리티 높은 속성(주민번호 등) 과도하게 선호
  • C4.5 해결책: 이득 비율(Gain Ratio) 사용 = 정보이득 / 분할정보(SplitInfo) → 카디널리티 패널티 부여
  • 가지치기: 오류 기반 사후 가지치기(Error-based Post-Pruning) → 과적합 방지
  • 연속형 속성: 임계값(Threshold) 기반 이진 분할 → 연속형 데이터 처리 가능
  • 결측값 처리: 결측값 있는 샘플도 학습에 활용 (확률적 처리)
  • 규칙 생성: 트리를 IF-THEN 규칙으로 변환 가능 → 해석 용이
정보이득 IG(A) = H(D) - Σ|Dᵥ|/|D| · H(Dᵥ) 분할정보 SplitInfo(A) = -Σ|Dᵥ|/|D| · log₂(|Dᵥ|/|D|) 이득비율 GainRatio(A) = IG(A) / SplitInfo(A) ← C4.5 핵심
ID3=정보이득(카디널리티 편향) / C4.5=이득비율(카디널리티 보정) / CART=지니불순도 — 세 알고리즘 비교가 핵심!
C4.5 vs ID3 vs CART 비교 ⭐
▾

C4.5=ID3개선·이득비율(GainRatio)·연속형·결측값처리·사후가지치기 / ID3=정보이득·카디널리티편향 / CART=지니·이진분할·분류+회귀

⑩ CART (Classification And Regression Trees) ⭐

ALGORITHM #10 · 분류+회귀 · 지도학습 · Breiman et al.(1984) · 이진분할
핵심 원리 — 지니 불순도 & 이진 분할 ⭐
▾
  • 지니 불순도: 노드의 순수도 측정 — 0(완전 순수)~0.5(완전 혼합, 이진 분류)
  • 이진 분할만: 모든 분할이 정확히 두 가지(Yes/No) — 완전 이진 트리 생성
  • 회귀 트리: 연속형 출력값 예측 — 분할 기준으로 분산 감소(MSE) 사용
  • 비용복잡도 가지치기(CCP): α 파라미터로 트리 복잡도 vs 오류율 트레이드오프 조절
  • sklearn 기본: scikit-learn의 DecisionTreeClassifier/Regressor가 CART 구현
  • 랜덤 포레스트·XGBoost의 기반: 여러 CART 트리를 앙상블
지니 불순도 G(t) = 1 - Σpᵢ² → pᵢ: 클래스 i의 비율 → 0에 가까울수록 순수한 노드 (같은 클래스만) 분할 이득 ΔG = G(부모) - [|왼|/|전체|·G(왼) + |오|/|전체|·G(오)]
CART=지니불순도·이진분할전용·분류+회귀모두·sklearn기본·랜덤포레스트 기반 / 분류트리=지니 / 회귀트리=MSE(분산감소)
⚔️

③ SVM (Support Vector Machine) ⭐

ALGORITHM #03 · 분류·회귀 · Vapnik(1963/1995) · 마진 최대화
마진 최대화 & 커널 트릭 ⭐
▾
  • 서포트 벡터(Support Vector): 결정 경계에 가장 가까운 소수의 데이터 포인트 — 이것만으로 결정 경계 결정
  • 최대 마진 분류기: 두 클래스를 가장 넓은 여백으로 분리하는 초평면(Hyperplane) 탐색
  • 하드 마진: 오분류 전혀 허용 안 함 (이상치 민감)
  • 소프트 마진(C 파라미터): 슬랙 변수로 일부 오분류 허용 — C↑=좁은마진·적은오류(과적합) / C↓=넓은마진·허용(과소적합)
  • 커널 트릭: 실제로 고차원 변환하지 않고 커널 함수(내적 계산)로 효율적 분류
  • SVR(Support Vector Regression): ε-튜브 내 오류를 무시하며 회귀 예측
커널 함수수식적용 상황
선형(Linear)K(x,z)=xᵀz선형 분리 가능·텍스트·고차원 희소
다항식(Polynomial)K(x,z)=(γxᵀz+r)ᵈ이미지·다항 경계
RBF(가우시안) ⭐K(x,z)=exp(-γ‖x-z‖²)범용·가장 많이 사용
시그모이드K(x,z)=tanh(γxᵀz+r)신경망 유사 경계
SVM=마진최대화·서포트벡터만 사용 / RBF커널=범용·가장 많이 / C↑=과적합·γ↑=복잡경계 / 스케일링 필수 / 대용량 느림
👥

⑧ kNN (k-Nearest Neighbors) ⭐

ALGORITHM #08 · 분류·회귀 · 게으른 학습(Lazy Learning) · 인스턴스 기반
동작 원리 & K 선택 ⭐
▾
  • 원리: 새 데이터 → 훈련 데이터 전체와 거리 계산 → 가장 가까운 K개 이웃 선택 → 다수결(분류) 또는 평균(회귀)
  • 게으른 학습(Lazy Learning): 별도 모델 학습 없음 / 모든 훈련 데이터를 메모리에 보관
  • K=1: 과적합 (노이즈에 민감) / K↑: 과소적합 / 홀수 K 권장 (동점 방지)
  • 거리 척도: 유클리드(기본)·맨해튼·코사인·민코프스키
  • 차원의 저주(Curse of Dimensionality): 고차원 공간에서 모든 점이 비슷한 거리 → 성능 저하
  • 가중 kNN: 가까운 이웃에 더 높은 가중치 부여 → 성능 개선
  • 스케일링 필수: 거리 기반이므로 단위 차이 극복 필요
유클리드 거리 d(x,y) = √Σ(xᵢ-yᵢ)² 맨해튼 거리 d(x,y) = Σ|xᵢ-yᵢ| 학습 복잡도: O(1) ← 저장만 / 예측 복잡도: O(n·d) ← 느림!
kNN=K이웃다수결·게으른학습·학습없음·예측O(nd)느림 / K홀수권장·스케일링필수 / 고차원=차원의저주 / 추천·의료진단
🧮

⑨ Naïve Bayes ⭐

ALGORITHM #09 · 분류 · 확률 기반 · 조건부 독립 가정 · 텍스트 분류 최강
베이즈 정리 & 조건부 독립 ⭐
▾
  • 핵심 가정(Naïve): 모든 특성이 클래스 주어졌을 때 서로 조건부 독립 → 현실에서 위배되지만 실용적으로 우수
  • 가우시안 NB: 연속형 특성이 정규 분포를 따른다고 가정
  • 다항식 NB ⭐: 특성이 다항 분포 (단어 빈도) → 텍스트 분류 표준
  • 베르누이 NB: 이진 특성 (단어 출현 여부)
  • 라플라스(Laplace) 스무딩 ⭐: 훈련 데이터에 없는 단어의 확률=0 방지 → 분자+1, 분모+k(클래스수)
P(C|X) ∝ P(C) · ΠP(xᵢ|C) ← Naïve 가정 사후확률 = 사전확률 × 우도(각 특성 조건부 확률 곱) Laplace: P(xᵢ|C) = (count(xᵢ,C)+1) / (count(C)+|V|)
Naïve Bayes=베이즈+조건부독립가정 / 다항식NB=텍스트·단어빈도 / 라플라스스무딩=0확률방지 / 빠름·소규모탁월 / 스팸분류

④ Apriori — 연관 규칙 마이닝 ⭐

ALGORITHM #04 · 연관규칙 · 비지도학습 · Agrawal & Srikant(1994) · 장바구니분석
핵심 원리 & 3대 지표 ⭐
▾
지지도(Support) sup(A→B) = P(A∩B) = |A∩B| / |전체| → A와 B가 동시에 발생할 비율 (빈도) 신뢰도(Confidence) conf(A→B) = P(B|A) = |A∩B| / |A| → A를 산 사람 중 B도 산 비율 (정확도) 향상도(Lift) lift(A→B) = conf(A→B) / P(B) = sup(A∪B) / (sup(A)·sup(B)) → > 1: 양의 연관 (독립보다 B 구매↑) → = 1: 독립 (무관) / < 1: 음의 연관
  • Apriori 원칙(Apriori Property) ⭐: 빈발 항목 집합의 모든 부분 집합도 빈발 → 비빈발 집합의 상위 집합은 탐색 불필요 (가지치기)
  • 동작: 최소 지지도(min_sup) 이상의 빈발 1-항목 집합 → 빈발 2-항목... 반복 → 연관 규칙 생성 (최소 신뢰도 필터)
  • 단점: 후보 집합 반복 생성·DB 다중 스캔 → 대용량 비효율
  • FP-Growth ⭐: Apriori 개선 — FP-Tree 구조로 DB 스캔 2회만·후보 생성 없음 → 빠름
연관 규칙 예시 & FP-Growth 비교
▾
# 거래 데이터: 5개 거래, 상품 {A,B,C,D,E}
T1: {A,B,C} T2: {A,C} T3: {B,C,D} T4: {A,B,D} T5: {B,C}

min_sup=0.6 (3/5거래)
빈발 1-항목: A(3)·B(4)·C(4) → {A:0.6, B:0.8, C:0.8}
빈발 2-항목: {A,C}(2) ← 0.4 제외 / {B,C}(3) ← 0.6 포함

규칙: B→C conf=3/4=0.75 lift=0.75/0.8=0.94

DB 스캔k번 (항목 크기마다)2번만
후보 생성반복 생성 (많음)없음
메모리적음FP-Tree 필요
속도느림 (대용량 비효율)빠름
Apriori: 지지도·신뢰도·향상도(>1=양의연관) / Apriori원칙=부분집합도빈발·비빈발가지치기 / FP-Growth=DB2회스캔·후보없음·빠름

⑤ EM (Expectation-Maximization) ⭐

ALGORITHM #05 · 군집화·밀도추정 · Dempster et al.(1977) · 잠재변수 모델
E단계·M단계 반복 원리 ⭐
▾
  • 목적: 관측 데이터에 숨겨진 잠재 변수(Latent Variable)가 있을 때 최우도 추정(MLE) 수행
  • 핵심 응용: GMM(가우시안 혼합 모델) — 여러 정규 분포의 혼합으로 복잡한 데이터 분포 모델링

E단계 (Expectation)

  • 현재 파라미터로 각 데이터가 어느 군집에 속할 확률(책임도, responsibility) 계산
  • 소프트 할당 — 하나의 데이터가 여러 군집에 확률적으로 속할 수 있음

M단계 (Maximization)

  • E단계의 책임도를 기반으로 파라미터(μ·σ·π) 업데이트 (로그 우도 최대화)
  • 각 군집의 평균·분산·혼합 비율 재계산

⑥ PageRank ⭐

ALGORITHM #06 · 링크분석 · 비지도 · Page·Brin(Google,1998) · 그래프 기반 랭킹
핵심 원리 — 링크 기반 중요도 ⭐
▾
  • 핵심 직관: 중요한 페이지로부터 많은 링크를 받을수록 그 페이지도 중요 — 민주적 투표 메커니즘
  • 랜덤 서퍼 모델: 웹 사용자가 무작위로 링크를 클릭하는 행동을 시뮬레이션
  • 댐핑 팩터(d) ⭐: 보통 0.85 — 직접 URL 입력(1-d=0.15) vs 링크 클릭(d=0.85) 비율
  • 수렴: 반복 계산으로 PageRank가 수렴 (마르코프 체인 정상 분포)
  • 댄글링 노드: 나가는 링크 없는 페이지 → PageRank 누수 → 모든 페이지로 균등 분배로 처리
PR(A) = (1-d) + d · Σ PR(Bᵢ)/L(Bᵢ) d: 댐핑 팩터 (보통 0.85) Bᵢ: A에 링크를 가진 페이지 L(Bᵢ): Bᵢ에서 나가는 링크 수 → 행렬 거듭제곱으로 수렴값 계산 (멱 반복법)
  • 응용: Google 검색 순위 / 학술 논문 영향력(인용 네트워크) / SNS 영향력 사용자 / 제품 추천
  • HITS 알고리즘: PageRank의 대안 — Hub(좋은 링크 제공)와 Authority(좋은 링크 받음) 구분
PageRank=중요한 페이지에서 많은 링크=중요 / 댐핑팩터d=0.85 / 랜덤서퍼모델 / 반복수렴 / Google검색·영향력분석
