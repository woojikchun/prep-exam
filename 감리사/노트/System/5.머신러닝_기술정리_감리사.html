머신러닝 (Machine Learning) 정의 및 분류

머신러닝: 명시적으로 프로그래밍하지 않고 데이터로부터 패턴을 학습하여 예측·분류·군집화 등의 작업을 수행하는 AI 기법 (Arthur Samuel, 1959)
학습 유형특징대표 알고리즘대표 문제
지도학습 (Supervised)입력(X)과 정답(y) 쌍으로 학습 / 레이블 데이터 필요선형회귀·로지스틱회귀·SVM·결정트리·KNN·나이브베이즈분류·회귀
비지도학습 (Unsupervised)정답 레이블 없이 데이터 구조·패턴 발견K-Means·DBSCAN·PCA·t-SNE·연관 규칙군집화·차원축소·이상탐지
준지도학습 (Semi-supervised)소량 레이블 + 대량 비레이블 데이터 함께 활용자기 훈련(Self-training), 그래프 기반레이블 비용 절감
강화학습 (Reinforcement)에이전트가 환경과 상호작용하며 보상 최대화Q-Learning·SARSA·Policy Gradient게임AI·로봇·자율주행
자기지도학습 (Self-supervised)데이터 자체에서 레이블 생성하여 학습Contrastive Learning, BERT 사전학습표현 학습
ML 학습 유형: 지도(레이블O·분류·회귀) / 비지도(레이블X·군집·차원축소) / 강화(보상최대화·환경상호작용)

ML 핵심 용어

  • 특성(Feature): 입력 변수 X — 모델 학습에 사용
  • 레이블(Label·타깃): 예측해야 할 정답 y
  • 파라미터: 학습을 통해 결정되는 모델 내부 값
  • 하이퍼파라미터: 사람이 설정하는 학습 설정값 (학습률·깊이 등)
  • 손실 함수(Loss): 예측값과 실제값 차이를 측정
  • 경사 하강법: 손실 함수를 최소화하는 최적화 알고리즘

데이터 분할

  • 훈련 세트(Train): 모델 학습에 사용 (보통 60~80%)
  • 검증 세트(Validation): 하이퍼파라미터 튜닝 (10~20%)
  • 테스트 세트(Test): 최종 성능 평가 (10~20%)
  • k-겹 교차검증: k개로 나눠 k회 학습·검증 반복
  • 홀드아웃: 단순 훈련·테스트 2분할
ML 파이프라인: 비즈니스이해→데이터이해→데이터준비→모델링→평가→배포 (CRISP-DM 6단계)

지도학습 주요 알고리즘

선형 회귀 (Linear Regression) 회귀
▾
  • 원리: 입력 특성의 선형 결합으로 연속형 값 예측 — ŷ = w₁x₁ + w₂x₂ + ... + b
  • 손실 함수: MSE (평균 제곱 오차) — 예측값과 실제값 차이의 제곱 평균
  • 가정: 선형성·독립성·등분산성·정규성 (선형 관계 없으면 부적합)
  • 규제: Ridge(L2)·Lasso(L1)·ElasticNet으로 과적합 방지
  • Ridge vs Lasso: Ridge=모든 특성 유지·계수 축소 / Lasso=불필요 특성 계수를 0으로(특성 선택)
MSE = (1/n) Σ(yᵢ - ŷᵢ)² Ridge: MSE + λΣwⱼ² (L2 패널티) Lasso: MSE + λΣ|wⱼ| (L1 패널티 → 희소 모델)
로지스틱 회귀 (Logistic Regression) 분류
▾
  • 원리: 선형 결합의 결과를 시그모이드(σ) 함수로 0~1 확률로 변환 → 이진 분류
  • 결정 경계: P(y=1|x) ≥ 0.5 이면 클래스 1로 분류
  • 손실 함수: Binary Cross-Entropy (로그 손실)
  • 다중 분류: Softmax 함수 사용 (Multinomial Logistic Regression)
  • 특징: 확률 출력 / 선형 결정 경계 / 해석 용이 / 이상치에 강인
σ(z) = 1 / (1 + e-z) # 시그모이드 함수 Binary CE = -[y·log(ŷ) + (1-y)·log(1-ŷ)]
서포트 벡터 머신 (SVM) ⭐핵심
▾
  • 원리: 클래스 간 마진(Margin)을 최대화하는 결정 경계(초평면) 찾기
  • 서포트 벡터: 결정 경계에 가장 가까운 데이터 포인트 — 이것들이 경계를 결정
  • 커널 트릭(Kernel Trick): 선형 분리 불가 데이터를 고차원으로 변환하여 분리
  • 주요 커널: 선형(Linear)·다항식(Polynomial)·RBF(가우시안·가장 많이 사용)
  • C 하이퍼파라미터: 마진과 오분류 트레이드오프 / C 크면 과적합, 작으면 과소적합
  • 장점: 고차원 효과적 / 메모리 효율 / 강력한 일반화
  • 단점: 대용량 데이터에 느림 / 확률 출력 없음
# 마진 = 결정 경계로부터 서포트 벡터까지 거리의 2배 하드마진: 완벽한 분리 요구 (이상치 허용 안함) 소프트마진: C 파라미터로 오분류 허용 정도 조절
결정 트리 (Decision Tree) ⭐핵심
▾
  • 원리: 데이터를 특성 기준으로 트리 구조로 재귀 분할 — 조건 분기로 결과 예측
  • 분할 기준: 지니 불순도(Gini Impurity)·엔트로피(정보 이득)·분산 축소(회귀)
  • 지니 불순도: G = 1 - Σpᵢ² / 값이 0에 가까울수록 순수한 노드
  • 엔트로피: H = -Σpᵢ·log₂(pᵢ) / 정보 이득 = 분할 전 엔트로피 - 분할 후 엔트로피
  • 장점: 해석 용이(화이트박스) / 전처리 불필요 / 비선형 패턴 포착
  • 단점: 과적합 취약 (깊이 제한·가지치기로 대응) / 불안정(데이터 변화에 민감)
  • 가지치기(Pruning): 사전(Pre-pruning: max_depth·min_samples) / 사후(Post-pruning)
K-최근접 이웃 (KNN) 인스턴스 기반
▾
  • 원리: 새 데이터와 가장 가까운 K개 이웃의 클래스(다수결)·값(평균)으로 예측
  • 게으른 학습(Lazy Learning): 모델을 따로 구축하지 않고 전체 훈련 데이터를 저장
  • K 선택: K 너무 작으면 과적합 / K 너무 크면 과소적합 (홀수 권장 — 동점 방지)
  • 거리 척도: 유클리드·맨해튼·민코프스키 거리
  • 특성 스케일링 필수: 거리 기반이므로 표준화·정규화 필요
  • 단점: 예측 느림(O(n)) / 고차원에서 성능 저하(차원의 저주)
나이브 베이즈 (Naïve Bayes) 확률적 분류
▾
  • 원리: 베이즈 정리를 기반으로 클래스 사후 확률 계산 — 특성 간 조건부 독립 가정(Naïve)
  • 베이즈 정리: P(y|X) ∝ P(X|y)·P(y) — 사후확률 ∝ 우도 × 사전확률
  • 종류: 가우시안(연속 특성)·다항식(단어 빈도)·베르누이(이진 특성)
  • 장점: 빠른 학습·예측 / 소규모 데이터 효과적 / 텍스트 분류에 강력
  • 단점: 조건부 독립 가정이 현실에서 위배됨
P(y|X) = P(X|y)·P(y) / P(X) # 스팸 필터: P(스팸|단어들) ∝ P(단어들|스팸)·P(스팸)
지도학습: 선형회귀(MSE)·로지스틱(시그모이드·분류)·SVM(마진최대화·커널)·결정트리(지니·엔트로피)·KNN(K이웃)·나이브베이즈(베이즈정리)

비지도학습 — 군집화

K-Means 군집화 ⭐핵심
▾
  • 원리: 데이터를 K개 군집으로 나누어 각 군집의 중심(centroid)까지 거리 합 최소화
  • 알고리즘: ①K개 중심 초기화 → ②각 점을 가장 가까운 중심에 할당 → ③중심 재계산 → ④수렴까지 반복
  • K 선택 방법: 엘보우 방법(Elbow Method) — 이너셔(Within-cluster SSE) 감소율이 급격히 줄어드는 K 선택
  • 단점: K 사전 지정 필요 / 원형 군집만 / 이상치에 민감 / 초기값 의존
  • K-Means++ : 초기 중심을 더 분산되게 선택하여 수렴 개선
DBSCAN 밀도 기반 군집화
▾
  • 원리: 밀도가 높은 영역을 군집으로 정의 — ε(엡실론·반경)과 MinPts(최소 포인트 수) 기반
  • 포인트 분류: 핵심(Core)·경계(Border)·잡음(Noise·이상치)
  • 장점: K 사전 지정 불필요 / 임의 형태 군집 / 이상치 자동 탐지
  • 단점: 밀도 차이 큰 군집에 취약 / ε·MinPts 선택 어려움 / 고차원에서 성능 저하
계층적 군집화 (Hierarchical Clustering)
▾
  • 응집형(Agglomerative): 각 점을 개별 군집으로 시작 → 가장 가까운 군집끼리 병합 (Bottom-up)
  • 분리형(Divisive): 전체를 하나의 군집으로 시작 → 재귀적으로 분리 (Top-down)
  • 덴드로그램: 군집화 과정을 트리로 시각화 — 적절한 K 파악에 활용
  • 링키지: 단순(최소거리)·완전(최대거리)·평균·와드(분산 최소화)

ROC 곡선 · AUC · 기타 평가지표 ⭐

ROC 곡선 & AUC

  • ROC 곡선: 임계값 변화에 따른 TPR(재현율) vs FPR 그래프
  • AUC: ROC 곡선 아래 면적 — 1에 가까울수록 우수 / 0.5=무작위
  • FPR: FP/(FP+TN) = 1-특이도
  • 임계값 조정: 낮추면 재현율↑·정밀도↓ / 높이면 반대
  • 불균형 데이터: AUC보다 PR(정밀도-재현율) 곡선 권장

회귀 평가지표

  • MAE: 평균 절대 오차 — 이상치에 덜 민감
  • MSE: 평균 제곱 오차 — 큰 오차에 더 민감(제곱)
  • RMSE: MSE의 제곱근 — 원래 단위와 동일
  • R²(결정계수): 0~1 / 1에 가까울수록 모델이 분산 잘 설명 / 음수 가능(매우 나쁨)
  • MAPE: 평균 절대 백분율 오차 — 0에 가까울수록 좋음

의료=재현율(FN 최소화) / 스팸=정밀도(FP 최소화) / 불균형=F1+AUC / AUC=1에 가까울수록 좋음·0.5=무작위

편향-분산 트레이드오프 (Bias-Variance Tradeoff) ⭐

과적합 (Overfitting)

  • 훈련 데이터에 너무 잘 맞음 → 새 데이터(테스트)에서 성능 저하
  • 고분산(High Variance), 저편향
  • 복잡한 모델 (깊은 트리·많은 파라미터)
  • 대응: 정규화·드롭아웃·더 많은 데이터·교차검증

과소적합 (Underfitting)

  • 훈련·테스트 모두 성능 낮음 — 패턴을 충분히 학습 못함
  • 고편향(High Bias), 저분산
  • 지나치게 단순한 모델
  • 대응: 복잡도 증가·더 많은 특성·규제 감소
총 오류 = 편향² + 분산 + 불가피한 노이즈 # 편향 ↑ → 분산 ↓ (단순 모델) # 편향 ↓ → 분산 ↑ (복잡 모델) → 최적점 찾는 것이 핵심
편향↑=과소적합(단순모델) / 분산↑=과적합(복잡모델) / 총오류=편향²+분산+노이즈 → 최적 균형 찾기
🛡️

정규화 (Regularization) · 검증 방법

기법설명효과
L2 정규화 (Ridge)손실 함수에 가중치 제곱합(λΣw²) 패널티 추가모든 특성 유지 / 계수 축소 / 과적합 방지
L1 정규화 (Lasso)손실 함수에 가중치 절댓값 합(λΣ|w|) 패널티 추가일부 계수를 정확히 0으로 → 자동 특성 선택
ElasticNetL1 + L2 결합특성 선택 + 안정적 계수 / 다중공선성에 유리
k-겹 교차검증 (k-fold CV)데이터를 k개로 분할, k회 순환하며 학습·검증편향 없는 성능 추정 / k=5 또는 10 일반적
계층적 k-겹 (Stratified)각 폴드에서 클래스 비율 유지불균형 데이터에서 안정적 평가
조기 종료 (Early Stopping)검증 손실이 증가하기 시작하면 학습 중단과적합 방지 / 최적 에포크 자동 결정
Ridge=L2·계수축소·특성유지 / Lasso=L1·계수0화·특성선택 / k-fold=편향없는 성능추정 / 조기종료=검증손실 증가시 중단

앙상블 학습 (Ensemble Learning) ⭐

방법원리대표 알고리즘특징
배깅 (Bagging)부트스트랩 샘플로 여러 모델 병렬 학습 → 집계(평균·다수결)랜덤 포레스트 ⭐분산 감소 / 과적합 방지 / 병렬 학습
부스팅 (Boosting)이전 모델이 틀린 샘플에 가중치를 높여 순차적으로 학습XGBoost·LightGBM·AdaBoost편향 감소 / 높은 성능 / 과적합 주의
스태킹 (Stacking)여러 기본 모델(Level-0)의 예측을 메타 모델(Level-1)의 입력으로 사용Blending복잡도↑ / 다양한 모델 결합
보팅 (Voting)여러 모델의 예측을 다수결(Hard) 또는 확률 평균(Soft)으로 결합VotingClassifier구현 단순 / 다양한 모델 혼합
랜덤 포레스트 (Random Forest) — 배깅의 대표 ⭐최중요
▾
  • 원리: 부트스트랩 샘플 + 무작위 특성 부분집합으로 다수의 결정 트리를 학습 → 앙상블
  • 두 가지 무작위성: ①부트스트랩 샘플링 ②각 분할 시 무작위 특성 부분집합 사용
  • OOB(Out-of-Bag): 부트스트랩에 포함되지 않은 샘플로 자체 검증 가능
  • 특성 중요도: 각 특성이 불순도 감소에 기여한 정도로 특성 중요도 산출
  • 장점: 과적합 방지 / 높은 정확도 / 결측값 처리 / 특성 중요도 제공
XGBoost · LightGBM — 부스팅의 대표 ⭐최중요
▾
  • Gradient Boosting: 이전 모델의 잔차(Residual)를 예측하는 새 트리를 순차적으로 추가
  • XGBoost: eXtreme Gradient Boosting / L1·L2 정규화 내장 / 병렬 처리 / 결측값 자체 처리
  • LightGBM: 리프 중심 트리 성장(Leaf-wise, XGB는 깊이 우선) → 더 빠름·더 정확
  • CatBoost: 범주형 특성 자동 처리 / 순서 부스팅
  • 주요 하이퍼파라미터: n_estimators(트리 수)·learning_rate·max_depth·subsample·colsample_bytree
배깅=병렬·분산감소(랜덤포레스트) / 부스팅=순차·편향감소(XGBoost·LightGBM) / 스태킹=메타모델 결합

특성 공학 (Feature Engineering)

특성 전처리

  • 표준화(Standardization): (x-μ)/σ → 평균 0·표준편차 1 (Z-score)
  • 정규화(Normalization·MinMax): (x-min)/(max-min) → 0~1 범위
  • 로그 변환: 왜도 있는 분포 → 정규화
  • 결측값 처리: 평균·중위수·최빈값 대체 / KNN Imputer
  • 이상치 처리: IQR·Z-score로 탐지 후 제거·변환

범주형 특성 인코딩

  • 원-핫 인코딩: 각 범주를 이진 열로 변환 — 순서 없는 명목 변수
  • 레이블 인코딩: 범주를 정수로 변환 — 순서 있는 변수
  • 타깃 인코딩: 범주별 타깃 평균으로 대체 — 고차원 범주에 효과적
  • 특성 선택: 분산 임계값·상관계수·RFE(재귀적 특성 제거)·특성 중요도
  • 특성 생성: 다항 특성·날짜 분해·상호작용 특성
