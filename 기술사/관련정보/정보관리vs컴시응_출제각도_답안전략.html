핵심 결론: 동일 토픽이라도 정보관리는 "왜·어떻게 관리할 것인가"를, 컴시응은 "어떻게 구현·동작하는가"를 묻는다. 두 시험을 동시 준비한다면 같은 토픽을 두 개의 렌즈로 바라보는 훈련이 필요하다.

답안 구조 비교 — 같은 토픽, 다른 뼈대

정보관리 답안 구조
권장 4단 구성
I. 개요 — 정의 + 등장 배경 (경영 환경 변화)
II. 핵심 개념·원리 — 기술 설명 (관리 관점 강조)
III. 도입·추진 전략 — 단계별 로드맵, 거버넌스, 위험 관리
IV. 결론 — 기대효과(ROI), 추진 과제, 발전 방향

COBIT·ITIL·PMBOK 등 표준 프레임워크 인용이 가산점
컴시응 답안 구조
권장 4단 구성
I. 개요 — 정의 + 기술적 등장 배경 (한계·병목)
II. 구조·동작 원리 — 아키텍처, 알고리즘, 프로토콜 상세
III. 성능·트레이드오프 분석 — 복잡도, 비교표, 최적화 방법
IV. 결론 — 설계 선택 기준, 기술 발전 방향
결론에 "성능 트레이드오프"와 "적합 사용 시나리오" 명시
그림은 아키텍처·시퀀스·메모리 레이아웃 중심
복잡도 O() 표기, 수식, 의사코드 포함이 가산점

정보관리에서 감점 패턴

📌 사례 1 — 클라우드 컴퓨팅

정보관리 출제 방식
기업의 클라우드 전환(Cloud Migration) 전략을 수립하고자 한다. 고려해야 할 사항과 추진 단계를 설명하시오.
핵심: 왜 전환하는가 + 어떤 절차로 관리할 것인가
① 현황 분석(As-Is) → ② CSP 선정 기준(비용·SLA·보안) → ③ 마이그레이션 전략(6R) → ④ 거버넌스·운영 체계 → ⑤ 위험 관리·벤더 종속성 해소
컴시응 출제 방식
컨테이너 오케스트레이션 플랫폼(Kubernetes)의 아키텍처와 핵심 구성 요소의 동작 원리를 설명하시오.
핵심: 내부적으로 어떻게 동작하는가
① Control Plane 구조(kube-apiserver·etcd·scheduler·controller-manager) → ② Data Plane(kubelet·kube-proxy·CNI) → ③ Pod 스케줄링 알고리즘 → ④ 서비스 디스커버리 메커니즘
두 시험 모두 준비 중이라면: "클라우드" 토픽을 관리 시나리오 답안과 기술 구현 답안을 별도로 작성해 두어야 한다.

📌 사례 2 — 데이터베이스

정보관리
기업의 데이터 거버넌스(Data Governance) 체계 구축 방안을 설명하시오.
핵심: 데이터를 어떻게 관리·통제할 것인가
① 데이터 거버넌스 프레임워크(DAMA) → ② 데이터 품질 관리(DQM) 체계 → ③ MDM 구축 전략 → ④ 데이터 오너십·스튜어드십 조직 → ⑤ 개인정보보호법 준수
컴시응
DBMS의 트랜잭션 동시성 제어 기법을 비교하고 MVCC의 동작 원리를 설명하시오.
핵심: 내부적으로 어떻게 구현되는가
① 2PL vs MVCC 비교 → ② MVCC 버전 체인 구조(Undo 체인) → ③ Snapshot Isolation 동작 → ④ Write Skew 이상 현상 → ⑤ PostgreSQL·InnoDB 구현 방식

📌 사례 3 — 정보보안 (가장 차이가 큰 영역)

정보관리
조직의 정보보호 관리체계(ISMS-P) 인증 취득을 위한 추진 방안과 유지 관리 방법을 설명하시오.
핵심: 어떻게 인증을 획득·운영할 것인가
① ISMS-P 인증 범위 설정 → ② 102개 통제 항목 갭 분석 → ③ 취약점 개선 로드맵 → ④ 내부 심사·인증 심사 절차 → ⑤ 사후 관리 체계·개인정보처리방침
컴시응
AES-256 암호화 알고리즘의 구조와 운용 모드(CBC, GCM)를 비교하고 보안 특성을 설명하시오.
핵심: 알고리즘이 어떻게 구현되는가
① AES 라운드 구조(SubBytes·ShiftRows·MixColumns·AddRoundKey) → ② CBC vs GCM 비교 → ③ GCM의 인증 태그 생성(GHASH) → ④ 패딩 오라클 공격 취약점

📌 사례 4 — 소프트웨어공학 (가장 공통 범위 넓은 영역)

정보관리
공공기관의 SW 개발 사업에서 PMO(Project Management Office) 운영 방안을 설명하시오.
핵심: 프로젝트를 어떻게 관리·감독할 것인가
① PMO 역할·기능(계획·통제·지원) → ② SW 사업 대가 기준 적용 → ③ 산출물 관리·검수 절차 → ④ 하도급 적정성 관리 → ⑤ 감리와 PMO의 역할 분담
컴시응
소프트웨어 아키텍처 패턴 중 이벤트 기반 아키텍처(EDA)와 CQRS 패턴을 설명하고 적용 기준을 논하시오.
핵심: 어떻게 설계·구현할 것인가
① EDA 구성(Producer·Broker·Consumer) → ② CQRS Command/Query 분리 원리 → ③ Event Sourcing과의 결합 → ④ Eventual Consistency 처리 → ⑤ 적합/비적합 시나리오

📌 사례 5 — AI (신기술 영역, 두 시험 모두 빠르게 비중 증가)

정보관리
생성형 AI 도입 시 기업이 수립해야 할 AI 거버넌스 체계를 설명하시오.
핵심: AI를 어떻게 책임 있게 관리할 것인가
① AI 거버넌스 프레임워크(EU AI Act 대응) → ② 편향·공정성 감사 체계 → ③ 개인정보 처리 방침 → ④ AI 윤리위원회 구성 → ⑤ AI 위험 등급 분류 및 통제
컴시응
대규모 언어 모델(LLM)의 Transformer 아키텍처와 Self-Attention 메커니즘의 동작 원리를 설명하시오.
핵심: 내부적으로 어떻게 동작하는가
① Self-Attention Q·K·V 행렬 계산 → ② Multi-Head Attention 구조 → ③ Positional Encoding → ④ 시간 복잡도 O(n²) 한계 → ⑤ Flash Attention 최적화
두 시험 동시 준비 전략: 같은 토픽을 한 번 공부할 때 "관리 답안"과 "기술 답안" 두 버전의 서브노트를 만들면 학습 효율이 2배가 된다.
