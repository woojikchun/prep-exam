Ⅰ. 연관 규칙의 개념 및 핵심 평가 척도
1. 연관 규칙(Association Rule)의 정의

연관 규칙 마이닝(Association Rule Mining)이란 대용량 트랜잭션 데이터에서 항목들 간의 흥미로운 공동 출현(Co-occurrence) 패턴을 자동으로 발견하는 데이터 마이닝 기법이다. 1993년 Agrawal 등이 장바구니 분석(Market Basket Analysis)에 처음 적용하였다.

연관 규칙은 X → Y 형태로 표현되며, "항목 집합 X를 구매한 고객은 항목 집합 Y도 구매하는 경향이 있다"를 의미한다. X를 전건(Antecedent), Y를 후건(Consequent)이라 한다.

3. 빈발 항목집합(Frequent Itemset) 문제의 난이도

n개의 아이템에서 가능한 항목집합 수는 2ⁿ − 1개이다. 아이템이 1,000개면 후보 집합이 2¹⁰⁰⁰개에 달해 완전 탐색(Brute Force)이 불가능하며, 효율적 알고리즘이 필수적이다.

Ⅱ. Apriori 알고리즘의 동작 원리, 단조성 성질 및 예제
1. Apriori 핵심 원리 — 반단조성(Anti-monotone) 성질

Apriori 성질(Anti-monotone Property): 빈발 항목집합의 모든 부분집합은 반드시 빈발하고, 비빈발 항목집합의 모든 상위집합(Superset)도 반드시 비빈발하다.

sup(X) < min_sup  ⟹  ∀Y ⊇ X : sup(Y) < min_sup

이 성질로 비빈발 집합의 모든 상위집합을 가지치기(Pruning)하여 후보 집합을 대폭 감소시킨다.

2. Apriori 알고리즘 동작 단계

① 후보 생성(Candidate Generation, Cₖ): 이전 단계의 빈발 k-1 항목집합(Lₖ₋₁)에서 k-항목집합 후보를 생성
② 지지도 계산 — DB 스캔: 후보 집합 Cₖ의 지지도를 전체 트랜잭션 DB 스캔으로 계산
③ 가지치기(Pruning): min_sup 미만 집합과 부분집합에 비빈발 포함 집합 제거 → 빈발 집합 Lₖ 확정
④ Lₖ가 공집합이 될 때까지 ①~③ 반복

# Apriori 알고리즘 수도코드
L₁ ← {빈발 1-항목집합}  # DB 1회 스캔
for k = 2, 3, ... while Lₖ₋₁ ≠ ∅:
    Cₖ ← apriori_gen(Lₖ₋₁)   # ① Join: Lₖ₋₁ ⊕ Lₖ₋₁ → k-항목집합 후보
    for each 트랜잭션 t ∈ D:  # ② DB 스캔 (k번째 반복마다 전체 스캔)
        Cₜ ← subset(Cₖ, t)
        for each c ∈ Cₜ: c.count++
    prune: 부분집합에 비빈발 포함 후보 제거  # ③ Anti-monotone 가지치기
    Lₖ ← {c ∈ Cₖ | c.count / |D| ≥ min_sup}
return ∪ₖ Lₖ

4. Apriori의 한계

① 반복적 DB 스캔: 최대 k-항목집합 크기만큼 전체 DB를 반복 스캔 → I/O 비용 막대
② 방대한 후보 집합: 빈발 아이템이 많으면 후보 집합의 수가 폭발적 증가
③ 메모리 오버헤드: 후보 집합 전체를 메모리에 유지해야 함

Ⅲ. FP-Growth 알고리즘의 동작 원리 — FP-Tree 구축 및 패턴 탐사
1. FP-Growth 핵심 아이디어

Han et al.(2000)이 제안한 FP-Growth(Frequent Pattern Growth)는 Apriori의 반복 DB 스캔과 후보 생성을 완전히 제거한다. 전체 DB를 단 2회만 스캔하여 FP-Tree(Frequent Pattern Tree)를 구축하고, 이 압축된 자료구조에서 조건 패턴 베이스(Conditional Pattern Base)를 반복 생성하여 빈발 패턴을 탐사한다.

2. FP-Tree 구축 절차 (2회 DB 스캔)
스캔작업결과
1차 스캔 각 아이템의 지지도 계산 → min_sup 이상인 아이템만 추출 빈발 1-항목 목록, 지지도 내림차순 정렬
2차 스캔 각 트랜잭션을 빈발 아이템만 남기고 지지도 내림차순 재정렬 → FP-Tree에 삽입 FP-Tree 완성 + 헤더 테이블(Header Table)

4. 조건 패턴 베이스 및 조건 FP-Tree (패턴 탐사)

FP-Tree 구축 후 헤더 테이블을 지지도 오름차순(하위 → 상위)으로 순회하며 각 아이템의 조건 패턴 베이스(Conditional Pattern Base)를 추출하고, 이를 바탕으로 조건 FP-Tree를 재귀 생성하여 빈발 패턴을 탐사한다.

아이템조건 패턴 베이스 (접두사 경로)조건 FP-Tree생성 빈발 패턴
D {A,B}:1, {B}:1 (D의 접두사) {B}:2 만 min_sup 충족 {D,B}:2, {D}:2
C {A,B}:1, {A}:1, {B}:1 (C의 접두사) {A}:2, {B}:2 둘 다 충족 {C,A}:2, {C,B}:2, {C,A,B}:2(재귀), {C}:3
B {A}:2 (B를 포함하는 경로의 접두사) {A}:2 min_sup 충족 {B,A}:2, {B}:3
A ∅ (root 자식이므로 접두사 없음) 공 FP-Tree {A}:3

5. FP-Growth 수도코드
def fp_growth(FP_Tree, α):
    if FP_Tree 에 단일 경로 P:                  # 단일 경로 → 조합만 생성
        for each 경로 노드 집합 β ⊆ P:
            yield α ∪ β,  min(β.count)
    else:
        for each aᵢ ∈ header_table (하위 지지도 순):
            β = α ∪ {aᵢ}
            yield β, aᵢ.count          # 빈발 패턴 출력
            CPB ← conditional_pattern_base(aᵢ, FP_Tree)
            T_β ← build_fp_tree(CPB, min_sup) # 조건 FP-Tree 구축
            if T_β ≠ ∅:
                fp_growth(T_β, β)         # 재귀 탐사

2. 실무 적용 시 고려사항

① min_sup·min_conf 임계값 설정: 너무 낮으면 규칙 수 폭발·품질 저하, 너무 높으면 유의미한 규칙 누락. 도메인 지식 기반 반복 실험으로 조정 필요

② 향상도(Lift) 활용: 신뢰도가 높아도 Lift ≤ 1이면 Y가 단독으로 발생 확률이 높은 것이므로 무의미한 규칙. 반드시 Lift > 1을 함께 검증해야 한다

③ 데이터 전처리: 아이템 수가 매우 많을 경우 도메인 기반 사전 집약(카테고리 통합), 거래 빈도가 낮은 롱테일 아이템 제거로 후보 폭발 억제

④ 대용량 확장: Hadoop/Spark 기반 분산 처리 — Apriori는 파티션 방식(SON Algorithm), FP-Growth는 PFP-Growth(Parallel FP-Growth)로 확장

⑤ 연관 규칙의 활용: 장바구니 분석·추천 시스템·이상 탐지(보안)·의료 진단 연관 증상 발굴 등. 규칙의 통계적 유의미성뿐 아니라 비즈니스 해석 가능성을 함께 검증해야 한다

Ⅴ. 결론

Apriori는 반단조성 성질을 이용한 후보 가지치기로 탐색 공간을 축소하는 직관적 방식이며, FP-Growth는 FP-Tree 압축 구조와 조건 패턴 베이스 재귀 탐사로 후보 생성과 반복 DB 스캔을 완전히 제거한 진화된 방식이다. 두 알고리즘은 연관 규칙 마이닝의 이론적 토대를 공유하며, 실무에서는 데이터 밀도·규모·환경에 따라 적합한 방식을 선택하고 향상도(Lift) 검증과 도메인 해석을 통해 의미 있는 패턴을 추출해야 한다.

★ 기술사 핵심 코멘트
연관 규칙 마이닝의 본질은 지지도·신뢰도·향상도 세 척도의 균형 잡힌 해석이며, 시스템·알고리즘 설계 감리 시 다음을 중점 검토해야 한다.
① 임계값 설정 근거: min_sup·min_conf가 도메인 지식이나 실험적 근거 없이 임의 설정된 경우 결과 신뢰도 훼손 — 설정 근거 문서화 여부 확인
② 향상도(Lift) 검증 누락: 신뢰도만으로 규칙을 채택하면 자주 발생하는 아이템 쌍이 무의미한 규칙으로 남용될 수 있음 — Lift > 1 필터링 적용 여부 점검
③ FP-Growth 메모리 관리: 밀집 대용량 DB에서 FP-Tree가 메인 메모리를 초과할 경우 DB 분할(Partitioned FP-Growth) 또는 스트리밍 방식 채택 여부
④ 분산 처리 정확성: PFP-Growth 등 분산 처리 시 파티션 간 경계 트랜잭션 처리에 의한 지지도 오차 발생 가능 — 샘플링 검증 수행 여부 확인
