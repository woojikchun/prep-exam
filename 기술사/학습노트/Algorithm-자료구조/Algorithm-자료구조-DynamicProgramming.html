1. 정의

동적 프로그래밍(Dynamic Programming, DP)은 복잡한 문제를 더 작은 부분 문제(Subproblem)로 분해하고, 각 부분 문제의 해를 단 한 번 계산하여 저장(Cache)한 후 재사용함으로써 중복 연산을 제거하는 알고리즘 설계 패러다임이다. 1950년대 Richard Bellman이 최적 제어 이론에서 고안하였으며, 이름의 "Dynamic"은 수학적 의미보다 연구 예산 확보를 위한 마케팅 용어로 사용되었다.

2. DP 적용을 위한 두 가지 필수 조건

1. 메모이제이션 (Top-Down DP)

메모이제이션(Memoization)은 재귀 함수로 문제를 하향식(Top-Down)으로 분해하면서, 이미 계산된 결과를 해시맵(HashMap) 또는 배열에 저장해 두고 동일 입력이 재호출될 때 즉시 반환하는 방식이다. 자연스러운 재귀 구조를 유지하므로 구현이 직관적이며, 실제로 필요한 부분 문제만 계산하는 지연 계산(Lazy Evaluation) 특성을 갖는다.

▶ 피보나치 수열 — 메모이제이션 (Python)
def fib_memo(n, memo={}):
    if n in memo:
        return memo[n]          # ① 캐시 적중 → 즉시 반환
    if n <= 1:
        return n
    memo[n] = fib_memo(n-1, memo) + fib_memo(n-2, memo)  # ② 계산 후 저장
    return memo[n]
# 시간: O(n)  공간: O(n) — 재귀 스택 + memo 딕셔너리
2. 타뷸레이션 (Bottom-Up DP)

타뷸레이션(Tabulation)은 가장 작은 부분 문제부터 상향식(Bottom-Up)으로 순차 계산하여 테이블(배열)을 채워나가는 방식이다. 재귀 호출이 없으므로 스택 오버플로우 위험이 없고, 메모리 접근 패턴이 순차적이어서 캐시 효율(Cache Locality)이 높다. 실무에서 성능이 중요한 경우 타뷸레이션이 선호된다.

▶ 피보나치 수열 — 타뷸레이션 (Python)
def fib_tab(n):
    if n <= 1: return n
    dp = [0] * (n + 1)
    dp[1] = 1
    for i in range(2, n + 1):   # ① 작은 문제부터 순차 계산
        dp[i] = dp[i-1] + dp[i-2] # ② 이전 결과 참조
    return dp[n]
# 시간: O(n)  공간: O(n) → 변수 2개로 줄이면 O(1)

Ⅲ. 대표 알고리즘 — LCS와 0/1 배낭 문제
1. 최장 공통 부분 수열 (LCS, Longest Common Subsequence)
가. 문제 정의

두 수열 X = x₁x₂…xₘ 과 Y = y₁y₂…yₙ 이 주어질 때, 두 수열에 모두 포함되는 부분 수열(연속 불필요) 중 최장 길이를 구하는 문제. Git diff, 생물정보학(DNA 비교), 표절 탐지 등에 활용된다.

나. 점화식 (Recurrence Relation)
dp[i][j] = dp[i-1][j-1] + 1             if X[i] == Y[j]
dp[i][j] = max(dp[i-1][j], dp[i][j-1])   if X[i] ≠ Y[j]

시간 복잡도: O(m × n)  |  공간 복잡도: O(m × n) → 슬라이딩으로 O(min(m,n)) 가능

다. DP 테이블 예시 — X = "ABCBDAB", Y = "BDCABA"

【표 4】 LCS DP 테이블 — 초록(대각선 이동·일치), 파랑(참조), 주황(최종해 LCS=4 "BCBA")
▶ LCS 타뷸레이션 (Python)
def lcs(X, Y):
    m, n = len(X), len(Y)
    dp = [[0] * (n+1) for _ in range(m+1)]
    for i in range(1, m+1):
        for j in range(1, n+1):
            if X[i-1] == Y[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1  # 문자 일치: 대각선 + 1
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])  # 불일치: 인접 최대
    return dp[m][n]   # → 4 (BCBA)
2. 0/1 배낭 문제 (0/1 Knapsack Problem)
가. 문제 정의

무게 한도 W인 배낭과 n개의 물건이 주어질 때(각 물건: 무게 wᵢ, 가치 vᵢ), 배낭에 담을 수 있는 물건의 가치 합을 최대화하는 문제. 각 물건은 넣거나(1) 넣지 않거나(0)만 선택 가능하다. 자원 배분 최적화, 포트폴리오 선택, 작업 스케줄링에 광범위하게 활용된다.

나. 점화식
dp[i][w] = dp[i-1][w]                             if w < wᵢ (물건 i를 넣을 수 없음)
dp[i][w] = max(dp[i-1][w], dp[i-1][w-wᵢ] + vᵢ)   if w ≥ wᵢ (넣을지 말지 선택)

시간 복잡도: O(n × W)  |  공간 복잡도: O(n × W) → 1D 배열로 O(W) 가능 (의사다항식)

다. DP 테이블 예시 — 물건: (2kg·3만원), (3kg·4만원), (4kg·5만원), (5kg·6만원) / W=8
물건\무게 012 345 678

【표 5】 0/1 Knapsack DP 테이블 — dp[4][8] = 10만원 (①+④ 선택, 7kg)
▶ 0/1 Knapsack 타뷸레이션 — 1D 공간 최적화 (Python)
def knapsack(W, weights, values):
    n = len(weights)
    dp = [0] * (W + 1)             # 1D 배열로 O(W) 공간만 사용
    for i in range(n):
        for w in range(W, weights[i]-1, -1):  # ★ 역순 순회 — 동일 물건 중복 방지
            dp[w] = max(dp[w], dp[w - weights[i]] + values[i])
    return dp[W]
# weights=[2,3,4,5], values=[3,4,5,6], W=8 → 10

3. 실무 적용 시 고려사항

① 상태(State) 정의의 명확성: DP의 성패는 상태를 어떻게 정의하느냐에 달려 있다. 상태는 문제 풀이에 필요한 최소 정보만 포함하도록 설계해야 중복·낭비를 막을 수 있다.

② 메모리 한계 인식: 상태 수가 기하급수적으로 증가하는 경우(예: 2D 이상 DP) 메모리 오버플로우 위험. 슬라이딩 윈도우·비트마스크 등 공간 최적화 기법을 선제적으로 검토해야 한다.

③ 의사다항식 복잡도 주의(Pseudo-Polynomial): 0/1 Knapsack의 O(n×W)는 W가 숫자 크기에 의존하므로 W가 매우 클 때 실용적이지 않을 수 있다 (NP 문제와의 관계 인식 필요).

④ 역추적(Traceback) 설계: 최적값뿐 아니라 최적 선택 경로 복원이 필요한 경우, 추가 역추적 포인터 테이블을 별도 유지해야 한다. (LCS에서 실제 수열 복원, Knapsack에서 선택 물건 목록 출력 등)

⑤ 병렬화 가능성 검토: 의존성 없는 대각선 방향 셀은 병렬 처리 가능. GPU·SIMD 활용 시 대형 DP 테이블 계산 속도 향상 가능.

Ⅴ. 결론

동적 프로그래밍은 중복 부분 문제의 재계산을 캐싱으로 제거함으로써, 지수 시간 알고리즘을 다항 시간으로 극적으로 개선하는 핵심 설계 패러다임이다. LCS·Knapsack 등 고전적 문제부터 최신 AI의 강화학습(Bellman Equation 기반 Q-Learning)에 이르기까지 광범위하게 적용된다. 실무에서는 올바른 상태 정의와 최적화 기법 선택이 DP의 실질적 가치를 결정하며, 단순 구현보다 메모리·시간 제약을 사전에 분석하는 설계 역량이 더 중요하다.

★ 기술사 핵심 코멘트
DP의 본질은 최적 부분 구조 + 중복 부분 문제라는 두 조건을 충족할 때만 적용 가능하며, 시스템 설계·감리 시 다음을 중점 검토해야 한다.
① 알고리즘 선택 적정성: 문제가 DP 적용 조건을 충족하는지 검증 — 최적 부분 구조 성립 여부 분석 필수
② 복잡도 실용성 검토: 의사다항식 복잡도(Knapsack 등)는 입력 크기(W)에 따른 실제 수행 시간 벤치마크 필요
③ 메모리 설계: 대용량 DP 테이블의 메모리 상한(Memory Bound) 분석 및 슬라이딩 윈도우·압축 기법 적용 여부 확인
④ 강화학습 연계: 현대 AI의 Q-Learning·Value Iteration은 Bellman 방정식 기반 DP의 확장으로, AI 시스템 감리 시 DP 이론 이해가 기반 역량이 됨
