Transformer Self-Attention 메커니즘
⏱ 목표 12분 📄 1.5페이지 💯 10점
Ⅰ. 개요 — Transformer와 Self-Attention의 등장 배경
Transformer는 2017년 Google이 제안한 "Attention is All You Need" 논문에서 소개된 신경망 아키텍처로, RNN의 순차 처리 한계(장기 의존성 소실, 병렬화 불가)를 극복하기 위해 Self-Attention 메커니즘을 전면 채택하였다. Self-Attention은 문장 내 모든 토큰이 서로의 관계를 동시에 계산하여 문맥을 포착하는 핵심 연산이다.
Ⅱ. Self-Attention 동작 원리 — Q·K·V 연산

핵심 개념: Query · Key · Value

• Q(Query): "내가 어떤 정보를 찾고 있는가" → 현재 토큰의 질의 벡터
• K(Key): "나는 어떤 정보를 가지고 있는가" → 각 토큰의 식별 벡터
• V(Value): "실제로 전달할 정보는 무엇인가" → 각 토큰의 내용 벡터
Scaled Dot-Product Attention 입력 X → 선형 변환(WQ, WK, WV) → Q = X·WQ, K = X·WK, V = X·WV Attention(Q,K,V) = softmax( Q·Kᵀ / √dk ) · V ① Q·Kᵀ : 모든 토큰 쌍 간 유사도 점수(Attention Score) 계산 ② / √dk : 차원 수(dk)의 제곱근으로 스케일링 → 기울기 소실 방지 ③ softmax : 확률 분포로 정규화 (Attention Weight) ④ · V : 가중합으로 문맥 반영된 출력 생성
Multi-Head Attention: h개의 서로 다른 선형 변환으로 Q·K·V를 분리하여 병렬 계산한 후 Concat. 각 헤드가 다양한 관점(구문적 관계, 의미적 관계 등)을 학습할 수 있다.
MultiHead(Q,K,V) = Concat(head₁,...,headₕ) · WO where headᵢ = Attention(Q·WiQ, K·WiK, V·WiV)
Ⅲ. Positional Encoding 및 특징
구성 요소역할특징
Positional Encoding위치 정보 주입 (sin/cos 함수)순서 정보가 없는 Self-Attention 보완
Masked Attention디코더에서 미래 토큰 가림자기회귀(AR) 생성 시 정보 유출 방지
Cross-Attention인코더 출력을 디코더가 참조번역·요약 등 Seq2Seq 태스크에 적용

💡 한 줄 암기 포인트

Self-Attention = "Q(질문)으로 K(키)를 검색하여 V(값)를 가중합"
계산 복잡도: O(n²·d) — 시퀀스 길이 n의 제곱에 비례 → 긴 문장 처리 한계(Linear Attention 등 개선 연구 진행)
Ⅳ. 결론 및 발전 방향

Self-Attention 기반 Transformer는 BERT·GPT·T5·LLaMA 등 현대 LLM의 공통 기반 아키텍처이다. 긴 시퀀스 처리 효율화를 위한 Flash Attention, Sparse Attention, Mamba(SSM) 등의 대안 아키텍처가 연구되고 있으며, 멀티모달(이미지·음성·동영상) 처리에도 Transformer가 확장 적용되고 있다.

LLM 파인튜닝 기법 LoRA(Low-Rank Adaptation)
⏱ 목표 12분 📄 1.5페이지 💯 10점
Ⅰ. 개요 — PEFT와 LoRA 등장 배경
LLM의 Full Fine-tuning은 수십~수백억 개의 파라미터를 모두 업데이트하므로 막대한 GPU 메모리와 연산 비용이 필요하다. PEFT(Parameter-Efficient Fine-Tuning)는 전체 파라미터 중 일부만 학습하여 비용을 절감하는 기법 범주이며, LoRA는 2021년 Microsoft가 제안한 대표적 PEFT 기법이다.
Ⅱ. LoRA 동작 원리 — 저랭크 행렬 분해

핵심 아이디어

사전 학습된 가중치 행렬 W₀(d×k)를 동결(Freeze)하고, 학습 가능한 두 저랭크 행렬 B(d×r)·A(r×k)의 곱으로 업데이트를 근사한다. (r ≪ min(d,k))
LoRA 수식 W₀ : 사전학습 가중치 (Frozen, 학습 안 함) ΔW : 업데이트 행렬 순전파: h = W₀x + ΔWx = W₀x + BAx A : r×k 행렬 (가우시안 초기화) B : d×r 행렬 (0으로 초기화 → 초기 ΔW=0 보장) 학습 파라미터 수: d×r + r×k (r=8 이면 Full 대비 ~0.1%) 추론 시 병합: W = W₀ + BA (지연 없음)
비교 항목Full Fine-tuningLoRA
학습 파라미터전체 (수십~수백억)r 랭크 행렬만 (~0.1~1%)
GPU 메모리매우 큼 (80GB+ 다수)소비 1/4 수준
학습 시간수일~수주수시간~1일
성능최고Full 대비 95% 이상 달성 가능
멀티태스크재학습 필요태스크별 LoRA 모듈 교체
Ⅲ. 변형 기법 및 활용
QLoRA (4bit 양자화+LoRA) LoRA+ (학습률 차등 적용) AdaLoRA (랭크 적응적 조정) DoRA (가중치 분해 학습)
QLoRA는 LLM을 4-bit NF4로 양자화한 상태에서 LoRA를 적용하여 단일 24GB GPU로 65B 모델 파인튜닝을 가능하게 한 획기적 기법이다.

💡 한 줄 암기 포인트

LoRA = "원본 가중치는 얼리고(Freeze), 저랭크 행렬 BA만 학습"
적용 위치: Transformer의 Attention 레이어(Wq, Wk, Wv, Wo)에 주로 적용
Ⅳ. 결론

LoRA는 LLM의 도메인 특화(법률·의료·금융·공공) 파인튜닝을 소규모 인프라로 가능하게 하여 AI 민주화에 기여하고 있다. Hugging Face PEFT 라이브러리로 표준화되어 있으며, 공공기관의 온프레미스 초거대 AI 도입 시 필수적인 기법으로 활용된다.

프롬프트 엔지니어링(Prompt Engineering)의 주요 기법을 설명하시오.
Zero/Few-Shot Chain-of-Thought ReAct 프롬프트 인젝션 보안
프롬프트 엔지니어링의 주요 기법
⏱ 목표 12분 📄 1.5~2페이지 💯 10점
Ⅰ. 개요 — 프롬프트 엔지니어링의 개념
프롬프트 엔지니어링이란 LLM이 원하는 출력을 생성하도록 입력(프롬프트)을 체계적으로 설계·최적화하는 기술 분야이다. 모델 가중치를 변경하지 않고 In-Context Learning만으로 성능을 향상시키므로 비용 효율이 높다.
Ⅱ. 주요 프롬프트 기법
기법원리특징·적용
Zero-Shot예시 없이 태스크만 지시사전학습 지식 활용. 간단하고 범용적
Few-Shot입력-출력 예시 N개 제공 후 질의예시 품질·순서가 성능에 큰 영향. 분류·추출에 효과적
Chain-of-Thought (CoT)"단계적으로 생각해보자" 지시. 중간 추론 단계 명시수학 문제·논리 추론 성능 대폭 향상. Zero-Shot CoT 가능
Self-Consistency동일 질문을 여러 번 샘플링 후 다수결CoT와 결합. 정답 신뢰도 향상
Tree of Thoughts (ToT)여러 사고 경로를 트리로 탐색·평가복잡한 계획 문제에 유효. BFS·DFS 적용
ReActReasoning + Acting. 추론과 외부 도구 호출을 교차 수행웹 검색·코드 실행 등 Tool Use와 결합. 에이전트 기반
Least-to-Most복잡한 문제를 하위 문제로 분해 후 순차 해결수학·코딩 문제 분해에 효과적
Ⅲ. 프롬프트 보안 위협 — Prompt Injection

프롬프트 인젝션 (Prompt Injection)

악의적 입력을 통해 시스템 프롬프트의 지시를 무력화하고 모델을 조종하는 공격.
• Direct Injection: 사용자가 직접 악성 지시 삽입
• Indirect Injection: 웹 페이지·문서 내 숨겨진 지시를 모델이 읽어 실행
대책: 입력 필터링, Instruction Hierarchy, 샌드박싱, 출력 검증

💡 한 줄 암기 포인트

CoT = "단계별 추론 명시화" / ReAct = "추론 + 도구 호출 교차" / ToT = "사고 트리 탐색"
성능 향상 순서 (단순→복잡): Zero-Shot → Few-Shot → CoT → Self-Consistency → ToT
Ⅳ. 결론

프롬프트 엔지니어링은 LLM을 재학습 없이 고도화하는 핵심 역량이다. 최근 Agentic AI 시스템에서는 ReAct·ToT 기반 멀티스텝 추론이 중요해지고 있으며, 보안 관점에서 프롬프트 인젝션 방어는 AI 서비스 신뢰성 확보의 필수 요건이다.

AI 할루시네이션(Hallucination)의 유형과 원인 및 완화 방안을 설명하시오.
Factual / Faithfulness 분류 RAG 연계 RLHF 공공 AI 신뢰성
AI 할루시네이션의 유형·원인 및 완화 방안
⏱ 목표 12분 📄 1.5~2페이지 💯 10점
Ⅰ. 개요 — AI 할루시네이션의 정의
AI 할루시네이션(환각)이란 LLM이 사실이 아닌 정보를 마치 사실인 것처럼 자신감 있게 생성하는 현상이다. 단순한 오류와 달리 모델이 높은 확신도를 보이며 허구를 제시하기 때문에 사용자 판별이 어렵고, 의료·법률·공공 분야에서 심각한 위험을 초래할 수 있다.
Ⅱ. 할루시네이션 유형 및 원인
유형설명예시
Factual Hallucination
(사실적 환각)		현실과 다른 사실 정보 생성존재하지 않는 논문 인용, 잘못된 날짜·수치 제시
Faithfulness Hallucination
(충실성 환각)		주어진 입력 문서와 다른 내용 생성요약 시 원문에 없는 내용 추가
Intrinsic Hallucination주어진 컨텍스트와 직접 모순입력 A라고 했는데 B라고 답변
Extrinsic Hallucination컨텍스트에서 검증 불가한 외부 정보 추가근거 없는 배경 정보 생성

주요 원인

① 확률적 샘플링: 다음 토큰을 확률로 선택하는 구조적 한계
② 지식 단절(Knowledge Cutoff): 학습 데이터 이후 정보 없음
③ 데이터 편향: 학습 데이터 내 오류·편향 학습
④ 과도한 일반화: 패턴 과학습으로 그럴듯한 허구 생성
Ⅲ. 완화 방안
  • 1
    RAG(검색 증강 생성): 외부 지식 저장소에서 관련 문서를 검색하여 프롬프트에 주입. 생성 근거를 명시적으로 제공
  • 2
    RLHF(인간 피드백 강화학습): 인간 평가자의 피드백으로 보상 모델 학습 후 PPO 알고리즘으로 정책 업데이트. 사실성·유용성 향상
  • 3
    낮은 Temperature 설정: Temperature↓ → 고확률 토큰 선택 → 창의성↓, 사실성↑
  • 4
    출처 인용 강제(Grounding): 답변에 근거 문서 인용 필수화. "모른다"고 응답하도록 학습
  • 5
    자동 평가(RAGAS·TruthfulQA): Faithfulness·Answer Relevancy 지표로 할루시네이션 자동 탐지

💡 한 줄 암기 포인트

할루시네이션 완화 3대 축: RAG(지식 주입) + RLHF(정렬 학습) + Grounding(출처 명시)
Ⅳ. 결론

할루시네이션은 LLM의 본질적 한계로 완전 제거는 불가능하다. 공공·의료·법률 분야 AI 도입 시 RAG와 Human-in-the-Loop 검증 체계를 의무화하고, EU AI Act 및 국내 AI 기본법 등 규제에서도 고위험 AI 시스템에 대한 정확성·신뢰성 요건으로 명시되고 있다.

Ⅰ. 개요 — 생성형 AI의 개념과 중요성
생성형 AI(Generative AI)는 학습 데이터의 분포를 모델링하여 새로운 텍스트·이미지·음성·코드·동영상 등 창작 콘텐츠를 생성하는 AI 기술의 총칭이다. 2023년 ChatGPT의 급속한 확산을 계기로 산업 전반의 패러다임 변화를 주도하고 있으며, 정부 및 기업의 생산성 혁신 핵심 수단으로 부상하였다.
Ⅱ. 가. 주요 아키텍처 유형 비교
아키텍처핵심 원리강점약점대표 모델
GAN
(적대적 생성망)		생성자(G)와 판별자(D)의 제로섬 게임. G는 가짜 생성, D는 진위 구별을 경쟁적으로 학습고품질 이미지 생성, 빠른 추론모드 붕괴, 학습 불안정, 다양성 부족StyleGAN, CycleGAN, DCGAN
VAE
(변분 오토인코더)		입력을 잠재 공간(μ, σ)으로 인코딩 후 확률적 복원. KL Divergence로 잠재 분포 정규화연속적·매끄러운 잠재 공간, 이상 탐지생성 품질이 GAN 대비 낮음(흐릿함)VQ-VAE, β-VAE
Diffusion Model
(확산 모델)		데이터에 점진적 노이즈 추가(Forward) 후 역방향으로 노이즈 제거(Reverse)를 학습. Score Matching 기반최고 품질 이미지, 다양성, 학습 안정추론 속도 느림(수십~수백 스텝)DALL·E 3, Stable Diffusion, Midjourney, Sora
Transformer/LLMSelf-Attention 기반 자기회귀(AR) 토큰 예측. 스케일링 법칙으로 능력 창발범용 언어 이해·생성, 코드·추론연산 비용, 환각, 긴 컨텍스트 한계GPT-4, Claude, Gemini, LLaMA, HyperCLOVA X
Ⅲ. 나. 각 아키텍처의 동작 원리 및 대표 모델
Diffusion Model 동작 원리 (가장 최신·중요) [Forward Process - 노이즈 추가] x₀(원본) → x₁ → x₂ → ... → xT(순수 가우시안 노이즈) 각 단계: q(xₜ|xₜ₋₁) = N(xₜ; √(1-βₜ)xₜ₋₁, βₜI) [Reverse Process - 노이즈 제거 학습] xT → ... → x₁ → x₀(복원) U-Net(또는 Transformer) θ로 각 단계의 노이즈 εθ(xₜ,t) 예측 [조건부 생성 - Classifier-Free Guidance] 텍스트 프롬프트 c를 조건으로 주입 → 텍스트-이미지 정렬

LLM 스케일링 법칙 (Scaling Law)

모델 성능(Loss)은 파라미터 수(N)·데이터 양(D)·연산량(C)의 거듭제곱 법칙을 따름.
Chinchilla 법칙: 최적 성능은 N과 D를 균형 있게 늘릴 때 달성 (N ≈ 20×D 토큰)

예: 질문 → "검색이 필요하다" → 웹 검색 API 호출 → 결과 관찰 → "추가 계산 필요" → 코드 실행 → 최종 답변

데이터 드리프트
(Data Drift)		입력 데이터의 통계적 분포 변화 (P(X) 변화)사용자 패턴 변화, 외부 환경 변화PSI, KS-Test, JS Divergence
개념 드리프트
(Concept Drift)		입력-출력 관계 자체가 변화 (P(Y|X) 변화)트렌드 변화, 정책·규정 개정모델 성능 지표 모니터링 (Acc·F1 급락)
레이블 드리프트
(Label Drift)		출력 레이블 분포 변화 (P(Y) 변화)클래스 불균형 심화예측 분포 통계 모니터링
피처 드리프트
(Feature Drift)		특정 입력 피처값의 분포 변화센서 노후화, 데이터 수집 오류피처별 개별 분포 모니터링

재학습(Retraining) 전략

① 일정 기반(Time-Based): 주기적 재학습 (매일·매주). 단순하나 드리프트 미감지 가능
② 성능 기반(Performance-Based): 메트릭이 임계값 이하로 하락 시 트리거. 정교하나 레이블 수집 필요
③ 드리프트 기반(Drift-Based): 입력 분포 변화 탐지 시 즉시 트리거. 레이블 없이 감지 가능
Ⅴ. 결론 및 발전 방향

MLOps는 ML 모델을 실험실에서 프로덕션으로 안정적으로 이전하는 필수 엔지니어링 체계이다. 최근 LLMOps로 발전하여 파운데이션 모델의 파인튜닝·배포·평가·모니터링을 통합 관리하는 방향으로 진화하고 있으며, 공공 AI 도입 감리 시 MLOps 성숙도(레벨 0~3) 평가가 새로운 점검 영역으로 부상하고 있다.

Ⅱ. 나. 핵심 기술 및 동작 원리

CLIP (Contrastive Language-Image Pretraining, OpenAI 2021)

이미지-텍스트 쌍 4억 개를 대조 학습(Contrastive Learning)으로 사전학습.
같은 쌍(Positive)의 이미지·텍스트 임베딩은 가깝게, 다른 쌍(Negative)은 멀게 학습.
→ Zero-Shot 이미지 분류, 이미지 검색, VLM의 기반 인코더로 활용.
VLM(Vision-Language Model) 아키텍처 이미지 입력 텍스트 입력 ↓ ↓ [Vision Encoder] [LLM Tokenizer] (ViT / CLIP 등) ↓ ↓ [LLM Encoder] [이미지 임베딩] ↓ ↓ [Cross-Attention or] [Projection Layer] ─→ [Prefix Fusion or] ↓ [Concat] ↓ [LLM Decoder] ↓ 텍스트 출력 (캡셔닝·VQA·추론)
모델특징핵심 기여
CLIP (2021)이미지-텍스트 대조 학습Zero-Shot 시각 분류, VLM 기반 구축
Flamingo (2022)Frozen LLM + 시각 인코더 + Perceiver ResamplerFew-Shot 멀티모달 학습, 인터리빙
LLaVA (2023)CLIP + LLaMA, 경량 Projection Layer오픈소스 VLM의 기준. 학술 연구 촉진
GPT-4V / Gemini (2023~)대규모 멀티모달 파운데이션 모델의료·법률·코드 등 전문 분야 멀티모달 추론
Sora (2024)Diffusion Transformer 기반 텍스트→동영상고품질 장편 동영상 생성 (최대 60초)

Ⅰ. 개요 — AI 보안 위협의 중요성
AI 시스템이 의료·금융·자율주행·공공서비스 등 핵심 인프라에 도입됨에 따라, AI 모델 자체를 공격 대상으로 하는 새로운 유형의 보안 위협이 급증하고 있다. 기존 사이버 보안과 달리 AI 보안은 모델의 추론 과정·학습 데이터·파라미터 자체를 공격 대상으로 하는 특수성을 가진다.
Ⅱ. 가. AI 모델 주요 공격 유형
공격 유형원리목표·피해사례
적대적 예제
(Adversarial Example)		사람이 인지하기 어려운 미세한 입력 변조(Perturbation)를 가하여 모델을 오분류 유도자율주행 표지판 오인식, 얼굴인식 우회정지 표지판 → 속도제한 표지판으로 오인
데이터 포이즈닝
(Data Poisoning)		학습 데이터에 악성 샘플을 삽입하여 모델에 백도어(Backdoor) 삽입특정 트리거 입력 시 오동작. 악성 분류얼굴인식에 특정 안경 착용 시 다른 사람으로 인증
모델 역전 공격
(Model Inversion)		API 응답을 반복 쿼리하여 학습 데이터를 역추론학습 데이터 내 개인정보 추출의료 모델에서 환자 얼굴 이미지 복원
멤버십 추론 공격
(Membership Inference)		특정 데이터가 학습에 사용되었는지 통계적으로 판별프라이버시 침해 (의료 기록 학습 여부 노출)GPT류 모델에 개인 문서 포함 여부 탐지
모델 추출
(Model Stealing)		공개 API 쿼리를 통해 모델의 기능을 복제지적재산권 침해, 경쟁 모델 무단 생성상용 ML API를 쿼리하여 유사 모델 구축

AI 보안은 기존 사이버 보안과 차별화된 AI 고유 위협(적대적 예제, 프롬프트 인젝션, 데이터 포이즈닝)에 대응하는 새로운 전문 영역이다. NIST AI RMF(AI 위험 관리 프레임워크)와 OWASP LLM Top 10을 감리 기준으로 채택하고, AI 시스템의 생명주기 전반에 걸쳐 Security by Design을 적용하는 것이 공공 AI 도입의 신뢰성 확보 핵심 전략이다.
