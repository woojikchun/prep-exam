⏱ 목표 15~20분 📄 1~2페이지 💯 10점
Ⅰ. 개요
IPv4(1981, RFC 791)는 32비트 주소 체계로 전 세계 인터넷의 기반을 구축하였으나, 약 43억 개의 제한된 주소 공간이 고갈(IANA 2011년 고갈 선언)되었다. IPv6(1998, RFC 2460)는 128비트 주소 체계로 사실상 무한대의 주소를 제공하며, 자동 설정·내장 보안(IPSec 필수) 등 다양한 개선 사항을 담고 있다.

TCP 프로토콜의 연결 제어 메커니즘에 대하여 다음을 설명하시오.
(1) 3-Way Handshake — 절차, 각 단계별 플래그와 상태 전이
(2) 4-Way Handshake — 절차, TIME_WAIT 상태의 필요성

⏱ 목표 15~20분 📄 1~2페이지 💯 10점
Ⅰ. 개요
TCP(Transmission Control Protocol)는 연결 지향(Connection-Oriented), 신뢰성 있는 전송 계층 프로토콜이다. 데이터 전송 전 반드시 논리적 연결을 설정(3-Way Handshake)하고, 전송 완료 후 연결을 해제(4-Way Handshake)하는 절차를 수행한다.
Ⅱ. 3-Way Handshake (연결 설정)
클라이언트(C) 서버(S) CLOSED LISTEN │ │ │──── ① SYN(seq=x) ─────────▶│ C: SYN_SENT │ │ S: SYN_RECEIVED │◀─── ② SYN+ACK(seq=y,ack=x+1)│ │ │ │──── ③ ACK(ack=y+1) ────────▶│ C·S: ESTABLISHED │ │
단계송신자→수신자플래그의미
① SYNC → SSYN=1연결 요청. 클라이언트 초기 순서번호(ISN) 전달
② SYN+ACKS → CSYN=1, ACK=1연결 수락. 서버 ISN 전달 + 클라이언트 SYN 확인
③ ACKC → SACK=1서버 SYN 확인. 연결 수립 완료
Ⅲ. 4-Way Handshake (연결 해제)
클라이언트(C) 서버(S) ESTABLISHED ESTABLISHED │ │ │──── ① FIN ─────────────────▶│ C: FIN_WAIT_1 │◀─── ② ACK ─────────────────│ S: CLOSE_WAIT, C: FIN_WAIT_2 │ (서버 데이터 전송 가능) │◀─── ③ FIN ─────────────────│ S: LAST_ACK │──── ④ ACK ─────────────────▶│ C: TIME_WAIT(2MSL) │ │ S: CLOSED (2MSL 대기 후) C: CLOSED
🔑 TIME_WAIT(2MSL) 상태가 필요한 이유
① 마지막 ACK가 소실될 경우 서버의 FIN 재전송에 대응하기 위해
② 이전 연결의 지연 패킷이 새 연결에 혼입되는 것을 방지하기 위해
(MSL: Maximum Segment Lifetime ≈ 60초 → TIME_WAIT = 최대 120초)
Ⅳ. SYN Flooding 공격 및 대응
SYN Flooding: 공격자가 출발지 IP를 위조한 대량의 SYN 패킷을 전송 → 서버의 반연결(Half-open) 큐를 가득 채워 정상 연결 불가 → DoS 유발
대응 방안내용
SYN Cookie서버가 반연결 상태를 저장하지 않고 SYN+ACK에 암호화된 쿠키(seq 번호)를 포함. ACK에서 쿠키 검증 후 연결 수립
연결 타임아웃 단축반연결 큐의 항목 만료 시간을 줄여 큐 고갈 지연
방화벽 Rate Limiting동일 IP에서 초당 SYN 패킷 수 제한
IPS/AntiDDoS비정상 SYN 패턴 탐지 및 차단
📌 한 줄 암기
3-Way: SYN → SYN+ACK → ACK (연결설정)
4-Way: FIN → ACK → FIN → ACK + TIME_WAIT(2MSL) (연결해제)
SYN Flood 대응 → SYN Cookie

Ⅰ. VLAN의 개념 및 도입 목적
VLAN(Virtual LAN)은 물리적 네트워크 위에 논리적인 브로드캐스트 도메인을 분리하는 L2/L3 기술이다. 동일 스위치에 연결된 단말들을 논리적으로 분리하거나, 다른 스위치의 단말들을 하나의 LAN으로 묶을 수 있다.
도입 목적
① 브로드캐스트 도메인 축소: 불필요한 브로드캐스트 트래픽 격리로 성능 향상
② 보안 격리: 부서/업무 단위 논리 분리 (HR·개발·운영 네트워크 격리)
③ 유연성: 물리적 이동 없이 소속 VLAN 변경 가능
④ 비용 절감: 단일 물리 스위치로 다수의 논리 네트워크 구현
Ⅱ. VLAN 구성 방식 비교
구성 방식분류 기준장점단점
포트 기반
(Port-based)		스위치 포트 번호구성 단순, 관리 용이단말 이동 시 재설정 필요
MAC 기반
(MAC-based)		NIC의 MAC 주소단말 이동에 자동 대응MAC 테이블 관리 부담, MAC 스푸핑 취약
프로토콜 기반L3 프로토콜(IP, IPX)프로토콜 별 분리현대 환경에서 활용 감소

Ⅲ. IEEE 802.1Q 트렁킹
트렁크 포트는 여러 VLAN의 트래픽을 하나의 물리 링크로 전달한다. IEEE 802.1Q는 이더넷 프레임에 4바이트 태그를 삽입하여 VLAN을 식별한다.
802.1Q 프레임 구조 ┌───────────┬────────────┬───────────────────────────┬──────────┐ │ 목적지 MAC │ 출발지 MAC │ 802.1Q 태그(4B) │ 페이로드 │ │ (6바이트) │ (6바이트) │[TPID:0x8100][PCP:3][DEI:1]│ + FCS │ │ │ │[VID:12bit(VLAN ID 0~4094)] │ │ └───────────┴────────────┴───────────────────────────┴──────────┘ VLAN ID(VID): 12비트 → 최대 4094개 VLAN (0, 4095 예약) Native VLAN : 태그 없이 전달 (기본 VLAN 1, 보안상 변경 권장)
Inter-VLAN 라우팅
VLAN 간 통신은 L3 스위치 또는 라우터 필요.
Router-on-a-stick: 단일 라우터 포트에 서브인터페이스(Sub-interface)를 구성하여 VLAN 간 라우팅 처리

Ⅰ. 개요 및 주요 파라미터
QoS(서비스 품질)는 네트워크에서 특정 트래픽 유형에 대해 대역폭·지연·지터·패킷 손실을 보장하는 기술 체계이다. VoIP·실시간 영상·금융 거래 등 지연에 민감한 서비스에서 필수적이다.
파라미터정의임계값 예시
대역폭(Bandwidth)초당 전송 가능한 데이터량VoIP: ≥64kbps
지연(Latency)패킷의 종단 간 전송 시간VoIP: <150ms
지터(Jitter)패킷 지연의 변동 폭VoIP: <30ms
패킷 손실률전송 중 유실된 패킷 비율VoIP: <1%
Ⅱ. QoS 구현 메커니즘
  • 1
    분류(Classification): 5-Tuple(SIP·DIP·Sport·Dport·Protocol) 또는 DSCP 값으로 트래픽 유형 식별
  • 2
    마킹(Marking): IP 헤더의 DSCP(6bit) 필드에 트래픽 우선순위 표시. EF(Expedited Forwarding, VoIP), AF(Assured Forwarding), BE(Best Effort)
  • 3
    큐잉(Queuing): 다중 우선순위 큐로 중요 트래픽 우선 처리
    PQ(Priority Queuing): 높은 우선순위 큐 완전 처리 후 낮은 큐 처리
    WFQ(Weighted Fair Queuing): 가중치 기반 공정 분배 → 가장 널리 사용
    CBWFQ: 클래스 기반 WFQ, LLQ(Low Latency Queue) 결합으로 VoIP 최적화
  • 4
    셰이핑/폴리싱: 트래픽 속도 제한. Shaping(지연 후 재전송) vs Policing(초과분 즉시 폐기)
Ⅲ. DiffServ vs IntServ 비교
항목DiffServ (차등화 서비스)IntServ (통합 서비스)
표준RFC 2474/2475RFC 1633

상태 관리상태 비저장 (Stateless)각 라우터에 흐름 상태 저장
확장성높음 (인터넷 규모 적용 가능)낮음 (대규모 네트워크 부적합)
보장 수준상대적 우선순위 보장절대적 QoS 보장 (하드 QoS)
적합 환경ISP 코어망, 엔터프라이즈 WAN소규모 기업망, 화상회의 특화

QoS 4단계: 분류 → 마킹(DSCP) → 큐잉(WFQ/LLQ) → 셰이핑/폴리싱

Ⅱ. OSPF 동작 원리
OSPF(Open Shortest Path First, RFC 2328)는 링크 상태(Link-State) 라우팅 알고리즘을 사용하는 IGP이다.
OSPF 동작 절차 ① 네이버 탐색: Hello 패킷 교환 → 네이버(Neighbor) 관계 수립 ② LSA 교환: 링크 상태 광고(LSA, Link-State Advertisement) 플러딩 ③ LSDB 동기화: 모든 라우터가 동일한 링크 상태 DB(LSDB) 보유 ④ SPF 계산: Dijkstra 알고리즘으로 최단 경로 트리 계산 ⑤ 라우팅 테이블 갱신: SPF 결과를 FIB(포워딩 테이블)에 반영 [OSPF 영역 구조] Area 0 (백본) ─── ABR ─── Area 1 (일반) │ │ ABR ─────── Area 2 ──── ASBR ── 외부망(RIP, BGP) ※ ABR(Area Border Router), ASBR(AS Boundary Router)
OSPF 비용(Cost) = 10⁸ / 인터페이스 대역폭(bps)
100Mbps 링크 Cost = 1, 10Mbps = 10, 1Gbps = 0.1 → 자동 조정 필요 (reference bandwidth 변경)
Ⅲ. BGP 동작 원리 및 경로 선택 속성
BGP-4(RFC 4271)는 경로 벡터(Path-Vector) 프로토콜로, AS 간 정책 기반 라우팅을 수행한다. TCP 179번 포트를 사용하며, eBGP(AS 간)와 iBGP(AS 내부)로 구분된다.

Ⅰ. 개요 — SDN의 개념 및 기존 네트워크와의 차이
SDN(소프트웨어 정의 네트워킹)은 네트워크 장비의 제어 기능(Control Plane)과 데이터 전달 기능(Data Plane)을 분리하여, 중앙 집중식 소프트웨어 컨트롤러로 네트워크 전체를 프로그래밍 방식으로 제어하는 네트워크 패러다임이다. 2008년 스탠퍼드대 Martin Casado·Nick McKeown이 제안하였으며, OpenFlow 프로토콜을 통해 표준화되었다.

Ⅰ. 개요
5G(5세대 이동통신)는 ITU-R IMT-2020 표준에 따라 3GPP Release 15(2018)부터 표준화된 차세대 이동통신 기술이다. 4G LTE 대비 최대 20Gbps 속도, 1ms 이하 지연, 100배 연결 밀도를 목표로, 단순 통신을 넘어 산업 디지털 전환의 핵심 인프라 역할을 수행한다.

구성 요소: NSSF(네트워크 슬라이스 선택 기능) → AMF(접속 관리) → SMF(세션 관리) → UPF(사용자 플레인 기능)

Ⅳ. 다. MEC 개념 및 5G 연계 효과
MEC(Multi-access Edge Computing, ETSI 표준)는 이동통신 네트워크의 기지국(Base Station) 또는 집선국 근처에 컴퓨팅·스토리지를 배치하여 초저지연·높은 대역폭·로컬 데이터 처리를 가능하게 하는 에지 컴퓨팅 아키텍처이다.
MEC 기대 효과내용
초저지연 실현코어 클라우드 대신 에지에서 처리 → 왕복 지연 5ms 이하 달성. URLLC 서비스 요구 충족
트래픽 최적화로컬 처리로 코어망·백홀 트래픽 감소. 동영상 캐싱으로 eMBB 효율화
데이터 주권민감 데이터가 외부 클라우드로 전송되지 않음. 의료·공공 데이터 보호
서비스 연계V2X(차량-사물 통신), 증강현실, 실시간 영상 분석 등 AI/IoT 서비스 지원
5G + 네트워크 슬라이싱 + MEC 통합 아키텍처 UE(단말) ── gNB(5G 기지국) │ ┌────────┴──────────┐ ▼ ▼ MEC 서버(에지) 5GC (코어) [로컬 UPF·앱 서버] [AMF·SMF·UPF] 지연 < 5ms [슬라이스 오케스트레이터] │ │ 로컬 서비스 중앙 클라우드 (자율주행·공장 AI) (인터넷·OTT·빅데이터)
Ⅴ. 결론

5G는 단순 고속 통신을 넘어 산업 디지털 전환의 핵심 플랫폼이다. SA 구조·네트워크 슬라이싱·MEC의 통합을 통해 eMBB·mMTC·URLLC 서비스를 단일 인프라에서 유연하게 제공할 수 있다. 국내에서도 공공 안전망(재난망)·스마트팩토리·자율주행 특화망에 5G SA 기반 슬라이싱이 적용되고 있으며, B5G·6G 표준화에서도 지능형 네트워크(AI-native) 방향으로 발전하고 있다.

Ⅰ. 가. CDN의 개념, 목적 및 구성요소
CDN(콘텐츠 전송 네트워크)은 지리적으로 분산된 엣지 서버(PoP, Point of Presence)에 콘텐츠를 캐시·배포하여, 최종 사용자와 가장 가까운 서버에서 콘텐츠를 전달함으로써 지연 최소화·가용성 향상·원본 서버 부하 감소를 달성하는 분산 인프라이다.

CDN 콘텐츠 요청 흐름 (DNS 기반 라우팅) ① 사용자: www.example.com 요청 ② 로컬 DNS 질의 → CDN DNS(글로벌 로드밸런서) 응답 → 사용자와 가장 가까운 PoP의 IP 반환 (Anycast 활용) ③ 사용자 → 선택된 엣지 서버 요청 ④-A [Cache Hit] 엣지 서버에 콘텐츠 존재 → 엣지 서버가 직접 응답 (지연 최소, 오리진 부하 없음) ④-B [Cache Miss] 엣지 서버에 콘텐츠 없음 → 엣지 → 상위 PoP(미들티어) → 오리진 서버 요청 → 응답 수신 후 엣지에 캐시 저장 → 사용자에 응답

보안 위협공격 방식대응 방안
Cache Poisoning공격자가 엣지 서버 캐시에 악성 콘텐츠를 주입하여 다수 사용자에게 배포캐시 키 정규화, Host 헤더 검증, 입력 검증 강화, HTTPS 강제
DDoS 증폭CDN의 글로벌 대역폭을 악용하여 오리진 또는 특정 타깃 공격레이트 리미팅, Bot 탐지, WAF(Web App Firewall) 연동

Ⅳ. 결론

CDN은 현대 인터넷 서비스의 성능·가용성·보안의 핵심 인프라로, 전 세계 웹 트래픽의 70% 이상이 CDN을 통해 전달된다. 최근에는 엣지 컴퓨팅과 결합한 Edge Functions(Cloudflare Workers, AWS Lambda@Edge) 형태로 발전하여, 단순 캐시를 넘어 엣지에서의 API 처리·인증·A/B 테스트까지 수행하는 지능형 엣지 플랫폼으로 진화하고 있다.

논술형 네트워크 보안 🔴 초고빈출
DDoS 공격과 Zero Trust 네트워크 보안에 대하여 다음을 설명하시오.
가.DDoS(Distributed Denial of Service) 공격의 유형 분류 (볼류메트릭·프로토콜·애플리케이션 계층)
나.DDoS 탐지·대응 체계 (블랙홀 라우팅, Scrubbing Center, BGP FlowSpec, Anycast)
다.Zero Trust 네트워크 아키텍처의 원칙과 네트워크 보안 적용 방안
볼류메트릭 공격SYN Flood·UDP FloodScrubbingZero Trust마이크로 세그멘테이션132회 유사
DDoS 공격 유형·탐지·대응 체계 및 Zero Trust 네트워크 보안 적용
⏱ 목표 25분 📄 3.5페이지 💯 25점
Ⅰ. 개요
DDoS(분산 서비스 거부) 공격은 다수의 분산된 공격 호스트(봇넷)가 협력하여 특정 서버·네트워크·서비스의 가용성을 고갈시키는 사이버 공격이다. 공격 규모가 Tbps 수준으로 대형화되고 있으며, 다계층 복합 공격으로 진화하고 있다.
Ⅱ. 가. DDoS 공격 유형 분류
계층공격 유형공격 방식목표
볼류메트릭
(L3/L4)		UDP Flood대량 UDP 패킷으로 대역폭 포화네트워크 대역폭 고갈
DNS/NTP 증폭작은 요청→큰 응답 유도로 트래픽 증폭 (증폭율 50배+)Tbps급 공격 생성
프로토콜
(L3~L4)		SYN Flood위조 SYN 패킷 대량 전송→반연결 큐 고갈서버 연결 자원 고갈
ICMP FloodPing 패킷 과부하로 장비 CPU·대역폭 점유네트워크 장비 마비
애플리케이션
(L7)		HTTP Flood정상 HTTP 요청 대량 전송 (GET·POST)웹서버 처리 자원 고갈
Slowloris연결 유지 패킷을 느리게 전송하여 스레드 점유소량 트래픽으로 서버 마비
Ⅲ. 나. DDoS 탐지·대응 체계
  • 1
    블랙홀 라우팅(RTBH): 공격 목적지 IP 트래픽을 Null 인터페이스로 라우팅하여 폐기. 구현 단순하지만 정상 트래픽도 차단되는 단점
  • 2
    Scrubbing Center: 트래픽을 전용 세정 센터로 우회(BGP Re-routing)하여 악성 트래픽 필터링 후 정상 트래픽만 목적지로 전달
    업스트림 BGP Peer에 공격 IP prefix 광고 → 트래픽 Scrubbing Center 유입
    DPI, 서명 기반, 행동 분석으로 악성 트래픽 차단 → 정상 트래픽 원래 목적지 전달
  • 3
    BGP FlowSpec (RFC 5575): BGP를 통해 라우터에 세밀한 트래픽 필터 규칙을 자동 배포. 5-Tuple 기반 Rate Limit·Drop 규칙을 수천 대 라우터에 즉시 적용
  • 4
    Anycast 기반 분산: CDN처럼 공격 트래픽을 전 세계 PoP에 분산 흡수. 단일 집중 대신 글로벌 대역폭으로 공격 희석
DDoS 대응 흐름 봇넷 공격 트래픽 ↓ [업스트림 ISP] → BGP 트래픽 우회 → [Scrubbing Center] ↓ 악성 필터링 (DPI·행동분석) ↓ [정상 트래픽만] → 원본 서버 [BGP FlowSpec 규칙 전파] → 엣지 라우터 차단
Ⅳ. 다. Zero Trust 네트워크 아키텍처 적용
Zero Trust 3대 원칙 (NIST SP 800-207)
① Never Trust, Always Verify: 내부망 접속도 신뢰하지 않음. 모든 접근을 검증
② 최소 권한(Least Privilege): 필요한 리소스에만 접근 권한 부여
③ 침해 가정(Assume Breach): 이미 침해되었다고 가정하고 폭발 반경 최소화
구현 기술내용
마이크로 세그멘테이션SDN/VLAN/방화벽으로 워크로드를 세밀하게 분리. 수평 이동(Lateral Movement) 차단
ZTNA (Zero Trust Network Access)VPN 대신 애플리케이션 단위 접근 제어. 사용자·기기·컨텍스트 기반 동적 권한 부여
지속적 검증세션 중에도 주기적으로 사용자·기기 상태 재검증. BehaviorAnalytics(UEBA) 적용
암호화 전 구간내부 트래픽도 mTLS로 암호화. 인증서 기반 서비스 간 인증
통합 가시성SIEM·SOAR로 전체 네트워크 트래픽 실시간 모니터링·자동 대응
Ⅴ. 결론

DDoS 공격은 규모·정교함 모두 지속 증가하여, 단일 대응 기법만으로는 한계가 있다. ISP 수준의 Scrubbing, BGP FlowSpec, Anycast 분산을 계층적으로 결합하는 Multi-Layer Defense가 필수적이다. 나아가 Zero Trust 아키텍처를 네트워크 설계의 기본으로 채택함으로써, DDoS 이후의 내부망 침투·수평이동까지 대비하는 심층 방어(Defense in Depth) 체계를 구축해야 한다.

논술형 MPLS·SD-WAN 🟡 중빈출
MPLS와 SD-WAN에 대하여 다음을 설명하시오.
가.MPLS(Multi-Protocol Label Switching)의 동작 원리와 MPLS VPN(L2/L3 VPN)
나.SD-WAN의 개념, 아키텍처 및 기존 MPLS WAN과의 비교
다.기업 WAN 전환 전략 (MPLS→SD-WAN) 및 하이브리드 WAN 구성 방안
레이블 스위칭LER·LSRVRF언더레이·오버레이애플리케이션 인식 라우팅
MPLS 동작 원리·VPN과 SD-WAN 아키텍처 비교 및 하이브리드 WAN 전환 전략
⏱ 목표 25분 📄 3.5페이지 💯 25점
Ⅰ. 개요
기업 WAN(광역 통신망)은 전통적으로 MPLS 전용선 기반으로 구성되었으나, 클라우드 서비스 확산과 분산 근무 환경에 따라 인터넷·4G/5G를 활용하는 SD-WAN으로의 전환이 가속화되고 있다.
Ⅱ. 가. MPLS 동작 원리 및 MPLS VPN
MPLS(다중 프로토콜 레이블 스위칭, RFC 3031)는 IP 라우팅 대신 20비트 레이블을 붙여 전달하는 고속 패킷 전달 기술이다. 레이블 기반으로 O(1) 룩업이 가능하여 대규모 코어망의 처리 속도를 향상시킨다.
MPLS 동작 구성요소 LER(Label Edge Router) : MPLS 도메인 진입/퇴출 라우터. 레이블 Push/Pop LSR(Label Switch Router): 코어 라우터. 레이블 Swap만 수행 (IP 미참조) LSP(Label Switched Path): 사전 설정된 레이블 경로 동작 순서 ① 진입 LER: IP 패킷 분류 → 레이블 Push → LSP로 전달 ② 중간 LSR: 입력 레이블 → 출력 레이블 Swap → 다음 LSR 전달 ③ 퇴출 LER: 레이블 Pop → 일반 IP 패킷으로 복원 → 목적지 전달
MPLS VPN 유형동작특징적용
L2 VPN
(VPLS·VPWS)		L2 프레임을 MPLS로 캡슐화. 이더넷 브리지처럼 동작고객 라우터 독립 운영. 투명한 L2 연결데이터센터 간 이더넷 연장
L3 VPN
(RFC 4364)		PE 라우터가 고객 라우팅 정보를 VRF에 격리·관리. MP-BGP로 PE 간 분배고객별 VRF로 주소 중첩 허용. 운용 단순기업 MPLS WAN 표준 구성
Ⅲ. 나. SD-WAN 아키텍처 및 MPLS WAN 비교

Ⅳ. 다. MPLS→SD-WAN 전환 전략 및 하이브리드 WAN
  • 1
    단계적 전환 전략
    1단계: 비중요 트래픽(인터넷·SaaS)을 SD-WAN으로 오프로드. MPLS는 미션 크리티컬 앱 전담
    2단계: 하이브리드 WAN 운영 (MPLS + SD-WAN 오버레이 병행)
    3단계: MPLS 회선 축소 또는 완전 대체. SD-WAN + 인터넷 + 4G/5G로 전환
  • 2
    하이브리드 WAN 구성: SD-WAN Edge에서 MPLS·인터넷·4G를 동시 연결. 애플리케이션 SLA에 따라 트래픽 자동 분산
    ERP·실시간 결제 → MPLS (낮은 지연·보장 대역폭)
    MS365·Zoom·SaaS → 로컬 인터넷 직접 연결 (최적 경로)
    MPLS 장애 시 → 인터넷·4G로 자동 페일오버 (Packet Loss < 0.1초)
  • 3
    SASE 연계: SD-WAN + 클라우드 보안(CASB·SWG·ZTNA)을 통합한 SASE(Secure Access Service Edge) 아키텍처로 진화. Zero Trust WAN 실현
Ⅴ. 결론

MPLS는 신뢰성 높은 엔터프라이즈 WAN의 사실상 표준이었으나, 클라우드 퍼스트 전략과 분산 근무 환경에서 SD-WAN이 비용·유연성·클라우드 최적화 측면에서 우위를 보인다. 미션 크리티컬 애플리케이션은 하이브리드 WAN으로 MPLS를 유지하면서 단계적으로 SD-WAN으로 전환하고, 장기적으로 SASE 통합을 통해 네트워크와 보안을 클라우드 네이티브로 통합하는 것이 최적의 전략이다.
