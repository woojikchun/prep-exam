MBD(Model-Based Development) 기반 검증 계층
임베디드 시스템·자율주행·항공·국방 등 안전 필수(Safety-Critical) 시스템 개발에서 실물 하드웨어 없이 소프트웨어·알고리즘을 단계적으로 검증하는 테스트 방법론이다. MIL → SIL → PIL → HIL 4단계로 구성되며, 단계가 올라갈수록 실제 환경에 가까워진다.

4단계 루프 테스팅 개요

한 줄 암기: MIL(모델) → SIL(소프트웨어+PC) → PIL(소프트웨어+실제MCU) → HIL(실제ECU+HW시뮬레이터) — 단계가 오를수록 실제 환경에 근접

등장 배경 — 왜 필요한가

기존 방식의 문제
  • 실차 테스트 → 비용·시간 폭발
  • 위험 시나리오 재현 불가 (충돌·폭발 등)
  • 하드웨어 프로토타입 확보 전 검증 불가
  • 버그 발견 시 수정 비용 기하급수 증가
  • 안전 기능(Fail-Safe) 검증 위험
루프 테스팅 해결
  • 개발 초기 소프트웨어 결함 조기 발견
  • 위험·극한 시나리오 안전 재현
  • HW 없이 SW 먼저 검증 → 개발 병렬화
  • 반복 테스트 자동화 → 회귀 테스트 용이
  • ISO 26262·DO-178C 등 안전 표준 준수
적용 도메인: 자동차(ADAS·EV·자율주행) / 항공(비행제어·항법) / 철도(신호제어) / 국방(무기체계) / 산업제어(PLC·DCS) — 모두 Safety-Critical 영역
SIL(Software-in-the-Loop)이란 모델에서 자동 생성된 코드(Auto-generated Code)를 실제 타깃 하드웨어 없이 호스트 PC의 시뮬레이션 환경에서 실행하여 기능적 정확성을 검증하는 테스트 기법이다.

HIL(Hardware-in-the-Loop)이란 실제 ECU(Electronic Control Unit) 등 대상 하드웨어를 실시간(Real-Time) 플랜트 시뮬레이터에 연결하여, 실제 환경을 모사하는 전기 신호를 주고받으며 통합 시스템을 검증하는 테스트 기법이다.
