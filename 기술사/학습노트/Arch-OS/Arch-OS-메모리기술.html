Ⅰ. 메모리 월의 개념 및 차세대 메모리 기술의 등장 배경
1. 메모리 월(Memory Wall)이란

메모리 월(Memory Wall)이란 CPU/GPU 연산 속도(FLOPS) 향상 속도에 비해 메모리 접근 대역폭(Bandwidth) 및 레이턴시(Latency) 개선 속도가 현저히 느려, 연산 유닛이 데이터를 기다리는 병목 현상이다. 1995년 Wulf·McKee가 최초 제시하였으며, AI 시대에 재차 심화되고 있다.

대형 언어 모델(LLM) 추론 시 수천억 개의 파라미터를 반복 로드해야 하므로, 연산 능력이 아닌 메모리 대역폭이 실질적인 성능 병목으로 작용한다.

【그림 1】 메모리 월 — CPU/GPU 성능과 메모리 대역폭 간 증가 격차
2. 등장 배경

① AI·LLM 워크로드 폭증: GPT-4 등 수천억 파라미터 모델의 추론 시 초당 수십 TB 메모리 대역폭 요구

② DDR DRAM의 한계: DDR5 최대 ~51 GB/s의 대역폭은 H100 GPU의 3.35 TFLOPS 연산 수요 대비 턱없이 부족

③ 데이터 이동 에너지 비용: 메모리↔연산유닛 간 데이터 이동이 전체 시스템 전력의 60% 이상을 소비 → 근접 연산(Near-Memory Computing) 필요

④ 메모리 용량 확장 한계: GPU HBM 온보드 메모리(80GB)는 대형 모델 전체 파라미터 탑재 불가 → CXL 기반 메모리 풀링 필요

Ⅱ. HBM(High Bandwidth Memory)의 구조 및 동작 원리
1. 정의 및 표준

HBM(High Bandwidth Memory)은 DRAM 다이(Die)를 수직으로 적층(3D Stacking)하고 TSV(Through-Silicon Via)로 연결한 후, GPU·AI 가속기와 동일한 인터포저(Interposer) 위에 패키징하는 고대역폭 메모리 표준이다. JEDEC(반도체 표준화 기구)에서 규정하며, 현재 HBM3E까지 상용화되어 있다.

Ⅲ. PIM(Processing-In-Memory)의 개념·유형 및 HBM과의 결합
1. PIM의 정의 및 필요성

PIM(Processing-In-Memory)은 DRAM 내부 또는 인접부에 연산 로직을 내장하여, 프로세서–메모리 간 데이터 이동 없이 메모리 내에서 직접 연산을 수행하는 기술이다. '근접 메모리 처리(Near-Memory Processing, NMP)'라고도 불린다.

데이터 이동이 연산 비용의 수십 배에 달하는 AI 워크로드에서, PIM은 대역폭·전력 효율을 동시에 개선하는 패러다임 전환 기술이다.

3. HBM-PIM 결합 구조 (HBM with PIM)

삼성·SK하이닉스는 HBM의 Base Die에 GFPU(General-purpose Floating-Point Unit) 또는 벡터 연산 유닛을 탑재한 HBM-PIM(Aquabolt-X, AiMX)을 개발하였다. HBM의 초고대역폭과 PIM의 근접 연산을 결합하여 AI 추론에서 최적 성능을 달성한다.

Ⅳ. CXL(Compute Express Link) 메모리 확장 기술
1. CXL의 정의

CXL(Compute Express Link)은 CPU·GPU·FPGA·가속기 간, 그리고 이들과 메모리·스토리지 장치 간의 고속·저지연 캐시 코히런트(Cache Coherent) 인터커넥트 표준이다. PCIe 5.0/6.0 물리 계층을 기반으로 하며, Intel·AMD·ARM·삼성·NVIDIA 등이 참여하는 CXL Consortium이 표준화를 주도한다.

핵심 가치는 메모리 풀링(Memory Pooling): 여러 서버가 대용량 메모리를 공유하여, GPU 온보드 HBM 용량의 한계를 소프트웨어 투명하게 확장한다.

2. 발전 방향

① HBM4 / HBM4E (2025~2026): 버스 폭 2048-bit, 1.5 TB/s 이상 대역폭, Logic Die를 TSMC N5 파운드리 공정으로 분리 제조하는 하이브리드 본딩 방식 도입

② HBM-PIM 상용화 확대: SK하이닉스 AiMX 2세대, 삼성 HBM3 PIM 탑재 AI 서버 확산. 소프트웨어 SDK 성숙으로 개발 복잡성 해소 예정

③ CXL 3.x 기반 메모리 패브릭: 수십 TB 메모리 풀을 여러 AI 서버가 공유하는 메모리 중심 컴퓨팅(Memory-Centric Computing) 데이터센터 아키텍처 실현

④ 3D IC 이종 집적(Heterogeneous Integration): CPU·GPU·HBM·PIM·CXL 컨트롤러를 단일 패키지에 통합하는 SoC-on-Interposer 방식으로 시스템 전체 성능·전력 최적화

Ⅵ. 결론

AI·HPC 워크로드의 메모리 집약성이 가속화되면서, HBM(대역폭)·PIM(근접 연산)·CXL(용량 확장)은 상호 보완적 역할을 수행하는 차세대 메모리 트리오로 부상하고 있다. 이들 기술의 융합은 단순한 메모리 성능 향상을 넘어 메모리 중심 컴퓨팅(Memory-Centric Computing) 아키텍처로의 패러다임 전환을 이끌 것이며, AI 반도체 설계 및 데이터센터 아키텍처의 핵심 경쟁력 요소가 될 것이다.

★ 기술사 핵심 코멘트
차세대 메모리 기술의 핵심은 대역폭(HBM) × 연산 효율(PIM) × 용량(CXL)의 삼각 최적화이며, 시스템 감리 시 다음을 중점 검토해야 한다.
① HBM 도입 시 — TSV 수율에 따른 수급 리스크, 인터포저 설계의 열방출(Thermal) 관리 적정성
② PIM 적용 시 — 기존 소프트웨어 스택(CUDA·ROCm) 호환성, PIM 프로그래밍 모델 표준화 여부
③ CXL 도입 시 — 메모리 코히런시 보장 메커니즘, 공유 메모리 접근 시 보안·격리 정책(테넌트 분리) 구현 여부
