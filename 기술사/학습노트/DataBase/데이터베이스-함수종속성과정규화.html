Ⅰ. 함수 종속성의 개념, 종류 및 이상 현상과의 관계
1. 함수 종속성(Functional Dependency)의 정의

릴레이션 R의 두 속성 집합 X, Y에 대해 X의 값이 결정되면 Y의 값이 유일하게 결정될 때 "Y는 X에 함수적으로 종속된다"라고 하며, X → Y로 표기한다. X를 결정자(Determinant), Y를 종속자(Dependent)라 한다.

함수 종속성은 릴레이션의 의미(Semantics)와 무결성 제약을 표현하는 핵심 수단이며, 정규화의 이론적 토대가 된다.

2. 함수 종속성의 종류
종류정의예시
완전 함수 종속
(Full FD)		 X → Y 이고, X의 어떤 진부분집합 X'에 대해서도 X' → Y 가 성립하지 않음 (기본키 전체에 종속) {학번, 과목코드} → 성적
(학번만으로 성적 결정 불가)
부분 함수 종속

(학번만으로 결정 가능 → 부분 종속)
이행 함수 종속

Ⅱ. 암스트롱 공리(Armstrong's Axioms)와 함수 종속성 추론
1. 암스트롱 공리 (건전성 · 완전성 보장)
공리정의표기
① 반사율
(Reflexivity)		Y ⊆ X 이면 X → Y 성립 (자명한 FD)Y ⊆ X  ⟹  X → Y
② 첨가율
(Augmentation)		X → Y 이면, XZ → YZ 성립 (양쪽에 Z 추가)X → Y  ⟹  XZ → YZ
③ 이행율
(Transitivity)		X → Y 이고 Y → Z 이면 X → Z 성립X→Y, Y→Z  ⟹  X→Z

분해 규칙X→YZ 이면 X→Y, X→Z반사율 + 이행율
가의율
(Pseudotransitivity)		X→Y, WY→Z 이면 WX→Z첨가율 + 이행율

3. 속성 폐포(Attribute Closure)와 최소 커버

속성 폐포(X⁺): 주어진 FD 집합 F 하에서 X로부터 암스트롱 공리를 반복 적용해 결정할 수 있는 모든 속성의 집합.
 → X⁺에 모든 속성이 포함되면 X가 슈퍼키(Superkey)임을 판별 가능

최소 커버(Minimal Cover, Fc): FD 집합에서 중복·불필요 종속성을 제거해 동치이면서 크기가 최소인 FD 집합. 정규화 절차의 기준으로 활용된다.

가. 제1정규형 (1NF) — 원자값(Atomic Value) 보장

정의: 모든 속성 값이 원자값(더 이상 분해 불가)이어야 하며, 반복 그룹(Repeating Group)이 존재하지 않아야 한다.

위 릴레이션은 모든 속성이 단일값이므로 1NF를 만족. 단, 부분 함수 종속이 존재하여 이상 현상 발생.

잔존 문제: 학번 → 학생이름(부분 종속), 학과코드 → 학과명(이행 종속)

나. 제2정규형 (2NF) — 부분 함수 종속 제거

정의: 1NF를 만족하고, 기본키에 대한 부분 함수 종속이 존재하지 않아야 한다. (모든 비주요 속성이 기본키에 완전 함수 종속)

학번 → 학생이름, 학과코드는 복합키 {학번, 과목코드}의 일부인 학번에만 종속 → 분리 필요

다. 제3정규형 (3NF) — 이행 함수 종속 제거

정의: 2NF를 만족하고, 이행 함수 종속이 존재하지 않아야 한다.

라. 보이스-코드 정규형 (BCNF) — 모든 결정자가 후보키

정의: 모든 함수 종속 X → Y에서 X가 반드시 슈퍼키(Superkey)이어야 한다. 3NF보다 강한 조건으로, 3NF를 만족하더라도 후보키가 여럿이고 결정자가 후보키가 아닌 경우 BCNF 위반이 된다.

Ⅳ. 정규화의 장단점 및 역정규화(Denormalization) 고려사항
1. 정규화의 장단점
구분내용
장점 ① 데이터 중복 최소화 → 저장 공간 절감
② 이상 현상(삽입·삭제·갱신) 제거 → 데이터 일관성 보장
③ 무결성 유지 용이 → 갱신 시 한 곳만 수정
④ 구조 유연성 → 새로운 요구사항에 대한 스키마 확장 용이
단점 ① 조인(JOIN) 연산 증가 → 분해된 테이블 재결합 필요로 쿼리 성능 저하
② 복잡한 쿼리 → 다수 테이블 참조로 SQL 복잡도 증가
③ 인덱스 관리 부담 → 테이블 수 증가에 따른 인덱스 증가

2. 역정규화(Denormalization) 개념 및 적용 기준

역정규화(Denormalization)란 정규화로 분해된 릴레이션을 의도적으로 합치거나 중복 데이터를 추가하여 조회 성능을 향상시키는 기법이다. 이는 설계 실수가 아닌 의식적 트레이드오프(Trade-off) 결정이다.

역정규화 기법내용적용 시 고려사항
테이블 병합자주 조인되는 1:1 또는 1:N 테이블을 하나로 합침갱신 이상 재발 가능성 검토 필수
컬럼 중복 추가자주 참조되는 FK 컬럼 값을 해당 테이블에 직접 저장원본 변경 시 동기화 로직 필요
파생 컬럼 추가집계값(합계·평균 등)을 미리 계산해 컬럼으로 저장데이터 변경 시 자동 갱신(트리거) 설계
파티셔닝대용량 테이블을 수직/수평으로 분할하여 접근 범위 축소파티션 키 설계, 쿼리 라우팅 전략 수립

【표 8】 정규화 vs. 역정규화 적용 판단 기준
Ⅴ. 결론

함수 종속성 이론은 관계형 데이터베이스 설계의 수학적 토대로, 이를 기반으로 한 정규화는 데이터 중복 제거와 이상 현상 방지라는 본질적 목표를 달성한다. 실무에서는 3NF 또는 BCNF를 목표 설계 기준으로 삼되, 성능 임계점에 도달한 특정 경로에 대해서만 역정규화를 선택적으로 적용하는 균형 잡힌 전략이 요구된다. 올바른 정규화 수준의 선택은 시스템의 유지보수성·확장성·성능 간의 균형을 결정하는 핵심 설계 역량이다.

★ 기술사 핵심 코멘트
정규화의 본질은 함수 종속성에 의한 이상 현상 제거이며, DB 설계 감리 시 다음을 중점 검토해야 한다.
① 정규형 적정성: 핵심 트랜잭션 테이블이 최소 3NF를 만족하는지, BCNF 위반(후보키가 아닌 결정자) 여부 확인
② 무손실 분해(Lossless Decomposition): 정규화 분해 후 자연 조인으로 원본 릴레이션이 완전 복원되는지 검증
③ 종속성 보존(Dependency Preservation): 분해 후 원래 FD가 각 분해 릴레이션에서 여전히 검증 가능한지 확인 (BCNF 분해 시 종속성 보존이 불가한 경우 존재)
④ 역정규화 문서화: 역정규화 적용 시 반드시 사유·동기화 전략·영향 범위를 설계 문서에 명시하여 감리 추적 가능성 확보
