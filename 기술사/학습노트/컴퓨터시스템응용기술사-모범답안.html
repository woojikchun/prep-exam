Ⅰ. 개요 — OSI 7계층 모델의 개념
OSI(Open Systems Interconnection) 참조 모델은 1984년 ISO가 제정한 네트워크 통신 표준으로, 이기종 시스템 간의 상호 연결을 위해 통신 기능을 7개의 계층으로 계층화·모듈화한 참조 아키텍처이다. 각 계층은 독립적으로 동작하며 인접 계층과만 인터페이스를 공유함으로써 유지보수성과 확장성을 보장한다.

계층명칭주요 기능PDU
7응용 계층 (Application)사용자·앱에 네트워크 서비스 제공. HTTP, FTP, SMTP 등 동작메시지
6표현 계층 (Presentation)데이터 형식 변환(ASCII↔EBCDIC), 암호화/복호화, 압축·해제메시지

4전송 계층 (Transport)종단 간 신뢰성 있는 데이터 전송, 흐름·오류·혼잡 제어, 포트 기반 다중화세그먼트(TCP) / 데이터그램(UDP)

OSI 7계층TCP/IP 4계층특징
응용(7)·표현(6)·세션(5)응용 계층TCP/IP는 세 계층을 단일 계층으로 통합. 실용성 중시

📌 캡슐화(Encapsulation) 과정
OSI 7→1 방향(송신 측): 상위 계층 데이터에 각 계층 헤더(또는 트레일러)를 순서대로 부착하여 하위 계층으로 전달.
수신 측(1→7)은 역캡슐화(Decapsulation)를 통해 헤더를 제거하며 상위 계층으로 전달한다.
송신: [데이터] → [세그먼트:TCP헤더+데이터] → [패킷:IP헤더+세그먼트] → [프레임:MAC헤더+패킷+FCS] → [비트 스트림 전송] 수신: [비트 스트림] → FCS 검증/MAC헤더 제거 → IP헤더 제거 → TCP헤더 제거 → 원본 데이터 복원
Ⅴ. 결론 및 발전 방향

OSI 7계층은 이기종 시스템 간 호환성 확보를 위한 개념적 참조 모델로서 네트워크 설계·문제 진단의 기준이 되며, TCP/IP는 실제 인터넷의 사실상 표준(de facto standard)으로 동작한다. SDN·NFV 환경에서는 계층 간 경계가 소프트웨어로 추상화되어 운영되며, Wi-Fi 7·6G 등 차세대 통신 표준도 OSI 계층 구조를 기반으로 설계된다.

Ⅰ. 개요 — PKI의 개념
PKI(공개키 기반 구조)는 공개키 암호화 기술을 기반으로 디지털 인증서의 발급·관리·폐지를 통해 인터넷상에서 안전한 통신, 전자서명, 신원 인증을 가능하게 하는 복합 보안 인프라이다. X.509 표준을 기반으로 하며, 제3의 신뢰기관(CA)을 중심으로 신뢰 체계를 형성한다.
Ⅱ. 가. PKI 구성요소
  • 1
    CA(인증기관): 디지털 인증서를 발급·서명·폐지하는 최상위 신뢰기관. 루트 CA → 중간 CA 계층 구조 형성. (국내: KISA, 금융결제원)
  • 2
    RA(등록기관): 사용자 신원 검증 후 CA에 인증서 발급 요청 위임. CA와 최종 개체 사이 중계 역할
  • 3
    디지털 인증서(X.509): 주체 공개키·신원·유효기간·CA 서명을 포함한 전자 문서. 버전·일련번호·서명 알고리즘·발급자·공개키 포함
  • 4
    CRL / OCSP: 인증서 폐지 목록(CRL) 및 실시간 폐지 확인 프로토콜(OCSP)을 통해 인증서 유효성 실시간 검증
  • 5
    저장소(Repository): LDAP 등 디렉터리 서비스를 통해 공개된 인증서와 CRL을 배포·저장

공개키 암호화 원리 (기밀성)
① 수신자가 공개키(PK)·개인키(SK) 쌍 생성 → ② 송신자가 수신자의 공개키로 평문 암호화 → ③ 수신자만이 자신의 개인키로 복호화

Ⅰ. 개요 — 교착상태의 개념
교착상태(Deadlock)란 둘 이상의 프로세스가 서로 상대방이 점유한 자원을 무한정 대기하면서, 어떤 프로세스도 진행하지 못하는 무한 대기 상태를 의미한다. 1965년 Dijkstra가 정의한 개념으로, 운영체제 자원 관리의 핵심 문제이다.
Ⅱ. 가. 교착상태 4가지 필요조건 (Coffman, 1971)
  • 1
    상호 배제(Mutual Exclusion): 자원은 동시에 하나의 프로세스만 사용 가능. 공유 불가 자원에 한해 성립
  • 2
    점유와 대기(Hold and Wait): 자원을 최소 하나 점유한 상태에서 다른 프로세스가 점유한 추가 자원을 요청·대기
  • 3
    비선점(No Preemption): 프로세스가 사용 중인 자원을 OS가 강제 회수 불가. 자발적 반납만 허용
  • 4
    환형 대기(Circular Wait): {P0,P1,...,Pn}에서 P0→P1→...→Pn→P0 형태의 순환 대기 체인 형성
핵심 암기
4가지 조건이 모두 동시에 충족될 때만 교착상태 발생. 하나라도 부정하면 교착상태 불가 → 예방(Prevention)의 기본 원리
Ⅲ. 나. 교착상태 해결 방법 비교
기법접근 방식장점단점대표 알고리즘
예방 (Prevention)4가지 조건 중 하나 원천 차단교착상태 불가자원 낭비, 처리율 저하—
회피 (Avoidance)자원 할당 전 안전 상태(Safe State) 검사자원 이용률 향상최대 자원 요구량 사전 신고 필요은행원 알고리즘
탐지 (Detection)발생 허용 후 주기적 탐지자원 이용률 최대탐지 비용, 회복 오버헤드자원 할당 그래프
회복 (Recovery)탐지 후 프로세스 종료 또는 자원 선점교착상태 해소작업 손실, 기아(Starvation) 가능롤백, 희생자 선택
Ⅳ. 다. 자원 할당 그래프(RAG)를 이용한 탐지
자원 할당 그래프(Resource Allocation Graph) [표기 규칙] ○ → □ : 요청 간선 (프로세스가 자원을 요청) □ → ○ : 할당 간선 (자원이 프로세스에 할당됨) [교착상태 탐지 예시] P1 ─요청→ R2 ─할당→ P2 P2 ─요청→ R1 ─할당→ P1 ↑_________↑ ← 사이클(환형) 형성 = 교착상태 확정 ✓ 단일 인스턴스 자원: 사이클 존재 → 교착상태 확정 다중 인스턴스 자원: 사이클 존재 + 대기 불가 프로세스 無 → 교착상태 확정
탐지 알고리즘은 Wait-for 그래프에서 DFS·BFS를 통해 사이클을 검출한다. 탐지 후 회복은 ①교착 관련 프로세스 전부 종료, ②희생자 선택 후 순차 종료, ③자원 선점(Preemption) 중 선택한다.
Ⅴ. 결론

운영체제는 일반적으로 탐지+회복 또는 회피(은행원 알고리즘) 전략을 채택한다. 현대 DBMS·클라우드 환경에서는 분산 교착상태(Distributed Deadlock)가 더욱 복잡한 문제로 대두되며, 타임아웃 기반 탐지와 분산 Wait-for 그래프 기법이 활용된다.

RAG(검색 증강 생성)는 대규모 언어 모델(LLM)의 생성(Generation) 단계에 외부 지식 저장소에서 관련 문서를 실시간 검색(Retrieval)하여 맥락으로 주입하는 하이브리드 AI 아키텍처이다. 2020년 Meta(Lewis et al.)가 제안하였다.
RAG 등장 배경 — LLM 3대 한계
① 환각(Hallucination): 사실과 다른 내용을 자신감 있게 생성
② 지식 단절(Knowledge Cutoff): 학습 데이터 시점 이후 정보 반영 불가
③ 도메인 지식 부재: 기업·공공 내부 문서 기반 응답 불가
Ⅱ. 나. RAG 아키텍처 및 동작 원리
RAG 아키텍처 (Naive RAG 기준) [오프라인 인덱싱] 원본 문서 → 청킹(Chunking) → 임베딩(Embedding) → 벡터 DB 저장 (Pinecone / Chroma / FAISS) [온라인 추론] 사용자 질문(Query) → 질문 임베딩 → 벡터 DB 유사도 검색(Top-K) → 관련 청크 Retrieve → 프롬프트에 컨텍스트 삽입 → LLM 생성(Generation) → 최종 답변 반환
  • 1
    문서 청킹(Chunking): 원본 문서를 의미 단위(512~1024 토큰)로 분할. 오버랩 청킹으로 문맥 단절 방지
  • 2
    임베딩(Embedding): 각 청크를 고차원 벡터로 변환 (text-embedding-ada-002, BGE-M3 등)
  • 3
    벡터 유사도 검색: 코사인 유사도·내적 기반 Top-K 청크 검색
  • 4
    증강 프롬프팅: 검색된 문서를 시스템 프롬프트에 컨텍스트로 삽입 후 LLM 호출

Ⅳ. 결론

RAG는 LLM의 실용적 한계를 해결하는 핵심 기술로, 공공·금융·의료 분야에서 신뢰성 있는 AI 서비스 구현의 사실상 표준으로 자리잡고 있다. 향후 Advanced RAG(재순위화, 쿼리 분해)와 Agentic RAG(도구 호출·멀티홉 추론) 방향으로 발전하며, 초거대 AI 공공 도입 가이드라인에서도 환각 방지 수단으로 명시되고 있다.

투입공수 방식 산정 공식
사업대가 = 직접인건비 + 직접경비 + 제경비 + 기술료 + 부가가치세

• 직접인건비 = 투입 M/M × 직종별 노임단가 (한국SW산업협회 고시 기준)
• 제경비 = 직접인건비 × 110~120%
