Ⅰ. 방화벽의 개념 및 기본 동작 원리
방화벽(Firewall)은 신뢰할 수 있는 내부 네트워크와 신뢰할 수 없는 외부 네트워크(인터넷) 사이에 위치하여, 미리 정의된 보안 정책(ACL)에 따라 트래픽을 허용(Allow)·차단(Deny)·폐기(Drop)하는 네트워크 보안 장비이다. Default Deny(기본 거부) 원칙에 따라 명시적으로 허용된 트래픽만 통과시키는 Positive Security Model을 채택한다.
방화벽 기본 동작 흐름 외부 트래픽 유입 │ ▼ [정책 매칭] → 출발지 IP · 목적지 IP · 포트 · 프로토콜 확인 │ ├─ 허용(Allow) → 패킷 전달 ├─ 거부(Deny) → 패킷 폐기 + 이벤트 로그 기록 └─ 묵시적 거부 → Default Deny (정책 미매칭 시 자동 차단)

(응용)		프록시로서 클라이언트-서버 사이에서 애플리케이션 프로토콜 완전 분석. HTTP/FTP/SMTP 등 특정 프로토콜 심층 검사애플리케이션 계층 완전 검사, 콘텐츠 필터링, 내부 IP 은닉처리 속도 느림, 프로토콜별 별도 프록시 필요, 새 서비스 대응 어려움
차세대 방화벽
(NGFW)		2~7계층
(전 계층 통합)		스테이트풀 + DPI(심층 패킷 검사) + IPS + 애플리케이션 인식 + 사용자 ID 기반 정책 + SSL 복호화 통합APT 대응, 암호화 트래픽 검사, 단일 장비로 통합 보안고성능 요구, 높은 비용, SSL 복호화 시 개인정보 이슈

방화벽 세대별 발전 1세대 (1988~) : 패킷 필터링 → IP/Port 기반 단순 ACL 2세대 (1989~) : 스테이트풀 인스펙션 → 세션 상태 추적 (Check Point FW-1) 3세대 (1994~) : 애플리케이션 게이트웨이(Proxy) → L7 프로토콜 분석 4세대 (2007~) : NGFW → DPI + IPS + App-ID + User-ID + SSL 복호화 (Palo Alto Networks)

④ 통합 위협 관리(UTM): IPS·Anti-Virus·URL 필터링 통합
Ⅳ. 결론 — 감리 점검항목

Ⅰ. IDS 개념 및 탐지 방식 분류
IDS(Intrusion Detection System, 침입탐지시스템)는 네트워크 또는 호스트에서 비인가 접근·악의적 활동을 실시간으로 탐지하고 관리자에게 경보를 발생시키는 수동적 보안 장비이다. 탐지 방식은 크게 오용 탐지와 이상 탐지로 분류된다.
Ⅱ. 오용 탐지 vs 이상 탐지 비교
비교 항목오용 탐지 (Misuse Detection)
시그니처 기반		이상 탐지 (Anomaly Detection)
행위 기반

탐지 대상알려진 공격(Known Attack): CVE 취약점, 악성코드, 스캔 패턴알려지지 않은 공격(Zero-day), 내부자 위협, APT

구분정의보안 영향대응 방안
오탐
(False Positive)		정상 트래픽을 공격으로 잘못 탐지하여 불필요한 경보 발생담당자 경보 피로(Alert Fatigue), 업무 차단, 신뢰도 저하시그니처 튜닝, 화이트리스트 적용, 임계값 조정
미탐
(False Negative)		실제 공격을 정상으로 판단하여 탐지 누락침해사고 미탐지, 보안 공백 발생, 치명적 위험시그니처 DB 최신화, 탐지 규칙 강화, 계층 방어
📌 탐지율·오탐율 트레이드오프 (ROC Curve)
탐지 임계값을 낮추면 → 탐지율(DR) 향상, 오탐율(FPR) 증가
탐지 임계값을 높이면 → 오탐율 감소, 미탐율(FNR) 증가
최적 운영점: ROC 곡선 상에서 탐지율·오탐율 균형점 선택. AUC(Area Under Curve) 지표로 탐지 성능 평가
Ⅳ. 하이브리드 탐지 방식 (최신 트렌드)
최신 IDS/IPS는 오용 탐지의 정확성과 이상 탐지의 제로데이 대응력을 결합한 하이브리드 방식을 채택한다. AI/ML 기반 이상 탐지로 의심 행위를 1차 선별하고, 시그니처 엔진으로 2차 정밀 분석하여 오탐을 최소화한다.
Ⅴ. 결론 — 감리 포인트

IDS 탐지 방식 선택 시: ①알려진 공격 중심 환경 → 오용 탐지 중심, ②내부자 위협·APT 대응 → 이상 탐지(UEBA) 추가 권장. 감리 시에는 ①시그니처 DB 최종 업데이트 일자, ②오탐율 통계·튜닝 이력, ③탐지 정책 검토 주기, ④주요 경보에 대한 대응 절차(Playbook) 존재 여부를 점검한다.

Ⅰ. DMZ의 개념
DMZ(DeMilitarized Zone, 비무장지대)는 외부 인터넷과 내부 내부망 사이에 설정된 중립 네트워크 구간으로, 외부에서 접근이 필요한 서비스(웹서버·메일서버 등)를 내부망으로부터 격리하여 배치하는 네트워크 보안 아키텍처이다. DMZ 서버가 침해되더라도 내부망으로의 추가 침투를 방어하는 계층 방어(Defense-in-Depth) 전략이다.
Ⅱ. DMZ 구성 방식 비교
① 싱글 방화벽 구조 (3-legged Firewall) 인터넷 │ [방화벽 - 인터페이스 3개] ├── DMZ 구간 (eth1) ── [웹서버] [메일서버] [DNS] └── 내부망 (eth2) ── [DB서버] [내부시스템] 장점: 구성 단순, 비용 낮음 단점: 방화벽 단일 장애점(SPOF), 방화벽 침해 시 내부망 노출 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ ② 듀얼 방화벽 구조 (Screened Subnet) 인터넷 │ [외부 방화벽] ── 외부→DMZ 트래픽 통제 │ [ DMZ 구간 ] ── [웹서버] [메일서버] [리버스 프록시] │ [내부 방화벽] ── DMZ→내부망 트래픽 통제 │ [ 내부망 ] ── [DB서버] [내부 애플리케이션] 장점: 이중 보안 계층, DMZ 침해 시 내부망 보호 단점: 비용·관리 복잡성 증가. 공공 기관·금융권 권장 구조
Ⅲ. DMZ 배치 서비스 및 트래픽 정책

Ⅰ. 개요
방화벽 정책(Firewall Policy)은 네트워크를 통과하는 트래픽에 대한 허용·차단 기준을 정의한 보안 규칙의 집합이다. 공공기관은 「전자정부법」·「정보보안관리 실태평가 기준」(국가정보원)에 따라 방화벽을 필수 보안 장비로 구축하고 체계적인 정책 관리가 요구된다.

IDS/IPS는 방화벽이 차단하지 못하는 허용된 포트를 통한 공격(SQL Injection, 웜 전파, 내부자 위협)을 탐지·차단하는 보완적 보안 장비이다. 공공기관은 「정보보안관리 실태평가」에서 침입탐지·방지 시스템 구축을 필수 평가 항목으로 규정하고 있다.

Ⅰ. 가. APT 개념·Kill Chain·기존 보안의 한계
APT(Advanced Persistent Threat, 지능형 지속 공격)는 특정 조직·국가를 목표로 장기간(수개월~수년)에 걸쳐 은밀하게 침투하여 정보를 탈취하거나 시스템을 파괴하는 고도화된 사이버 공격이다. 전통적 방화벽·시그니처 기반 IDS는 APT의 제로데이 취약점 악용과 암호화 C2 통신을 탐지하는 데 구조적 한계가 있다.
Cyber Kill Chain (Lockheed Martin, 7단계) 1. Reconnaissance (정찰) : 표적 조직 정보 수집 (OSINT·소셜미디어) 2. Weaponization (무장화): 악성코드·익스플로잇 제작 (제로데이+페이로드) 3. Delivery (전달) : 이메일 첨부·USB·워터링홀로 악성코드 전달 4. Exploitation (취약점 악용): 취약점 실행으로 시스템 침투 (Zero-day) 5. Installation (설치) : 지속성 확보 (백도어·루트킷·레지스트리 등록) 6. C2 Communication(C2 통신): 공격자 서버와 암호화 통신 채널 수립 7. Actions on Objectives(목적달성): 데이터 탈취·파괴·랜섬웨어 실행

APT 대응 다층 방어(Defense-in-Depth) 아키텍처 [계층 1 — 네트워크 경계] NGFW (방화벽·DPI) + IPS (시그니처·이상 탐지) → 알려진 공격 패턴·비허가 포트 1차 차단 [계층 2 — 이메일·웹 게이트웨이] SEG (보안 이메일 게이트웨이): 악성 첨부·피싱 차단 SWG (보안 웹 게이트웨이): 악성 URL·다운로드 차단 [계층 3 — 샌드박스 (동적 분석)] 의심 파일·URL → 격리 가상 환경에서 실행 후 행위 분석 → 알려지지 않은 악성코드 탐지 (제로데이 대응) [계층 4 — 엔드포인트 (EDR)] EDR (Endpoint Detection & Response): 프로세스·메모리·API 호출 실시간 모니터링 → 설치(Installation)·C2 통신 단계 탐지·차단 [계층 5 — 네트워크 탐지 (NDR)] NDR (Network Detection & Response): 내부 트래픽 AI 분석 → C2 통신·횡적 이동(Lateral Movement) 행위 탐지 [계층 6 — 통합 분석 (SIEM/SOAR)] 전 계층 로그·이벤트 통합 수집 → 상관 분석(Correlation)으로 APT 공격 체인 탐지 → SOAR 자동 대응 플레이북 실행
Kill Chain 단계대응 보안 장비탐지·차단 방법
정찰 (1단계)웹 방화벽(WAF), IPS외부 포트 스캔·디렉터리 열거 탐지, 허니팟 배치

취약점 악용 (4단계)IPS, EDR(HIPS)익스플로잇 시그니처, 메모리 보호(DEP·ASLR)
설치 (5단계)EDR, HIDS비정상 프로세스·레지스트리 변경 탐지

Ⅲ. 다. 위협 인텔리전스와 MITRE ATT&CK 활용
위협 인텔리전스(Threat Intelligence)는 알려진 공격자 TTP(전술·기술·절차), IoC(침해 지표: 악성 IP·도메인·해시), 취약점 정보를 수집·분석하여 선제적 방어에 활용하는 정보이다.
  • 1
    MITRE ATT&CK 프레임워크: 실제 APT 그룹의 TTP를 14가지 전술(Tactic)·수백 개의 기술(Technique)로 체계화한 지식 베이스
    14개 전술: Reconnaissance·Resource Development·Initial Access·Execution·Persistence·Privilege Escalation·Defense Evasion·Credential Access·Discovery·Lateral Movement·Collection·C2·Exfiltration·Impact
    활용: IDS/IPS 탐지 규칙을 ATT&CK 기술 번호(T-ID)로 매핑 → 탐지 커버리지 가시화
  • 2
    IoC 기반 자동 차단: STIX/TAXII 표준으로 위협 인텔리전스 공유. SIEM·방화벽·IPS에 악성 IP·도메인 자동 반영 (Threat Intel Feed)
  • 3
    레드팀·퍼플팀 연계: ATT&CK 기반 공격 시뮬레이션(Atomic Red Team)으로 탐지 공백 식별 후 탐지 규칙 보강
Ⅳ. 결론

APT 공격은 단일 보안 장비로 대응이 불가능하며, Kill Chain 각 단계별 다층 방어 아키텍처와 MITRE ATT&CK 기반 탐지 규칙 정렬이 필수이다. 공공기관은 국가사이버안보센터(NCSC)의 사이버위협 인텔리전스 공유 체계(KrCERT/CC)를 활용하고, SIEM-SOAR 통합으로 대응 자동화를 구현하여 APT 대응 역량을 강화해야 한다.

Ⅳ. 결론

SIEM-SOC-SOAR의 통합 보안 운영 체계는 방화벽·IDS/IPS 등 개별 보안 장비의 사각지대를 해소하고 신속한 침해사고 대응을 가능하게 한다. 공공기관은 「정보보호 및 개인정보보호 관리체계(ISMS-P)」 인증 요건에 따라 사고 대응 체계를 필수 구축해야 하며, SOAR 자동화 도입으로 분석관 역량을 고부가가치 위협 헌팅에 집중시키는 성숙한 보안 운영 모델로 발전시켜야 한다.
